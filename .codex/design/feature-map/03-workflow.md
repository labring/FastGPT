# 工作流编排功能介绍

工作流编排是 FastGPT 构建复杂 AI 应用的核心能力。用户通过可视化节点、连线、变量引用和调试功能，将模型、知识库、工具、条件分支和用户交互组合成完整业务流程。

## Flow 编辑器

Flow 编辑器提供节点拖拽、连线、复制、删除、搜索、折叠、自动对齐、演示模式和编辑模式。它将复杂应用逻辑以图形方式表达，降低维护成本。

## 节点输入输出

每个节点可以声明输入和输出。输入支持固定值、引用其他节点输出、动态变量、模型选择、知识库选择、文件选择、时间选择、密码输入等类型。输出支持静态、动态、错误和隐藏输出。

## 变量引用

工作流支持引用上游节点输出、系统输入、聊天历史、知识库引用、全局变量和外部变量。变量引用是节点间传递数据的基础。

## 调试与运行状态

工作流调试会检查节点配置和连线，运行后展示节点状态、跳过状态、运行结果和错误信息。调试结果可以帮助开发者快速定位配置错误和运行异常。

## 错误捕获

部分节点支持错误输出或错误文本，用于在 API 失败、工具失败、模型失败时进入兜底逻辑，提升生产应用稳定性。

## AI 对话节点

AI 对话节点负责调用大模型生成回答。它可配置模型、系统提示词、聊天历史、温度、最大 token、工具调用、知识库引用等参数，是对话类应用的核心节点。

## Agent 节点

Agent 节点用于规划和执行多步骤任务。它可以结合工具、知识库和文件能力，将用户目标拆解为多个子任务并逐步完成。

## 知识库搜索节点

知识库搜索节点负责从一个或多个知识库中检索相关内容，支持向量检索、全文检索、混合检索、重排、引用提示词和元数据过滤。

## 知识库搜索结果合并 / 文本拼接节点

该类节点用于合并多个知识库搜索结果或拼接文本内容，常见于多源资料整合、上下文压缩和后续模型生成前的数据准备。

## 指定回复节点

指定回复节点直接输出固定内容或引用变量内容，适合欢迎语、条件分支结果、兜底回复和流程终止提示。

## 问题分类节点

问题分类节点使用模型将用户输入归入预设类别，并根据分类结果进入不同分支，适合客服分流、意图识别和多业务线问答。

## 内容提取节点

内容提取节点从文本中抽取结构化字段，适合表单识别、合同信息提取、简历解析、工单字段抽取等场景。

## HTTP 请求节点

HTTP 请求节点用于调用外部 API。它支持配置 URL、方法、Headers、Params、Body、变量注入和输出提取，是连接业务系统的主要方式。

## 工具调用节点

工具调用节点将系统工具、HTTP 工具、MCP 工具或工作流工具作为可调用能力，用于让 Agent 或流程访问外部功能。

## 工具参数配置节点

工具参数配置节点用于声明工具输入参数，让工作流工具可以被其他应用识别和调用。

## 工具调用终止节点

工具调用终止节点用于中断工具调用链路，适合在工具流程中提前返回或阻止后续执行。

## 文件读取 / 文档解析节点

文件读取节点用于读取用户上传文件或流程中产生的文件，将文件内容转换为可被模型和后续节点消费的文本或结构化数据。

## 指代消解 / 问题优化节点

该节点用于结合聊天历史优化用户问题，解决多轮对话中的代词、缩写和上下文依赖问题，提高知识库检索和模型回答质量。

## Laf 函数调用节点

Laf 节点用于调用 Laf 云函数，将工作流与自定义云端逻辑结合，适合快速扩展业务函数。

## 条件分支节点

条件分支节点根据变量值进行判断，支持相等、不相等、为空、不为空、包含、数值比较、长度比较等条件，控制工作流走向。

## 变量更新节点

变量更新节点用于更新工作流变量值，支持流程中的状态记录、循环计数、临时结果存储和跨节点数据传递。

## 代码沙盒执行节点

代码沙盒节点用于运行自定义代码，适合复杂数据处理、格式转换、计算、文件生成和工具补足。沙盒能力依赖代码沙箱服务。

## 并行运行节点

并行运行节点用于同时执行多个分支，减少总耗时，适合多知识库查询、多工具调用和多模型并发处理。

## 循环运行与跳出循环

循环运行节点用于重复执行子流程。跳出循环节点用于满足条件时提前结束循环，适合批量处理、分页抓取和多轮迭代任务。

## 用户选择节点

用户选择节点在流程中暂停并让用户选择选项，适合确认、分流、人工决策和多路径交互。

## 表单输入节点

表单输入节点用于在流程中向用户收集结构化输入，适合预约、报名、工单提交和业务信息补充。

## 自定义反馈节点

自定义反馈节点允许应用在回答后收集特定反馈内容，并将反馈结果纳入日志和运营分析。

## 插件输入 / 插件输出节点

插件输入和输出节点用于定义工作流工具的入参和出参，使工作流可以被封装为工具或插件供其他应用调用。

## 调用应用与调用插件

调用应用和调用插件节点用于在一个工作流中复用其他应用或插件，形成模块化的流程组合。
