# 模型与 AI 配置功能介绍

模型与 AI 配置模块负责管理大模型、模型渠道、默认参数、价格、能力标签和调用监控。它是 FastGPT 多模型适配和商业计费的基础。

## 可视化模型配置

管理员可以通过界面配置模型，而不必直接修改配置文件。可视化配置降低自部署和商业环境中的模型接入门槛。

## 模型列表与详情

模型列表展示当前可用模型。详情页用于查看模型 ID、别名、请求地址、认证方式、能力、价格和用途。

## 创建、编辑与删除模型

模型可以新增、修改或删除。编辑时可配置请求 URL、默认 token、请求认证、参数、价格和模型能力。

## 默认模型配置

默认模型配置用于指定系统在不同场景下默认使用的模型，例如对话、评测、知识库处理、分类和字段提取。

## JSON 配置导入

JSON 模型配置用于批量导入或更新模型，适合自部署环境快速迁移完整模型配置。

## 模型测试

模型测试用于验证模型渠道是否可用、响应是否正常、参数是否正确和返回格式是否符合预期。

## 模型能力配置

模型能力包括 Function Call、Tool Choice、视觉、多模态、Reasoning、Response Format、内容审核、Normalization、Top P、停止符等。能力配置会影响前端可选项和运行时行为。

## 模型价格配置

模型可配置输入价格、输出价格、字符价格和阶梯价格。价格配置用于用量统计、积分消耗和成本核算。

## 模型用途配置

模型可以声明是否用于评测、分类、字段提取、工具调用、数据处理等任务。这样可以避免用户在不适合的场景中选择错误模型。

## AI Proxy 渠道

AI Proxy 渠道用于管理模型供应商通道，包括渠道名称、类型、Base URL、API Key、优先级和状态。

## 渠道限流

渠道可配置 RPM 和 TPM 限制，用于避免超过模型供应商限制，并在多渠道场景中实现负载控制。

## 渠道状态

渠道状态包括启用、禁用、自动禁用和未知。系统可以根据错误情况或管理员操作控制渠道可用性。

## 模型监控

监控能力包括总调用量、失败调用量、错误率、平均响应时间、平均 TTFB、Token 使用和缓存命中率。

## 模型调用日志

调用日志支持按 request id 搜索，查看请求时间、状态、耗时、重试次数、模型、渠道和错误详情，用于排查模型供应商或配置问题。
