## Automated API Documentation

The automated API documentation covers all endpoints in the current version, regardless of whether they can be called via API Key.

All future endpoints will be auto-generated, with documentation continuously improved.

- [China API Documentation](https://cloud.fastgpt.cn/openapi)

## Usage Guide

FastGPT OpenAPI lets you authenticate with an API Key to access FastGPT services and resources, such as: calling app chat endpoints, uploading knowledge base data, testing searches, and more. For compatibility and security reasons, not all endpoints support API Key access.

## How to Get an API Key

FastGPT has **2 types** of API Keys: a global key (cannot directly call app chat) and an app-specific key with an AppId (can directly call app chat).

We recommend using `app-specific keys` only for app or chat-related endpoints, and `global keys` for everything else.

## Basic Configuration

In OpenAPI, all endpoints authenticate via Header.Authorization.

```
baseUrl: "http://localhost:3000/api"
headers: {
    Authorization: "Bearer {{apikey}}"
}
```

**Example: Start an App Chat**

```sh
curl --location --request POST 'http://localhost:3000/api/v1/chat/completions' \
--header 'Authorization: Bearer fastgpt-xxxxxx' \
--header 'Content-Type: application/json' \
--data-raw '{
    "chatId": "111",
    "stream": false,
    "detail": false,
    "messages": [
        {
            "content": "Who is the director",
            "role": "user"
        }
    ]
}'
```

## Custom User ID

Since `v4.8.13`, you can pass a custom user ID that will be saved in the chat history.

```sh
curl --location --request POST 'http://localhost:3000/api/v1/chat/completions' \
--header 'Authorization: Bearer fastgpt-xxxxxx' \
--header 'Content-Type: application/json' \
--data-raw '{
    "chatId": "111",
    "stream": false,
    "detail": false,
    "messages": [
        {
            "content": "Who is the director",
            "role": "user"
        }
    ],
    "customUid": "xxxxxx"
}'
```

In the chat history, this record will show the user as `xxxxxx`.
