## Features

- User interaction
- Can be added multiple times
- Trigger-based execution

![](/imgs/form_input1.png)

## Function

The "Form Input" node is a user interaction node. When this node is triggered, the conversation enters an "interactive" state, recording the workflow state and continuing execution after the user completes the interaction.

![](/imgs/form_input2.png)

In the example above, when the form input node is triggered, the chat box is hidden and the conversation enters "interactive state."

![](/imgs/form_input3.png)

After the user fills in required information and clicks submit, the node collects the form data and passes it to subsequent nodes.

## Use Cases

Precisely collect required user information, then perform subsequent operations based on that information.
