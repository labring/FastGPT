# FastGPT 外部接入指南

本文档介绍如何通过各种方式将外部系统接入 FastGPT，实现应用调用、知识库查询、工作流执行等功能。

## 一、接入概述

FastGPT 提供多种外部接入方式，满足不同场景需求：

| 接入方式 | 协议 | 场景 | 特点 |
|---------|------|------|------|
| **HTTP API** | REST + OpenAI 兼容 | 通用集成、第三方系统调用 | 标准 HTTP，易于集成，支持流式/非流式 |
| **MCP 服务器** | Model Context Protocol | AI 工具集成、Agent 扩展 | 标准化工具协议，支持工具发现与调用 |
| **预签名 URL** | S3 兼容协议 | 文件上传下载 | 安全的临时访问凭证 |
| **分享链接** | Web 嵌入、访问控制 | 外部用户集成、知识库分享 | 无需认证，支持自定义 UID |

---

## 二、HTTP API 接入

### 2.1 认证方式

FastGPT 支持三种认证方式：

#### 2.1.1 API Key 认证（推荐）

**获取方式**：
1. 登录 FastGPT Web 应用
2. 进入 **个人中心 → API 密钥** 创建团队级 Key（用于跨应用访问）
3. 或在 **应用详情 → 接入 → API 接入** 创建应用级 Key（推荐，自动绑定 appId）

**Key 格式**：`fastgpt-xxxxxxxxxxxxx`

**请求头格式**：
```http
Authorization: Bearer fastgpt-xxxxxxxxxxxxx
Content-Type: application/json
```

#### 2.1.2 Cookie 认证（Web 浏览器）

浏览器访问时会自动携带登录 Cookie，无需显式设置。

#### 2.1.3 分享链接认证

通过分享链接的 `shareId` 和 `outLinkUid` 进行认证，无需 Bearer Token，适合外部嵌入场景。

### 2.2 基础 URL

- **自建部署**：`http(s)://your-domain/api`
- **云端 FastGPT**：`https://api.fastgpt.io/api`

### 2.3 聊天补全接口（核心对外接口）

**路径**：`POST /api/v1/chat/completions`

**简述**：兼容 OpenAI ChatCompletion API 的对话接口，支持流式和非流式返回。可传入应用 ID（模型字段），系统自动加载应用工作流、知识库等配置。

#### 2.3.1 请求体

```json
{
  "model": "<App ID or Application Key>",
  "messages": [
    {"role": "user", "content": "你好"}
  ],
  "stream": false,
  "chatId": "xxx",
  "variables": {},
  "detail": false
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `model` | string | 是 | 应用 ID（MongoDB ObjectId）或应用 API Key。使用应用级 Key 时可省略 `appId` |
| `messages` | array | 是 | OpenAI 兼容消息列表，每项包含 `role`（user/assistant/system）和 `content` |
| `stream` | boolean | 否 | 是否流式返回，默认 `false` |
| `chatId` | string | 否 | 会话 ID；传入则自动续接历史，未传视为新对话 |
| `appId` | string | 否 | 应用 ID；使用团队级 Key 时必填，应用级 Key 可不传 |
| `variables` | object | 否 | 工作流全局变量，默认 `{}` |
| `detail` | boolean | 否 | 是否返回详细信息（nodeResponses、newVariables），默认 `false` |
| `customUid` | string | 否 | 分享链接场景的自定义用户标识 |
| `shareId` | string | 否 | 分享链接 ID（用于分享链接认证） |
| `outLinkUid` | string | 否 | 分享链接用户 ID（用于分享链接认证） |
| `teamId` | string | 否 | 团队空间 Token 认证时的团队 ID |
| `teamToken` | string | 否 | 团队空间 Token（用于团队空间认证） |
| `responseChatItemId` | string | 否 | 自定义响应消息 ID，不传自动生成 |
| `retainDatasetCite` | boolean | 否 | 是否保留答案中的数据集引用标记，默认 `false` |
| `showSkillReferences` | boolean | 否 | 是否在响应中保留 Agent 技能引用，默认 `false` |
| `metadata` | object | 否 | 自定义元数据，落库到聊天记录 |

#### 2.3.2 响应体（非流式）

```json
{
  "id": "chat-uuid",
  "model": "",
  "usage": {
    "prompt_tokens": 1,
    "completion_tokens": 1,
    "total_tokens": 1
  },
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "您好！有什么我可以帮助您的吗？",
        "reasoning_content": null
      },
      "finish_reason": "stop",
      "index": 0
    }
  ],
  "responseData": null,
  "newVariables": null
}
```

**响应字段说明**：

| 字段 | 说明 |
|------|------|
| `id` | 聊天 ID（chatId），用于续接历史 |
| `model` | 模型名称，v1 接口固定为空字符串 |
| `usage` | Token 用量统计（v1 接口为占位值，详细用量见 `detail=true` 时的 `responseData`） |
| `choices[].message` | 助手回复消息，包含 `role`、`content`、可选的 `reasoning_content` |
| `choices[].finish_reason` | 完成原因，通常为 `stop`（正常完成）或 `max_tokens`（超过限制） |
| `responseData` | 各工作流节点详细响应（仅 `detail=true` 时返回） |
| `newVariables` | 工作流执行后更新的变量（仅 `detail=true` 时返回） |

#### 2.3.3 流式响应（Server-Sent Events）

当 `stream=true` 时，响应为 SSE 格式，包含多个事件：

```
data: {"event":"answer","text":"您好"}
data: {"event":"answer","text":"！有什么"}
data: [DONE]
```

**事件类型**：

| 事件 | 说明 |
|------|------|
| `answer` | 增量回答文本（兼容 OpenAI `choices[].delta.content`） |
| `fastAnswer` | 直接回答文本（无动画效果） |
| `flowNodeStatus` | 节点执行状态变更 |
| `flowNodeResponse` | 节点响应数据 |
| `toolCall` | Tool 调用信息 |
| `toolParams` | Tool 参数 |
| `toolResponse` | Tool 响应 |
| `flowResponses` | 工作流节点详细响应（`detail=true` 时） |
| `updateVariables` | 工作流变量更新 |
| `interactive` | 命中交互节点（需用户交互） |
| `error` | 错误信息 |
| `workflowDuration` | 工作流执行耗时 |
| `plan` | Agent 计划响应 |
| `stepTitle` | Agent 步骤标题 |
| `sandboxStatus` | 沙箱生命周期状态 |
| `skillCall` | 技能调用通知（SKILL.md 加载时） |
| `collectionForm` | HelperBot 集合表单 |
| `topAgentConfig` | TopAgent 配置表单 |
| `[DONE]` | 流式结束标记 |

#### 2.3.4 FastGPT 扩展参数

以下参数为 FastGPT 扩展，OpenAI SDK 可能不支持，需自定义 HTTP 请求：

```python
# Python 示例（使用自定义 HTTP 请求）
import requests
import json

response = requests.post(
    'https://your-domain/api/v1/chat/completions',
    headers={
        'Authorization': 'Bearer fastgpt-xxxxx',
        'Content-Type': 'application/json'
    },
    json={
        'model': 'app-id-xxx',
        'messages': [{'role': 'user', 'content': '你好'}],
        'stream': False,
        'variables': {'user_name': 'Alice'},  # 工作流变量
        'detail': True,  # 返回详细信息
        'metadata': {'user_id': '123'}  # 自定义元数据
    }
)
```

### 2.4 对话管理接口

#### 2.4.1 获取对话历史列表

**路径**：`POST /api/core/chat/history/getHistories`

**请求体**：
```json
{
  "appId": "<app-id>",
  "offset": 0,
  "pageSize": 20,
  "source": "online",
  "startCreateTime": "2024-01-01T00:00:00Z",
  "endCreateTime": "2024-12-31T23:59:59Z",
  "startUpdateTime": "2024-01-01T00:00:00Z",
  "endUpdateTime": "2024-12-31T23:59:59Z"
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `appId` | string | 否 | 应用 ID；可通过 Authorization header 自动指定 |
| `offset` | number | 否 | 偏移量（与 `pageNum` 二选一） |
| `pageNum` | number | 否 | 页码（与 `offset` 二选一） |
| `pageSize` | number | 否 | 每页条数 |
| `source` | string | 否 | 对话来源：`online`（在线）、`api`、`share`、`team` |
| `startCreateTime` | string | 否 | 创建时间起始（ISO 8601） |
| `endCreateTime` | string | 否 | 创建时间截止 |
| `startUpdateTime` | string | 否 | 更新时间起始 |
| `endUpdateTime` | string | 否 | 更新时间截止 |
| `shareId` | string | 否 | 分享链接认证 ID |
| `outLinkUid` | string | 否 | 分享链接用户标识 |

**响应示例**：
```json
{
  "code": 200,
  "statusText": "",
  "data": {
    "total": 10,
    "list": [
      {
        "chatId": "chat-uuid",
        "updateTime": "2024-01-15T10:30:00.000Z",
        "appId": "app-id",
        "customTitle": "自定义标题",
        "title": "用户初次提问",
        "top": false,
        "chatGenerateStatus": 1,
        "hasBeenRead": false
      }
    ]
  }
}
```

#### 2.4.8 恢复流式响应

**路径**：`GET /api/core/chat/resume`

**请求参数**：
```query
?appId=<app-id>&chatId=<chat-id>
```

**请求头**：
```http
Accept: text/event-stream
```

**说明**：恢复被中断的流式对话。已完成对话返回 JSON，仍生成中则返回 SSE 流。需在请求头中声明 `Accept: text/event-stream`，否则生成中的对话会返回 406。

**路径**：`POST /api/core/chat/history/getHistoryStatus`

**请求体**：
```json
{

}
```

**响应示例**：
```json
{
  "code": 200,
```

**说明**：批量查询对话的生成状态和已读状态，用于侧栏轻量轮询同步。单次最多查询 200 个对话 ID。

#### 2.4.10 批量删除对话历史

**路径**：`POST /api/core/chat/history/batchDelete`

**请求体**：
```json
{
  "appId": "<app-id>",
  "chatIds": ["chat-id-1", "chat-id-2"]
}
```

**说明**：批量真实删除指定应用的多个对话记录，需应用日志管理权限。

### 2.5 语音转写接口

**路径**：`POST /api/v1/audio/transcriptions`

**请求格式**：`multipart/form-data`

**字段**：
- `file`：音频文件（WAV、MP3 等）
- `appId`：应用 ID
- `duration`：音频时长（秒），用于计费
- `shareId`、`outLinkUid`、`teamId`、`teamToken`：可选认证字段

**响应**：
```json
{
  "code": 200,
  "statusText": "",
  "data": {
    "text": "转写出来的文本内容"
  }
}
```

**特性**：
- IP 频率限制：同一 IP 每秒最多 1 次转写
- 使用默认 STT 模型
- 按 `duration` 计费

## 三、文件上传接入

### 3.1 获取上传预签名 URL

FastGPT 使用 S3 兼容协议管理文件，支持通过预签名 URL 进行安全上传。

#### 3.1.1 临时文件上传预签名

**路径**：`POST /api/common/file/presignTempFilePostUrl`

**请求体**：
```json
{
  "filename": "document.pdf"
}
```

**响应示例**：
```json
{
  "code": 200,
  "statusText": "",
  "data": {
    "url": "https://s3.example.com/temp/...",
    "key": "temp/xxx/document.pdf",
    "headers": {
      "Content-Type": "application/octet-stream"
    },
    "maxSize": 10485760
  }
}
```

**使用步骤**：
1. 调用该接口获取预签名 URL
2. 使用 `PUT` 请求将文件上传到返回的 URL
3. 获得 `key`，用于后续知识库导入
#### 3.1.1.2 临时文件下载预签名

**路径**：`GET /api/common/file/presignTempFileGetUrl`

**请求参数**：
```query
?key=<file-key>
```

**响应示例**：
```json
{
  "code": 200,
  "statusText": "",
  "data": {
    "url": "https://s3.example.com/temp/..."
  }
}
```

**说明**：获取临时文件的下载预签名 URL，用于通过 `GET` 请求直接下载文件。

#### 3.1.2 知识库文件上传预签名

**路径**：`POST /api/core/dataset/file/presignDatasetFilePostUrl`

**请求体**：
```json
{
  "datasetId": "dataset-id",
  "filename": "manual.pdf"
}
```

**响应格式同上**。

#### 3.1.3 上传频率限制

- **限制**：30 秒窗口内，按团队套餐配置限制上传次数
- **超限响应**：HTTP 429，`code: 429`，`message: "uploadFileIntervalLimit"`

## 四、MCP 服务器接入

### 4.1 概述

FastGPT MCP（Model Context Protocol）服务器允许外部 AI 工具或 Agent 通过标准化协议调用 FastGPT 的工具、API 等功能。

**项目位置**：`FastGPT/projects/mcp_server/`

### 4.2 启动 MCP 服务器

#### 4.2.1 本地启动（开发环境）

```bash
cd projects/mcp_server
pnpm install
pnpm dev
```

服务将在 `http://localhost:3000` 启动。

#### 4.2.2 生产部署

```bash
cd projects/mcp_server
pnpm install
bun run build
bun start
```

或使用 Docker：
```bash
docker run -p 3000:3000 fastgpt-mcp-server
```

### 4.3 协议详解

#### 4.3.1 连接建立

MCP 使用 SSE（Server-Sent Events）作为传输协议。

**客户端连接**：
```
GET /<api-key>/sse
```

服务器返回 SSE 流，建立双向通信。

#### 4.3.2 工具发现

**请求**（JSON-RPC）：
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/list"
}
```

**响应**：
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "tools": [
      {
        "name": "search_dataset",
        "description": "在知识库中搜索文档",
        "inputSchema": {
          "type": "object",
          "properties": {
            "datasetId": {"type": "string"},
            "query": {"type": "string"}
          },
          "required": ["datasetId", "query"]
        }
      }
    ]
  }
}
```

#### 4.3.3 工具调用

**请求**（JSON-RPC）：
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/call",
  "params": {
    "name": "search_dataset",
    "arguments": {
      "datasetId": "ds-xxx",
      "query": "产品介绍"
    }
  }
}
```

**响应**：
```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "搜索结果..."
      }
    ],
    "isError": false
  }
}
```

### 4.4 集成示例（Claude.ai）

在 Claude 的自定义指令中配置 MCP 服务器：

```json
{
  "type": "server",
  "url": "http://your-domain:3000",
  "auth": {
    "type": "apiKey",
    "apiKey": "fastgpt-xxxxx"
  }
}
```

Claude 将自动发现并调用 FastGPT 的所有工具。

## 五、常见接入场景示例

### 5.1 场景一：Web 应用集成（使用 OpenAI SDK）

**目标**：在现有 Web 应用中集成 FastGPT 对话功能

**Python 示例**：
```python
from openai import OpenAI

client = OpenAI(
    api_key="fastgpt-xxxxx",
    base_url="https://your-domain/api/v1"
)

# 创建对话
response = client.chat.completions.create(
    model="app-id-xxx",  # FastGPT 应用 ID
    messages=[
        {"role": "user", "content": "介绍一下你的产品"}
    ],
    stream=False
)

print(response.choices[0].message.content)

# 续接历史对话
response2 = client.chat.completions.create(
    model="app-id-xxx",
    messages=[
        {"role": "user", "content": "能给出价格吗？"}
    ],
    extra_body={
        "chatId": response.id,  # 使用前一轮的 chatId
        "variables": {"currency": "CNY"}
    }
)
```

**JavaScript 示例**：
```javascript

async function streamChat(messages, appId, chatId) {
  const response = await fetch('https://your-domain/api/v1/chat/completions', {

    headers: {
      'Authorization': 'Bearer fastgpt-xxxxx',
      'Content-Type': 'application/json'

      messages,

      chatId

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let fullContent = '';

      if (done) break;

      const text = decoder.decode(value);
      const lines = text.split('\n');

      for (const line of lines) {
        if (line.startsWith('data: ')) {
          const json = JSON.parse(line.slice(6));
          
          if (json.event === 'answer') {
            fullContent += json.text;
            // 实时更新 UI
            setResponseText(fullContent);
          } else if (json.event === '[DONE]') {
            newChatId = json.chatId;
          }
        }
      }
    }
  } finally {
    reader.releaseLock();
  }

  return { content: fullContent, chatId: newChatId };
}
```

## 七、速率限制与配额

### 7.1 限制类型

| 限制 | 默认策略 | 超限响应 |
|------|---------|---------|
| **聊天频率（团队）** | 按套餐 `chatRequestPerMinute` | HTTP 429, code 429 |
| **IP 转写频率** | 1 次/秒 | HTTP 429, code 429 |
| **文件上传频率** | 30 秒窗口按套餐限制 | HTTP 429, code 429 |
| **账户额度** | 按套餐余额 | HTTP 510, code 510 |

## 八、错误处理

### 8.1 常见错误码

| HTTP | Code | 说明 | 解决方案 |
|------|------|------|---------|
| 200 | 200 | 成功 | - |
| 400 | 400 | 参数验证失败（Zod Schema） | 检查请求参数格式 |
| 401 | 401 | 未认证 | 检查 Authorization 请求头 |
| 403 | 403 | 权限不足 | 确认 API Key 有应用访问权限 |
| 404 | 404 | 资源不存在 | 检查 appId、chatId 是否正确 |
| 422 | 422 | 业务参数错误 | 检查业务逻辑参数 |
| 429 | 429 | 频率限制 | 等待后重试，或升级套餐 |
| 500 | 500 | 服务端错误 | 查看服务日志 |
| 510 | 510 | 额度不足 | 充值或升级套餐 |
| 511 | 511 | 未授权模型 | 在管理后台授权模型 |
| 514 | 514 | API Key 无效 | 检查 API Key 是否正确/过期 |

### 8.2 错误响应格式

```json
{
```

## 九、安全建议

### 9.1 API Key 管理

- **不要在前端代码中硬编码** API Key，改用后端代理
- **定期轮换** API Key（建议每 90 天）
- **使用应用级 Key**（而非团队级 Key），限制权限范围
- **启用 IP 白名单**（在创建 Key 时配置）

### 9.2 数据隐私

- 敏感信息（如用户隐私数据）不应在 `metadata` 中传递
- 启用知识库**访问控制**，限制数据访问范围
- 定期审计 API Key 的使用日志
- 对接工作流中的敏感操作（如删除数据），需在应用层额外验证

## 十、常见问题

### Q1：可以使用 OpenAI SDK 吗？

**A**：可以。FastGPT v1 API 完全兼容 OpenAI ChatCompletion，只需配置 `base_url` 指向 FastGPT 服务器，使用 `fastgpt-xxxxx` 替代 OpenAI API Key。

### Q2：如何创建新的对话？

**A**：每次请求不传 `chatId` 或传递空值，系统自动创建新对话。

### Q3：如何续接历史对话？

**A**：在请求体中传递 `chatId`（来自前一轮响应的 `id` 字段），系统自动加载历史消息。

### Q4：流式响应如何解析？

**A**：响应为 SSE 格式，逐行解析。每行以 `data:` 前缀开头，JSON 格式。当收到 `data: [DONE]` 时流式结束。

### Q5：如何传递工作流变量？

**A**：在请求体的 `variables` 字段传递 JSON 对象，工作流可通过**全局变量**节点访问。

### Q6：分享链接和 API Key 有什么区别？

**A**：
- **API Key**：需认证，权限由 Key 配置决定，适合内部系统/后端
- **分享链接**：无需认证，权限由分享链接配置决定，适合外部嵌入/临时访问

### Q7：如何获取详细的工作流执行结果？

**A**：在请求体中设置 `detail: true`，响应将包含 `responseData`（各节点执行结果）和 `newVariables`（工作流更新的变量）。

### Q8：MCP 服务器有什么用途？

**A**：MCP 服务器允许外部 AI 工具（如 Claude）通过标准协议调用 FastGPT 的工具、API，实现 Agent 扩展和工具集成。

### Python - 完整示例

```python
import requests
import json
import time

class FastGPTClient:
    def __init__(self, api_key, base_url):
        self.api_key = api_key
        self.base_url = base_url
        self.headers = {
            'Authorization': f'Bearer {api_key}',
            'Content-Type': 'application/json'
        }

    def chat(self, app_id, messages, stream=False, variables=None, chat_id=None):
        """调用聊天接口"""
        payload = {
            'model': app_id,
            'messages': messages,
            'stream': stream,
            'variables': variables or {},
        }
        if chat_id:
            payload['chatId'] = chat_id

        response = requests.post(
            f'{self.base_url}/v1/chat/completions',
            headers=self.headers,
            json=payload,
            stream=stream
        )

        if stream:
            return self._parse_stream(response)
        else:
            return response.json()

    def _parse_stream(self, response):
        """解析 SSE 流式响应"""
        full_text = ''
        for line in response.iter_lines():
            if line:
                line = line.decode('utf-8')
                if line.startswith('data: '):
                    data = json.loads(line[6:])
                    if data.get('event') == 'answer':
                        full_text += data.get('text', '')
                    elif data.get('event') == '[DONE]':
                        break
        return full_text

    def get_histories(self, app_id, limit=20):
        """获取对话历史"""
        response = requests.post(
            f'{self.base_url}/core/chat/history/getHistories',
            headers=self.headers,
            json={'appId': app_id, 'pageSize': limit}
        )
        return response.json()

# 使用示例
client = FastGPTClient(
    api_key='fastgpt-xxxxx',
    base_url='https://your-domain/api'
)

# 创建新对话
response = client.chat(
    app_id='app-id-xxx',
    messages=[{'role': 'user', 'content': '你好'}]
)
print(response['choices'][0]['message']['content'])

# 续接对话
response2 = client.chat(
    app_id='app-id-xxx',
    messages=[{'role': 'user', 'content': '能给出建议吗？'}],
    chat_id=response['id']  # 使用前一轮的 chatId
)
print(response2['choices'][0]['message']['content'])
```

### Node.js - 完整示例

```javascript
class FastGPTClient {
  constructor(apiKey, baseUrl) {
    this.apiKey = apiKey;
    this.baseUrl = baseUrl;
  }

  async chat(appId, messages, options = {}) {
    const response = await fetch(`${this.baseUrl}/v1/chat/completions`, {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${this.apiKey}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        model: appId,
        messages,
        stream: options.stream || false,
        variables: options.variables || {},
        chatId: options.chatId || undefined
      })
    });

    if (options.stream) {
      return this.parseStream(response);
    } else {
      return response.json();
    }
  }

  async parseStream(response) {
    const reader = response.body.getReader();
    const decoder = new TextDecoder();
    let fullText = '';

    try {
      while (true) {
        const { done, value } = await reader.read();
        if (done) break;

        const chunk = decoder.decode(value);
        const lines = chunk.split('\n');

        for (const line of lines) {
          if (line.startsWith('data: ')) {
            const data = JSON.parse(line.slice(6));
            if (data.event === 'answer') {
              fullText += data.text || '';
            }
          }
        }
      }
    } finally {
      reader.releaseLock();
    }

    return fullText;
  }

  async getHistories(appId, limit = 20) {
    const response = await fetch(
      `${this.baseUrl}/core/chat/history/getHistories`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${this.apiKey}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({ appId, pageSize: limit })
      }
    );
    return response.json();
  }
}

// 使用示例
const client = new FastGPTClient(
  'fastgpt-xxxxx',
  'https://your-domain/api'
);

// 创建新对话

  { role: 'user', content: '你好' }
]);
console.log(response.choices[0].message.content);

// 流式对话
const streamText = await client.chat('app-id-xxx', [
  { role: 'user', content: '写一首诗' }
], { stream: true });
console.log(streamText);
```
