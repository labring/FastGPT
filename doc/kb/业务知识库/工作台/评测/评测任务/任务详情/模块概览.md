# 任务详情 — 模块概览

## 1. 模块业务概述

任务详情模块是评测系统的核心查看与操作界面，用于展示单个评测任务的完整执行结果。用户从评测任务列表点击某个任务后进入此页面，可以查看每条评测数据的评分情况、各维度的得分统计，以及对单条数据进行编辑、重试、删除等操作。

模块采用三栏布局：左侧为评测数据列表（支持搜索和分页），中间为单条数据的详情展示（问题、参考答案、实际回答、评测维度评分），右侧面板展示综合评分仪表盘和任务基本信息。评测执行过程中，页面每 15 秒自动轮询刷新最新数据，用户无需手动刷新。

**子能力关系**：本模块为「评测任务」下的叶子节点，无子能力。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 团队成员（有写入权限） | 全部页面 | 查看评测结果、编辑数据、删除数据、重试单项、重试失败项、导出数据、刷新评分、修改评分配置 | 可点击应用名称跳转至应用详情页 |
| 团队成员（无写入权限） | 全部页面 | 查看评测结果 | 应用名称不可点击跳转，操作按钮不可见 |

**权限模型**: 基于团队（Team）的访问控制。评测任务归属团队，团队成员均可查看；写入权限由 `appDetail.permission.hasWritePer` 控制，影响应用的跳转链接是否可点击。数据操作（编辑/删除/重试）通过 API 层的团队 ID 校验进行权限控制。
