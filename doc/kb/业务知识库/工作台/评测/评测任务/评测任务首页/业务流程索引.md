# 评测任务首页 — 业务流程索引

## 查看评测任务列表

- **业务描述**: 进入评测任务首页后，查看当前团队下所有评测任务的列表，包括任务名称、进度、关联应用、应用版本、评测结果、起止时间和执行者信息。
- **触发入口**: 导航至"工作台 → 评测 → 评测任务"（通过评测首页 FillRowTabs 切换到"评测任务"Tab）
- **路由路径**: `/dashboard/evaluation/task?evaluationTab=tasks`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/task/index.tsx`
- **涉及角色**: admin, team_member
- **前置条件**: 用户已登录，属于某个团队，拥有评测功能访问权限（`feConfigs.show_evaluation` 不为 false）

## 搜索评测任务

- **业务描述**: 通过搜索框输入关键词，在任务名称中模糊搜索匹配的评测任务，搜索结果经防抖处理后展示。
- **触发入口**: 评测任务首页顶部的搜索输入框
- **路由路径**: `/dashboard/evaluation/task?evaluationTab=tasks`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/task/index.tsx`
- **涉及角色**: admin, team_member
- **前置条件**: 已进入评测任务首页

## 按应用筛选任务

- **业务描述**: 通过应用下拉选择器筛选特定应用关联的评测任务，支持"全部应用"选项以清除筛选。
- **触发入口**: 评测任务首页顶部的应用选择器（AppSelectWithAll 组件）
- **路由路径**: `/dashboard/evaluation/task?evaluationTab=tasks`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/task/index.tsx`
- **涉及角色**: admin, team_member
- **前置条件**: 已进入评测任务首页

## 创建评测任务

- **业务描述**: 打开创建评测任务弹窗，填写任务名称、选择目标应用及版本、选择评测数据集、配置评测维度后提交，系统自动创建评测任务并启动评测。
- **触发入口**: 评测任务首页顶部的"创建新任务"按钮
- **路由路径**: 不涉及路由变更（弹窗模式）
- **页面组件位置**: `projects/app/src/pageComponents/dashboard/evaluation/task/CreateModal.tsx`
- **涉及角色**: admin, team_member
- **前置条件**: 已进入评测任务首页，系统中存在可评测的应用和数据集

## 重命名评测任务

- **业务描述**: 在任务行操作菜单中触发重命名，弹出编辑框修改任务名称后确认保存。
- **触发入口**: 任务列表行右侧"更多"菜单 → "重命名"
- **路由路径**: `/dashboard/evaluation/task?evaluationTab=tasks`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/task/index.tsx`
- **涉及角色**: admin, team_member
- **前置条件**: 任务列表中至少存在一条任务

## 删除评测任务

- **业务描述**: 在任务行操作菜单中触发删除，弹出确认对话框，确认后删除该评测任务及其关联的所有评测项和队列任务。
- **触发入口**: 任务列表行右侧"更多"菜单 → "删除"
- **路由路径**: `/dashboard/evaluation/task?evaluationTab=tasks`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/task/index.tsx`
- **涉及角色**: admin, team_member
- **前置条件**: 任务列表中至少存在一条任务

## 重试失败评测项

- **业务描述**: 当评测任务中存在失败项时，可通过操作菜单触发重试，系统重新执行所有失败评测项的评测逻辑。
- **触发入口**: 任务列表行右侧"更多"菜单 → "重试失败项"（仅当 `statistics.error > 0` 时显示）
- **路由路径**: `/dashboard/evaluation/task?evaluationTab=tasks`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/task/index.tsx`
- **涉及角色**: admin, team_member
- **前置条件**: 任务存在失败（error）状态的评测项

## 跳转任务详情

- **业务描述**: 点击任务列表中的任意一行，跳转至该任务的详情页，查看评测项的详细执行结果和总结报告。
- **触发入口**: 点击任务列表中的某一行（行级点击事件）
- **路由路径**: `/dashboard/evaluation/task/detail?taskId={task._id}`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/task/detail/index.tsx`
- **涉及角色**: admin, team_member
- **前置条件**: 任务列表中至少存在一条任务

## 场景关系

- **查看评测任务列表** ← 页面进入时的默认场景，各操作场景均以此为入口上下文
- **搜索评测任务** → 影响列表数据刷新
- **按应用筛选任务** → 影响列表数据刷新
- **创建评测任务** → 完成后自动刷新列表
- **重命名评测任务** → 完成后自动刷新列表
- **删除评测任务** → 完成后自动刷新列表
- **重试失败评测项** → 完成后自动刷新列表
- **跳转任务详情** → 离开当前页面，进入详情子页面
