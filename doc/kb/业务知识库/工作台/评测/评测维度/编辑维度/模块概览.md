# 编辑维度 — 模块概览

## 1. 模块业务概述

编辑维度是评测维度模块的子能力，面向已拥有评测权限的用户，用于**修改已有自定义评测维度**的定义信息。用户从评测维度首页点击自定义维度条目进入该页面，可修改维度名称、描述和评分提示词，提交后更新存储，并自动跳转回维度列表。

本模块与创建维度共享表单组件（EditForm 和 TestRun），核心流程为：根据 URL 参数 `id` 加载维度详情 -> 在 EditForm 中回显并允许编辑 -> 可试运行验证评分效果 -> 提交更新 -> 跳回维度列表。内置维度不可进入编辑页，仅自定义维度支持编辑操作。

本模块为叶子节点，无子能力。

主要业务规则：
- 通过路由 query 参数 `id` 获取维度 ID 进行数据加载
- 加载维度数据失败或数据不存在时，展示错误提示并自动跳回维度列表
- 表单验证未通过时，"保存"和"试运行"按钮均处于禁用状态
- 保存成功后展示成功提示并自动跳转至 `/dashboard/evaluation?evaluationTab=dimensions`
- 页面加载中或维度数据为空时，均包裹在 DashboardContainer 中展示状态提示

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 有评测创建权限的用户（hasEvaluationCreatePer） | 编辑维度页面 | 编辑维度名称/描述/提示词、试运行、保存更新 | 可编辑自己或其他团队成员创建的自定义维度，无维度级别的所有者限制 |
| 无评测创建权限的用户 | 不可见 | 无 | 侧边栏「应用测评」菜单不可见，无法进入编辑页 |

**权限模型**: 入口受 DashboardContainer 侧边栏双重控制——`feConfigs.show_evaluation`（系统功能开关）和 `userInfo.team.permission.hasEvaluationCreatePer`（团队级评测权限）。页面本身无额外独立权限校验。内置维度无法通过路由进入编辑（维度列表中对内置维度不提供编辑入口，但页面本身未对维度类型做后端校验）。
