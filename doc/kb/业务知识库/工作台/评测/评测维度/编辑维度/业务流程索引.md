# 编辑维度 — 业务流程索引

## 编辑评测维度

- **业务描述**: 用户从评测维度列表中进入编辑页面，系统加载已有维度数据（名称、描述、提示词），用户修改后点击保存，系统更新维度记录并跳转回评测首页的维度列表。
- **触发入口**: 评测首页 → 评测维度 Tab → 点击自定义维度行的编辑按钮
- **路由路径**: `/dashboard/evaluation/dimension/edit?id={dimensionId}`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/dimension/edit.tsx`
- **涉及角色**: hasEvaluationCreatePer（有评测创建权限的用户）
- **前置条件**: 维度 ID 有效且数据加载成功；用户具有评测权限；待编辑维度为自定义维度（内置维度不可编辑）

## 试运行评测维度

- **业务描述**: 在编辑表单中填写或修改完毕后，用户点击「试运行」按钮打开测试弹窗，输入测试问题、参考答案和实际回答，选择评测模型后运行，查看维度在当前配置下的评分结果和反馈，验证修改后的评测效果。
- **触发入口**: 编辑维度页面 → 页面底部「试运行」按钮
- **路由路径**: `/dashboard/evaluation/dimension/edit?id={dimensionId}`（同页面弹窗，不跳转路由）
- **页面组件位置**: `projects/app/src/pageComponents/dashboard/evaluation/dimension/TestRun.tsx`
- **涉及角色**: hasEvaluationCreatePer（有评测创建权限的用户）
- **前置条件**: 表单必填字段（名称、提示词）已填写有效，表单验证通过

## 场景关系

- **编辑评测维度** ← **试运行评测维度**：先编辑后试运行。试运行是编辑过程中的可选验证步骤，必须在进入编辑页面并完成表单填写后才能触发，用于确认修改后的维度评分效果符合预期。编辑与试运行在同一页面内完成，无需跳转。
