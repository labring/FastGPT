# 评测首页 — 模块概览

## 1. 模块业务概述

评测首页是 FastGPT 评测系统的统一入口页面，路由为 `/dashboard/evaluation`。页面以三个 Tab 面板聚合展示评测相关的三大核心能力：评测任务管理、评测数据集管理和评测维度配置。用户在此页面可以快速切换查看不同评测资源的列表视图，无需跳转即可概览评测系统的全貌。

本模块是"工作台 > 评测"下的子能力，采用单页面多 Tab 架构设计。页面通过 Next.js 路由的 `evaluationTab` query 参数驱动 Tab 切换，使用 `router.replace` 实现无刷新 Tab 切换并保持浏览器历史记录。模块受双重权限控制：系统级功能开关 `feConfigs.show_evaluation` 控制页面整体可见性（false 时自动重定向至 `/dashboard`），团队级权限 `hasEvaluationCreatePer` 控制侧边栏菜单入口的展示。

`has_tab_children: true`，本模块已被拆分为 3 个 Tab 子能力：评测任务、评测数据集、评测维度。本文档及 09/10 仅覆盖入口公共功能（页面布局、Tab 导航、权限守卫），各 Tab 的深度内容见对应子文档。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 所有登录用户 | 评测首页（feConfigs.show_evaluation 为 true 时） | 查看评测任务列表、数据集列表、维度列表 | 页面级入口可见性由系统开关控制；各 Tab 内具体操作权限（创建、删除等）由团队权限位控制，详见各子能力文档 |

**权限模型**：页面入口采用系统配置守卫模式。组件挂载时通过 `useEffect` 检查 `feConfigs.show_evaluation`，若为 `false` 则自动 `router.replace('/dashboard')` 重定向。侧边栏菜单入口由 `DashboardContainer` 根据 `feConfigs.show_evaluation` 和用户权限位联合控制渲染。页面内部各 Tab 的操作权限由各自子组件独立校验，不在此入口层处理。

## 3. 跨模块依赖摘要

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|

| FillRowTabs (@fastgpt/web) | Tab 切换标签栏组件，提供类型安全的 Tab 列表渲染和切换回调 | 组件 |
| useSystemStore (@/web/common/system) | 获取 feConfigs 系统配置，用于 show_evaluation 权限守卫 | Store |

| i18n 国际化 (next-i18next) | 服务端翻译加载（getServerSideProps），覆盖 8 个 namespace：dashboard_evaluation、evaluation、dataset、app、common、account、chat、file | 工具 |

## 4. 对外共享摘要

本模块为纯入口聚合页，不对外提供可复用的共享能力。组件未通过 `export` 暴露任何 Context 或 Hook，仅被父级路由框架引用。
