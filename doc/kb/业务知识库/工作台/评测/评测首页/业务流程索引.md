# 评测首页 — 业务流程索引

> 本模块为评测中心的入口页面，通过 3 个 Tab 切换不同功能模块。各 Tab 的深度业务场景见对应子能力的 [业务流程索引]。

## 业务场景列表

### 进入评测中心

- **业务描述**: 用户从工作台侧边栏点击"应用测评"进入评测中心。页面加载后首先校验评测功能可见性（`feConfigs.show_evaluation`），不可见时自动跳回工作台。默认展示"评测任务"Tab，顶部通过 FillRowTabs 组件提供 3 个 Tab 供用户切换。Tab 切换通过 URL query 参数 `evaluationTab` 驱动（可选值：`tasks`、`datasets`、`dimensions`），支持直接通过 URL 进入指定 Tab。
- **触发入口**: 工作台侧边栏 → 应用测评
- **路由路径**: `/dashboard/evaluation`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/index.tsx`
- **涉及角色**: 团队成员、团队管理员
- **前置条件**: 用户已登录并有团队权限；评测功能可见（`feConfigs.show_evaluation !== false`）

### 切换到评测任务 Tab

- **业务描述**: 用户点击"评测任务"Tab，页面向下传递 FillRowTabs 组件作为顶部导航（通过 `Tab` prop），渲染评测任务列表组件。在该 Tab 下可查看任务列表、搜索筛选任务、创建评测任务、查看任务详情等。
- **触发入口**: 评测首页 → 点击"评测任务" Tab
- **路由路径**: `/dashboard/evaluation?evaluationTab=tasks`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/task/index.tsx`
- **涉及角色**: 团队成员、团队管理员
- **前置条件**: 已进入评测首页

### 切换到评测数据集 Tab

- **业务描述**: 用户点击"评测数据集"Tab，页面向下传递 FillRowTabs 组件作为顶部导航，渲染评测数据集列表组件。在该 Tab 下可查看数据集列表、搜索数据集、智能生成或文件导入创建数据集、查看数据集详情等。
- **触发入口**: 评测首页 → 点击"评测数据集" Tab
- **路由路径**: `/dashboard/evaluation?evaluationTab=datasets`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/dataset/index.tsx`
- **涉及角色**: 团队成员、团队管理员
- **前置条件**: 已进入评测首页

### 切换到评测维度 Tab

- **业务描述**: 用户点击"评测维度"Tab，页面向下传递 FillRowTabs 组件作为顶部导航，渲染评测维度列表组件。在该 Tab 下可查看内置维度与自定义维度列表、创建自定义维度、编辑维度配置等。
- **触发入口**: 评测首页 → 点击"评测维度" Tab
- **路由路径**: `/dashboard/evaluation?evaluationTab=dimensions`
- **页面组件位置**: `projects/app/src/pages/dashboard/evaluation/dimension/index.tsx`
- **涉及角色**: 团队成员、团队管理员
- **前置条件**: 已进入评测首页

## 场景关系说明

- **进入评测中心**（S01）是所有业务场景的入口，页面加载时先完成权限校验再渲染 Tab 导航
- **S01-1（评测任务）**、**S01-2（评测数据集）**、**S01-3（评测维度）** 三个 Tab 场景互相独立、彼此无依赖，用户可自由切换
- Tab 切换通过 URL query 参数 `evaluationTab` 驱动，URL 变更后触发条件渲染，同一时刻仅展示一个 Tab 对应的子组件
- 每个 Tab 子组件共享同一个 FillRowTabs 导航（通过 `Tab` prop 传递），确保导航栏在各 Tab 下保持一致
- 典型评测工作流为：准备数据集（S01-2）→ 定义评测维度（S01-3）→ 创建评测任务（S01-1 内触发），但 Tab 间无强制先后顺序
