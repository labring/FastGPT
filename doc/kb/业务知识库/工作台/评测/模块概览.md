# 评测 — 模块概览

## 1. 模块业务概述

评测模块是 FastGPT 工作台中用于**评估 AI 应用回答质量**的功能区。用户可将包含测试问题和预期答案的数据集导入系统，选择待评测的 AI 应用和评测模型，启动自动化评测任务。系统会自动将数据集中的问题逐一发送至目标应用，由评测模型对回答质量进行评分，最终产出多维度评测报告。

本模块为 Beta 功能，面向拥有评测权限的团队成员开放（通过 `feConfigs.show_evaluation` 配置开关和 `hasEvaluationCreatePer` 权限控制）。评测能力是 AI 应用上线前的关键质量保障环节，帮助开发者量化应用表现、对比不同版本差异、定位回答质量短板。

评测模块下含五个子能力，业务关系如下：
- **评测首页**：作为评测入口，以三个 Tab 聚合展示评测任务、评测数据集和评测维度
- **创建评测**：提供评测任务创建表单，选择应用、模型、上传测试文件
- **评测数据集**：管理测试数据集，支持智能生成和文件导入两种创建方式
- **评测维度**：管理评测评分维度，内置常用维度模板，支持自定义维度
- **评测任务**：管理评测任务生命周期，查看任务执行进度和评分结果

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 团队管理员 | 全部评测页面 | 创建/删除评测任务、管理数据集和维度、查看评测结果 | 拥有 `hasEvaluationCreatePer` 权限，可执行所有操作 |
| 团队成员（有评测权限） | 全部评测页面 | 同管理员 | 权限模型基于 `hasEvaluationCreatePer` 和 `hasManagePer` |
| 团队成员（无评测权限） | 不可见评测入口 | 无 | 侧边栏不展示「应用测评」菜单项 |

**权限模型**: 侧边栏菜单入口受双重控制：`feConfigs.show_evaluation`（系统级功能开关）和 `userInfo.team.permission.hasEvaluationCreatePer`（团队级权限）。评测首页的访问守卫在组件挂载时通过 `useEffect` 检查 `feConfigs.show_evaluation`，若为 `false` 则重定向至 `/dashboard`。各子能力页面继承 DashboardContainer 的统一布局框架。

## 3. 跨模块依赖摘要

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| 应用 | 选择目标应用（AppSelect 组件）；获取应用详情、变量配置 | 组件/API |
| 对话 | 聊天模型（AIModelSelector）配置评测模型 | 组件 |
| 文件 | 文件上传组件（FileSelector）；CSV 文件解析和校验 | 组件/工具 |

> 评测模块对数据集模块的交互发生于评测数据集子能力（智能生成依赖知识库数据集），详见各子能力文档。

## 4. 对外共享

| 共享能力 | 消费模块 | 能力类型 |
|---------|---------|---------|
| 评测结果数据 | 应用详情（通过 API 查询版本关联的评测报告） | API |
| 评测维度模板 | 创建评测任务（选择评测维度时引用内置和自定义维度） | 数据 |
