# 评测 — 业务流程详解

> 评测模块为分组节点，详细业务流程由各子能力独立承载。本文档提供评测流程的全局视图。

## 评测总流程

评测模块的完整业务流程分为准备和执行两大阶段：

### 准备阶段

```mermaid
sequenceDiagram
    participant 用户
    participant 评测首页
    participant 评测数据集
    participant 评测维度
    participant 后端API

    用户->>评测首页: 进入应用测评入口
    评测首页->>后端API: 检查 feConfigs.show_evaluation
    alt 功能未开启
        评测首页->>用户: 重定向至 /dashboard
    end
    用户->>评测数据集: 创建测试数据集
    评测数据集->>后端API: POST /core/evaluation/dataset/create
    后端API-->>评测数据集: 返回数据集 ID
    用户->>评测维度: 定义评分维度
    评测维度->>后端API: POST /core/evaluation/dimension/create
    后端API-->>评测维度: 返回维度 ID
```

### 执行阶段

```mermaid
sequenceDiagram
    participant 用户
    participant 创建评测
    participant 评测首页
    participant 评测任务
    participant 后端API

    用户->>创建评测: 填写评测配置（应用、模型、数据集、维度）
    创建评测->>后端API: POST /core/evaluation/task/create
    后端API-->>创建评测: 创建成功，返回任务 ID
    创建评测->>用户: 跳转至评测首页
    评测首页->>后端API: 轮询任务状态
    loop 评测进行中
        评测首页->>后端API: GET /core/evaluation/task/status
        后端API-->>评测首页: 返回进度信息
    end
    用户->>评测任务: 查看评测详情
    评测任务->>后端API: GET /core/evaluation/task/detail
    后端API-->>评测任务: 返回评分结果和原因
```

## API 调用汇总

| API 路径 | 方法 | 用途 | 页面 |
|---------|------|------|------|
| `/core/evaluation/task/list` | GET | 获取评测任务列表 | 评测首页（评测任务 Tab） |
| `/core/evaluation/dataset/list` | GET | 获取评测数据集列表 | 评测首页（评测数据集 Tab） |
| `/core/evaluation/dimension/list` | GET | 获取评测维度列表 | 评测首页（评测维度 Tab） |
| `/core/evaluation/task/create` | POST | 创建评测任务 | 创建评测 |
| `/core/evaluation/dataset/create` | POST | 创建评测数据集 | 评测数据集 |
| `/core/evaluation/dimension/create` | POST | 创建评测维度 | 评测维度 |
| `/core/evaluation/task/detail` | GET | 获取评测任务详情 | 评测任务 |
