# MCP 服务器 — 模块概览

## 模块业务概述

MCP 服务器模块是 FastGPT 工作台中的一项管理功能，允许团队将自己的应用发布为符合 Model Context Protocol (MCP) 标准的服务器。用户通过此模块可以将一个或多个应用打包为一个 MCP Server，生成统一的访问端点（支持 Streamable HTTP 和 SSE 两种协议），供外部的 MCP 兼容客户端（如 Claude Desktop、Cursor 等 AI 工具）直接调用。

该模块是 FastGPT「应用构建 → 应用发布 → 外部消费」链路中的关键一环，它使得在 FastGPT 中搭建的 AI 应用能够无缝融入外部 AI 工作流生态，无需额外的适配层。

本模块为叶子节点，无子能力。

## 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 团队管理员 | MCP 服务器列表 | 创建、编辑、删除、查看使用方式 | 全部操作权限 |
| 团队成员（有 App 创建权限） | MCP 服务器列表 | 创建、编辑、删除、查看使用方式 | 需具有 `hasAppCreatePer` 权限才可创建 MCP 服务器；无此权限时「创建」按钮置灰 |
| 无 App 创建权限的成员 | MCP 服务器列表 | 仅查看列表、查看使用方式 | 创建、编辑、删除按钮不可用 |

**权限模型**: 基于用户 permission 对象的 `hasAppCreatePer` 属性控制创建权限，编辑和删除权限通过页面交互中的操作按钮隐式控制。

## 跨模块依赖

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| 应用管理 | 获取应用列表、获取应用基本信息、获取应用文件夹路径 | API |
| 用户/团队 | 用户信息（权限）、团队信息 | Hook/Store |
| 系统配置 | 系统配置项（自定义域名、SSE 代理端点等） | Store |

## 对外共享

本模块组件均为页面内部使用，不对外提供共享组件。MCP 服务器的运行时 API 端点（`/api/mcp/app/[key]/mcp` 和 SSE）对外部 MCP 客户端提供标准 MCP 协议服务。
