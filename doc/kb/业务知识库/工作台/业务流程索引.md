# 工作台 — 业务流程索引

## 业务场景列表

### 进入应用工作台

- **业务描述**：用户通过侧边栏"应用构建"入口进入应用工作台，查看和管理自己的 AI 应用列表。
- **触发入口**：侧边栏 → 应用构建 → 应用（App）
- **路由路径**：`/dashboard/agent`
- **页面组件位置**：`pages/dashboard/agent/index.tsx`
- **涉及角色**：全部用户
- **前置条件**：已登录

### 创建新应用

- **业务描述**：用户在工作台中创建新的 AI 应用，可选择智能问答、Chat Agent 或工作流三种类型。
- **触发入口**：应用工作台 → "创建新应用"按钮
- **路由路径**：`/dashboard/agent`（弹窗触发）
- **页面组件位置**：`pages/dashboard/create/CreateModal.tsx`
- **涉及角色**：hasAppCreatePer 权限用户
- **前置条件**：已登录，具备应用创建权限

### 导入应用配置

- **业务描述**：用户通过 JSON 配置文件导入应用，快速恢复或迁移应用配置。
- **触发入口**：应用工作台 → 创建按钮下拉菜单 → "导入 JSON 配置"
- **路由路径**：`/dashboard/agent`
- **页面组件位置**：`pageComponents/dashboard/agent/JsonImportModal.tsx`
- **涉及角色**：hasAppCreatePer 权限用户
- **前置条件**：已登录，具有 JSON 配置文件

### 管理应用文件夹

- **业务描述**：用户创建文件夹对应用进行分组管理，支持文件夹的编辑、移动和删除。
- **触发入口**：应用工作台 → "新建文件夹"按钮 → 选择文件夹 → 侧边卡片编辑/移动/删除
- **路由路径**：`/dashboard/agent`
- **页面组件位置**：`components/common/folder/SlideCard.tsx`
- **涉及角色**：hasAppCreatePer（创建文件夹）/ 文件夹权限（编辑/删除）
- **前置条件**：已登录，对应操作权限

### 搜索与筛选应用

- **业务描述**：用户通过搜索关键词或按应用类型（全部/智能问答/Chat Agent/工作流）筛选应用列表。
- **触发入口**：应用工作台 → 搜索框输入 / Tab 切换
- **路由路径**：`/dashboard/agent`
- **页面组件位置**：`pages/dashboard/agent/index.tsx`
- **涉及角色**：全部用户
- **前置条件**：已登录

### 浏览模板市场

- **业务描述**：用户浏览预置的应用模板，选择合适模板快速创建应用。
- **触发入口**：侧边栏 → "模板市场" / 创建按钮下拉菜单 → "从模板创建"
- **路由路径**：`/dashboard/templateMarket`
- **页面组件位置**：`pages/dashboard/templateMarket/index.tsx`
- **涉及角色**：全部用户
- **前置条件**：已登录

### 管理技能

- **业务描述**：用户管理 Agent 技能（Skills），为应用提供可复用的能力单元。
- **触发入口**：侧边栏 → 应用构建 → 技能（Skill）
- **路由路径**：`/dashboard/skill`
- **页面组件位置**：`pages/dashboard/skill/index.tsx`
- **涉及角色**：全部用户
- **前置条件**：已登录

### 管理工具

- **业务描述**：用户管理系统工具（Tools），支持 HTTP 工具、插件等类型的工具配置。
- **触发入口**：侧边栏 → 应用构建 → 工具（Tools）
- **路由路径**：`/dashboard/tool`
- **页面组件位置**：`pages/dashboard/tool/index.tsx`
- **涉及角色**：全部用户
- **前置条件**：已登录

### 管理 MCP 服务器

- **业务描述**：用户管理 MCP（Model Context Protocol）服务器连接配置。
- **触发入口**：侧边栏 → 应用构建 → MCP 服务器
- **路由路径**：`/dashboard/mcpServer`
- **页面组件位置**：`pages/dashboard/mcpServer/index.tsx`
- **涉及角色**：全部用户
- **前置条件**：已登录

### 执行应用评测

- **业务描述**：用户对 AI 应用进行系统性评测，包括准备数据集、配置评测维度、创建和执行评测任务。
- **触发入口**：侧边栏 → "应用评测"（Beta，需 show_evaluation 配置且 hasEvaluationCreatePer 权限）
- **路由路径**：`/dashboard/evaluation`
- **页面组件位置**：`pages/dashboard/evaluation/index.tsx`
- **涉及角色**：hasEvaluationCreatePer 权限用户
- **前置条件**：已登录，系统启用评测功能，具备评测创建权限

### 配置系统工具（root 专用）

- **业务描述**：root 用户管理平台级系统工具配置。
- **触发入口**：侧边栏 → 应用构建 → 工具 → 系统工具子页面
- **路由路径**：`/dashboard/systemTool`
- **页面组件位置**：`pages/dashboard/systemTool/index.tsx`
- **涉及角色**：仅 root 用户
- **前置条件**：已登录，用户名为 root

## 场景关系

- **应用工作台** → **创建应用**：工作台是应用管理的起点，创建应用是核心操作之一
- **应用工作台** → **模板市场**：可从模板市场选择模板快速创建应用
- **技能管理** / **工具管理** / **MCP 服务器**：应用构建的支撑能力，配置后可在应用中引用
- **评测** → **应用工作台**：评测可对工作台中的已有应用执行效果评估
- 大部分场景独立执行，无严格先后关系；创建应用、配置技能/工具后可进行评测
