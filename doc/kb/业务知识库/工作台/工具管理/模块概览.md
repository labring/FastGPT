# 工具管理 — 模块概览

## 模块业务概述

工具管理是工作台下的核心模块之一，为用户提供 AI 工具的集中管理能力。用户可在此创建、查看和管理三类工具：**工作流工具**（基于可视化工作流编排的自定义工具）、**HTTP 工具集**（通过 HTTP 请求调用外部 API 的工具集）和 **MCP 工具集**（基于 Model Context Protocol 的标准化工具集成）。

本模块是工具生命周期管理的入口，支持文件夹层级组织、搜索筛选、权限协作等能力。创建的工具有效后可被 Agent 应用在对话中调用，支撑智能体的能力扩展。

本模块为工作台下的一级能力节点，无子能力。

## 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 团队所有者 | 全部 | 创建/查看/编辑/删除/移动工具和文件夹，管理协作者权限，恢复权限继承 | 拥有完整管理权限，可删除文件夹 |
| 管理员 | 全部 | 创建/查看/编辑/移动工具和文件夹，管理协作者权限，恢复权限继承 | 与所有者类似，部分删除操作受限制 |
| 编辑者 | 全部 | 查看工具列表，在有写入权限的文件夹内创建/编辑工具和文件夹 | 仅在有写入权限的文件夹内可创建和编辑 |
| 只读成员 | 全部（查看） | 查看工具列表和文件夹详情 | 无创建/编辑/删除权限，仅可浏览 |

**权限模型**：基于位掩码的 RBAC 权限（读=0b100, 写=0b010, 管理=0b001），每个文件夹和应用独立控制权限。支持权限继承和协作者管理。

## 跨模块依赖

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| 应用工作台 | 复用 AppListContextProvider、List 组件、文件夹导航系统 | 组件/Context |
| 系统工具 | 通过 MyTabs 导航到系统工具列表页 | 页面导航 |
| 模板市场 | 创建工作流工具时可跳转选择模板 | 页面导航 |
| 应用详情 | 创建工具成功后跳转至应用详情页进行工作流编排 | 页面导航 |

## 对外共享

本模块主要作为消费者，复用应用工作台的基础设施（上下文管理、列表组件、文件夹系统），自身不对外提供独有的共享能力。

| 共享能力 | 消费模块 | 能力类型 |
|---------|---------|---------|
| 工具创建入口 | 应用详情（Agent 配置时可选择工具） | 数据引用 |
