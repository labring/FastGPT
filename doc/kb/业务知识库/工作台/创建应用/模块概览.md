# 创建应用 — 模块概览

## 1. 模块业务概述

创建应用模块是 FastGPT 工作台中的应用/工具创建入口，为用户提供统一的创建向导页面。用户可在此选择应用/工具类型、填写基本信息、使用模板快速创建或智能生成工作流。

支持的创建类型分为两大类：

- **Agent 类应用**：包括 Agent 对话助手（`chatAgent`）、简单对话机器人（`simple`）、工作流编排（`workflow`），适合构建 AI 对话、自动化工作流等场景。
- **工具类应用**：包括工作流工具（`workflowTool`）、MCP 工具集（`mcpToolSet`）、HTTP 工具集（`httpToolSet`），适合将工作流封装为可复用的工具或接入外部服务。

页面左侧为创建表单区（类型选择 → 图标名称 → 条件表单），右侧为选中类型的预览插图。支持的创建路径包括：从空白模板创建、从快速模板创建、智能生成工作流。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 团队管理员 | 全部创建页面 | 创建 Agent/工具应用，使用模板创建，智能生成工作流 | 完整创建权限 |
| 团队成员 | 全部创建页面 | 创建 Agent/工具应用，使用模板创建，智能生成工作流 | 与管理员权限相同 |

**权限模型**：创建应用页面无角色级权限校验，所有团队用户均可创建应用。创建后的应用权限由应用自身的成员管理控制。

## 3. 跨模块依赖

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| 工作台/应用工作台 | 从工作台页面跳转至创建页面（传递 `parentId`、`appType` 参数）；创建成功后跳转至应用详情 | 路由 / 状态 |
| 应用/应用详情 | 创建成功后路由跳转 `router.push('/app/detail?appId=${id}')` | 路由 |
| 工作台/模板市场 | 获取快速模板列表 API（`getTemplateMarketItemList`），获取模板详情 API（`getTemplateMarketItemDetail`），底部链接跳转至模板市场 | API / 路由 |
| 系统配置 | 获取模型列表（`useSystemStore.llmModelList`）、默认模型（`defaultModels`）用于智能生成 | Store |
| 文件上传 | 头像上传（`useUploadAvatar` + `getUploadAvatarPresignedUrl`） | Hook / API |
| MCP 工具 | 获取 MCP 工具列表 API（`getMCPTools`），创建 MCP 工具集 API（`postCreateMCPTools`） | API |
| HTTP 工具 | 创建 HTTP 工具集 API（`postCreateHttpTools`） | API |
| 鉴权配置 | Header 鉴权表单组件（`HeaderAuthForm`），鉴权数据转换函数（`headerValue2StoreHeader`） | 组件 / 工具函数 |

## 4. 对外共享

本模块为创建入口页面，不提供对外共享能力。

## 5. 关键页面与入口

| 页面名称 | 路由路径 | 页面组件位置 | 角色 | 备注 |
|---------|---------|-------------|------|------|
| 创建应用 | /dashboard/create | `projects/app/src/pages/dashboard/create/index.tsx` | 全体团队用户 | 接收 query 参数 `parentId`（父级目录/工具目录 ID）和 `appType`（预选类型） |

**入口来源**：
- 应用工作台（`/dashboard/agent`）→ 点击「创建」按钮
- 工具管理（`/dashboard/tool`）→ 点击「创建工具」按钮
- 两类入口通过 `parentId` 和 `appType` 参数区分，`appType` 为工具类型时进入工具创建模式

## 6. 子能力索引

本模块无子能力。
