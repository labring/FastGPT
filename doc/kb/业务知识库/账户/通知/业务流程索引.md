# 通知 — 业务流程索引

## 业务场景列表

### 查看通知列表

- **业务描述**: 用户进入通知中心，浏览系统推送的所有通知消息，包括官方通知和团队通知，按时间倒序排列
- **触发入口**: 账户中心侧边栏点击"通知"导航项（PC）或移动端顶部 Tab 栏切换到"通知"（移动端）
- **路由路径**: `/account/inform`
- **页面组件位置**: `projects/app/src/pages/account/inform.tsx`
- **涉及角色**: 所有登录用户（Plus版）
- **前置条件**: 用户已登录且订阅 Plus 版，`feConfigs.isPlus` 为 `true`

### 查看通知详情

- **业务描述**: 用户点击通知列表中的某条通知，在弹出的详情弹窗中查看通知的完整内容（Markdown 格式渲染）
- **触发入口**: 通知列表中点击任意通知卡片
- **路由路径**: `/account/inform`（弹窗覆盖，路由不变）
- **页面组件位置**: `projects/app/src/pageComponents/account/NotificationDetailsModal.tsx`
- **涉及角色**: 所有登录用户（Plus版）
- **前置条件**: 通知列表已加载且有通知数据

### 标记通知已读

- **业务描述**: 用户点击未读通知时，系统自动将该通知标记为已读
- **触发入口**: 通知列表中点击红点标记的未读通知卡片
- **路由路径**: `/account/inform`（操作不改变路由）
- **页面组件位置**: `projects/app/src/pages/account/inform.tsx`
- **涉及角色**: 所有登录用户（Plus版）
- **前置条件**: 存在未读通知（红点可见）

### 关闭重要通知弹窗

- **业务描述**: 重要级别通知会以全局浮层形式在页面顶部展示，用户点击关闭按钮将其标记为已读并移除
- **触发入口**: 任意页面的重要通知浮层上点击关闭图标
- **路由路径**: 任意页面（全局浮层，不绑定路由）
- **页面组件位置**: `projects/app/src/components/support/user/inform/ImportantInform.tsx`
- **涉及角色**: 所有登录用户（Plus版）
- **前置条件**: 存在级别为 `important` 或 `emergency` 的未读通知

## 场景关系

- **查看通知列表** → **查看通知详情**：列表点击触发详情弹窗
- **查看通知列表** → **标记通知已读**：点击未读通知时，标记已读与打开详情同时触发
- **浏览任意页面** → **关闭重要通知弹窗**：全局浮层不依赖通知页面，可在任何页面触发
