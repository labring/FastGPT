# 账单 — 业务流程索引

## 业务场景列表

### 账单与发票页面入口

- **业务描述**：团队管理员从账户侧边栏进入账单与发票管理页面，默认展示账单记录 Tab。页面顶部提供三个 Tab 切换（账单记录/发票/发票抬头），用户可在三个 Tab 间自由切换。
- **触发入口**：账户侧边栏 →「账单与发票」菜单项（仅在 `feConfigs.show_pay` 开启且用户具有管理权限时可见）
- **路由路径**：`/account/bill`
- **页面组件位置**：`projects/app/src/pages/account/bill/index.tsx`
- **涉及角色**：团队管理员（hasManagePer）
- **前置条件**：用户已登录、具有团队管理权限（`hasManagePer`）、系统开启支付功能（`show_pay`）

### 申请发票

- **业务描述**：在账单记录或发票 Tab 中，点击「开发票」按钮打开发票申请弹窗。用户从可开票账单列表中选择账单，确认金额后进入第二步填写发票抬头信息，提交后生成发票申请。
- **触发入口**：页面顶部「开发票」按钮（账单记录和发票 Tab 下可见，发票抬头 Tab 下隐藏）
- **路由路径**：`/account/bill`（弹窗形式，不切换路由）
- **页面组件位置**：`projects/app/src/pageComponents/account/bill/ApplyInvoiceModal.tsx`
- **涉及角色**：团队管理员
- **前置条件**：存在状态为已支付的账单（可开票账单）

## 场景关系

- **S01（页面入口）** → **S01-1/S01-2/S01-3**：页面入口后根据 Tab 选择进入不同子场景
- **S01-1 或 S01-2** → **S02**：在账单记录或发票 Tab 下可触发申请发票
- 各 Tab 子场景独立执行，无严格先后关系
