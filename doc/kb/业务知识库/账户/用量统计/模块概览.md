# 用量统计 — 模块概览

## 1. 模块业务概述

用量统计是账户中心下用于查看团队 AI 资源消耗情况的模块。它为团队成员提供用量明细查询和趋势看板，帮助用户了解积分消耗、Token 使用量等关键指标。管理员可以查看全团队所有成员的用量数据，并通过成员、群组、部门等多种维度进行筛选；普通成员则仅能看到自己的用量记录。

模块包含两个视图：「用量看板」以折线图形式展示指定时间范围内的积分和 Token 消耗趋势；「用量明细」以表格形式列出每次调用的详细记录，支持导出为 CSV 文件。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 普通成员 | 用量看板、用量明细 | 查看自己的用量趋势和数据详情；导出自己的用量记录 | 仅能看到自己的用量，无成员/群组/部门筛选入口 |
| 管理员 | 用量看板、用量明细 | 查看全团队所有成员的用量；按成员/群组/部门筛选；按用量来源筛选；导出用量记录 | 拥有成员筛选器，可切换筛选模式查看不同维度数据 |

**权限模型**: 基于用户信息的团队权限字段（`userInfo.team.permission.hasManagePer`），管理员可见成员筛选控件，普通成员仅展示数据。

## 3. 跨模块依赖摘要

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| 账户 | AccountContainer 页面容器（提供统一的账户中心布局） | 组件 |
| 团队管理 | 获取团队成员列表（getTeamMembers）、群组列表（getGroupList）、部门树（getOrgList） | API |
| 用户系统 | useUserStore 获取当前用户信息和权限 | Store |
| 钱包/计费 | 用量列表查询（getUserUsages）、看板数据查询（getDashboardData）、用量导出、用量来源枚举和标签映射 | API |
| 充值 | RechargeModal 充值弹窗（查看剩余积分的快捷入口） | 组件 |

本模块未检测到对外共享能力。模块内的组件（UsageTableList、UsageDashboard、DashboardChart、UsageDetail、OrgTreeSelector）均为模块内部使用，未被其他模块引用。
