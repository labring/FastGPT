# 业务流程详解

## 流程 1：查看推广概览数据

**触发时机**：页面加载时自动执行

**前置条件**：
- 用户已登录且为团队所有者
- 功能开关 `show_promotion` 已启用

**执行步骤**：

1. 页面组件挂载，发起 `getPromotionInitData` 请求
2. 服务端返回 `invitedAmount`（累计邀请人数）和 `earningsAmount`（收益金额）
3. 页面渲染四个统计卡片：
   - **累计邀请人数**：显示 `invitedAmount` 数值
   - **收益**：显示 `earningsAmount` 金额（单位：元）
   - **返现比例**：显示 `userInfo.promotionRate`（若无则默认 15%），附带说明气泡（"好友充值时你将获得一定比例的余额奖励"）
   - **邀请链接**：显示"复制邀请链接"按钮

**异常处理**：
- 请求失败时，统计数据默认显示 `0`
- `promotionRate` 为空时回退显示 `15%`

**输出结果**：统计卡片渲染完毕，用户可查看推广概况

## 流程 2：复制邀请链接

**触发时机**：用户点击"复制邀请链接"按钮

**前置条件**：用户信息已加载（`userInfo._id` 存在）

**执行步骤**：

1. 用户点击按钮
2. 系统拼接邀请 URL：`{当前域名}/?hiId={用户ID}`
3. 通过 `useCopyData` hook 将 URL 复制到系统剪贴板
4. 用户可将链接分享给好友

**业务规则**：
- 好友通过该链接注册后，永久与邀请人绑定
- 好友使用手机号注册时，邀请人立即获得 5 元奖励
- 好友后续充值时，邀请人按返现比例获得奖励
- 奖励发放至邀请人的默认团队

**异常处理**：复制失败时由 `useCopyData` hook 内部处理（通常由浏览器剪贴板 API 抛出）

**输出结果**：邀请链接已复制到剪贴板

## 流程 3：查看推广记录列表

**触发时机**：页面加载时自动执行，用户可翻页浏览

**前置条件**：
- 用户已登录且为团队所有者
- 功能开关 `show_promotion` 已启用

**执行步骤**：

1. 页面组件挂载，通过 `usePagination` hook 发起 `getPromotionRecords` 请求（默认每页 20 条）
2. 服务端返回分页数据：记录列表 + 总条数
3. 页面渲染表格，包含三列：
   - **时间**：记录创建时间，格式 `YYYY/MM/DD HH:mm:ss`，无数据时显示 `-`
   - **类型**：显示对应中文（"注册奖励" / "支付返佣"）
   - **金额**：显示奖励金额
4. 用户可通过底部翻页组件切换页码

**数据格式**：
- 记录类型 `type` 取值：`register`（注册奖励）、`pay`（支付返佣）
- 记录类型通过 i18n key `user:promotion.{type}` 映射为中文显示

**异常处理**：
- 请求加载中显示 Loading 状态
- 记录为空时显示"暂无邀请记录"空状态提示
- 请求失败由 `usePagination` 内部处理

**输出结果**：推广记录表格渲染完毕
