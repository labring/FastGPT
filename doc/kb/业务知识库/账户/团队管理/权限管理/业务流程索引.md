# 权限管理 — 业务流程索引

## 查看协作者权限列表

- **业务描述**: 以表格形式展示团队内所有协作者（成员、组织、分组）及其对各资源的创建权限，按成员/组织/分组分三段可折叠展示。
- **触发入口**: 团队管理页面 → 点击「权限管理」Tab
- **路由路径**: /account/team?teamTab=permission
- **页面组件位置**: projects/app/src/pageComponents/account/team/PermissionManage/index.tsx
- **涉及角色**: 团队所有者、团队管理员、普通成员
- **前置条件**: 用户已加入团队，团队权限信息已加载

## 搜索协作者

- **业务描述**: 在搜索框中输入关键字，实时过滤协作者列表，只显示匹配的成员、组织或分组。
- **触发入口**: 权限管理页面 → 顶部搜索框输入
- **路由路径**: /account/team?teamTab=permission
- **页面组件位置**: projects/app/src/pageComponents/account/team/PermissionManage/index.tsx
- **涉及角色**: 全部角色
- **前置条件**: 协作者列表已加载

## 修改单个协作者权限

- **业务描述**: 为某个协作者勾选或取消勾选某项资源创建权限（应用/模型/技能/数据集/API Key/评测/管理），实时生效。
- **触发入口**: 权限管理页面 → 在目标协作者行中勾选/取消勾选对应列
- **路由路径**: /account/team?teamTab=permission
- **页面组件位置**: projects/app/src/pageComponents/account/team/PermissionManage/index.tsx
- **涉及角色**: 团队所有者（全部）、团队管理员（非管理员协作者）
- **前置条件**: 当前用户拥有相应的管理权限；部分 Checkbox 对非所有者/非管理员不可操作

## 删除协作者

- **业务描述**: 从协作者列表中移除某个成员、组织或分组，该协作者将不再拥有团队内的任何权限。
- **触发入口**: 权限管理页面 → 目标协作者行最右侧 → 点击删除图标
- **路由路径**: /account/team?teamTab=permission
- **页面组件位置**: projects/app/src/pageComponents/account/team/PermissionManage/index.tsx
- **涉及角色**: 团队所有者（全部，除自己）、团队管理员（非管理员协作者）
- **前置条件**: 当前用户拥有删除权限（hasDeletePer 判断），且不能删除自己

## 添加协作者（管理权限入口）

- **业务描述**: 打开添加协作者的弹窗，为团队添加新的成员/组织/分组并设置初始权限。
- **触发入口**: 权限管理页面 → 顶部「管理权限」按钮
- **路由路径**: /account/team?teamTab=permission
- **页面组件位置**: projects/app/src/components/support/permission/MemberManager/MemberModal.tsx（通过 CollaboratorContextProvider 动态加载）
- **涉及角色**: 团队所有者、团队管理员
- **前置条件**: 当前用户拥有团队管理权限（userInfo.team.permission.hasManagePer）

## 场景关系

- **查看协作者权限列表** → **搜索协作者**: 搜索是对列表的实时过滤，无跳转
- **查看协作者权限列表** → **修改单个协作者权限**: 在列表内直接操作，无跳转
- **查看协作者权限列表** → **删除协作者**: 在列表内直接操作，无页面跳转，删除后列表自动刷新
- **查看协作者权限列表** → **添加协作者**: 点击按钮打开弹窗，完成后列表自动刷新

所有场景均在同一个页面内完成，无页面跳转。
