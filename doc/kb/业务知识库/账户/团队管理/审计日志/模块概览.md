# 审计日志 — 模块概览

## 1. 模块业务概述

审计日志模块为团队管理员提供全团队操作行为的集中查看与审计能力。系统自动记录团队成员在应用管理、知识库管理、权限变更、团队组织调整等关键业务操作中产生的日志，管理员可通过成员筛选、事件类型筛选和无限滚动分页快速定位和追溯历史操作记录。

本模块位于团队管理页面的审计日志标签页，仅对拥有团队管理权限的角色可见。它是团队安全合规和问题追溯的核心工具，帮助管理员监控成员行为、排查误操作和审计权限变更。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 团队管理员（hasManagePer） | 审计日志标签页 | 查看所有成员操作日志、按成员筛选、按事件类型筛选 | 仅团队管理员可访问此页面 |
| 普通成员 | 无 | 无 | 不显示审计日志标签页 |

**权限模型**: 基于 `userInfo.team.permission.hasManagePer` 的团队管理权限判断。标签页可见性在父组件 `Team` 中通过条件渲染控制，同时还需要当前套餐支持审计日志存储时长（`auditLogStoreDuration`）才允许切换进入。
