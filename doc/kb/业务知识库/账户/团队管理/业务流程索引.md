# 团队管理 — 业务流程索引

## 业务场景列表

### 进入团队管理页面

- **业务描述**: 用户从账户侧边栏选择"团队"进入团队管理页面，页面自动加载当前团队的成员统计和 Tab 内容
- **触发入口**: 账户侧边栏 → 点击"团队"菜单项；或通过邀请链接携带 invitelinkid 参数直接进入
- **路由路径**: /account/team
- **页面组件位置**: projects/app/src/pages/account/team/index.tsx
- **涉及角色**: 所有团队成员
- **前置条件**: 用户已登录且已加入至少一个团队

### 切换团队

- **业务描述**: 用户通过页面头部的团队选择器切换到其他已加入的团队，切换后页面自动刷新
- **触发入口**: 团队管理页面头部 → 点击团队选择器下拉
- **路由路径**: /account/team（页面刷新）
- **页面组件位置**: projects/app/src/pageComponents/account/TeamSelector.tsx
- **涉及角色**: 所有团队成员
- **前置条件**: 用户已加入多个团队

### 编辑团队信息

- **业务描述**: 团队所有者点击编辑图标打开编辑弹窗，修改团队名称、头像和通知账号
- **触发入口**: 团队管理页面头部 → 点击编辑图标（仅团队所有者可见）
- **路由路径**: /account/team
- **页面组件位置**: projects/app/src/pageComponents/account/team/context.tsx → EditInfoModal
- **涉及角色**: 团队所有者
- **前置条件**: 用户为当前团队的 owner 角色

### 接受团队邀请

- **业务描述**: 用户通过邀请链接进入团队管理页面时，自动弹出邀请处理弹窗，可选择接受或拒绝邀请
- **触发入口**: 通过包含 invitelinkid 参数的 URL 进入
- **路由路径**: /account/team?invitelinkid={id}
- **页面组件位置**: projects/app/src/pageComponents/account/team/Invite/HandleInviteModal.tsx
- **涉及角色**: 被邀请用户
- **前置条件**: URL 包含有效的 invitelinkid 参数，用户已登录

## 场景关系

各 Tab 场景独立执行，无严格先后关系。进入团队管理页面后按默认 Tab（成员）展示，用户可按需切换。审计日志 Tab 受订阅计划限制（需计划含审计日志存储时长配置），不满足时切换被拦截并提示无权限。
