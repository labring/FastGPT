# 分组 — 模块概览

## 1. 模块业务概述

分组模块是团队管理下的子能力，负责团队成员的分组管理。团队管理员可创建多个分组，将团队成员划分到不同分组中，每个分组拥有独立的负责人（owner）和管理员（admin），以实现团队内部的精细化权限和协作管理。

分组模块服务于团队内所有角色：团队管理员通过分组面板统一管理分组的创建、编辑、成员分配和负责人转让；分组成员可在权限范围内查看和修改所属分组的成员配置。分组信息是 FastGPT 权限体系的基础，资源（如应用、知识库）的协作授权可精确到分组级别。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 团队管理员 (hasManagePer) | 分组列表页 | 创建分组、编辑所有分组信息、管理所有分组成员、删除分组、转让分组负责人 | 最高权限，可操作所有分组 |
| 分组负责人 (isOwner) | 分组列表页 | 编辑分组信息、管理分组成员（增删成员、设置管理员）、转让负责人、删除分组 | 对自己负责的分组有完全管理权限 |
| 分组管理员 (admin) | 分组列表页 | 管理普通成员（增删） | 仅可管理 member 角色的成员，不可修改负责人和管理员 |
| 普通成员 (member) | 分组列表页 | 查看分组信息 | 仅可查看，无管理操作权限 |

**权限模型**: 基于 `Permission` 类的位运算 RBAC 模型。`hasManagePer` 控制团队级管理权限（创建分组按钮可见），`isOwner` 和 `hasManagePer`（分组级）控制分组级操作按钮（编辑、管理成员、转让、删除）。各个操作在组件中通过 `group.permission.hasManagePer` 和 `group.permission.isOwner` 条件渲染。

## 4. 对外共享

本模块为团队管理内部子模块，组件通过 `dynamic import` 懒加载方式被团队管理主页消费。

| 共享能力 | 消费模块 | 能力类型 |
|---------|---------|---------|
| GroupManage（分组管理主组件） | 团队管理（team/index.tsx） | 页面 |
| GroupInfoModal（分组信息编辑弹窗） | 团队管理-分组（内部使用） | 组件 |
| GroupManageMember（成员管理弹窗） | 团队管理-分组（内部使用） | 组件 |
