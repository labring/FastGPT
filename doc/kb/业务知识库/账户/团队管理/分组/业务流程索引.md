# 分组 — 业务流程索引

## 查看分组列表

- **业务描述**: 进入团队管理的分组 Tab，查看当前团队所有分组的列表，包含分组名称、负责人、成员头像及人数，以及对应的操作菜单。
- **触发入口**: 团队管理页面 → 点击"分组" Tab
- **路由路径**: /account/team?teamTab=group
- **页面组件位置**: pageComponents/account/team/GroupManage/index.tsx
- **涉及角色**: 全部团队成员
- **前置条件**: 用户已加入团队，当前团队有已创建的分组

## 创建分组

- **业务描述**: 团队管理员创建新的成员分组，设置分组名称和头像。
- **触发入口**: 分组列表页 → 点击"创建分组"按钮
- **路由路径**: /account/team?teamTab=group（弹窗操作，不改变路由）
- **页面组件位置**: pageComponents/account/team/GroupManage/GroupInfoModal.tsx
- **涉及角色**: 团队管理员（hasManagePer）
- **前置条件**: 用户拥有团队管理权限

## 编辑分组信息

- **业务描述**: 分组负责人或管理员修改分组的名称和头像。
- **触发入口**: 分组列表页 → 点击分组行的操作菜单 → 选择"编辑信息"
- **路由路径**: /account/team?teamTab=group（弹窗操作，不改变路由）
- **页面组件位置**: pageComponents/account/team/GroupManage/GroupInfoModal.tsx
- **涉及角色**: 分组负责人、分组管理员
- **前置条件**: 用户拥有该分组的管理权限（hasManagePer）

## 管理分组成员

- **业务描述**: 分组负责人或管理员添加、移除分组成员，设置/取消分组管理员角色。
- **触发入口**: 分组列表页 → 点击分组行的操作菜单 → 选择"管理成员"（或直接点击成员头像区域）
- **路由路径**: /account/team?teamTab=group（弹窗操作，不改变路由）
- **页面组件位置**: pageComponents/account/team/GroupManage/GroupManageMember.tsx
- **涉及角色**: 分组负责人、分组管理员
- **前置条件**: 用户拥有该分组的管理权限（hasManagePer）

## 转让分组负责人

- **业务描述**: 分组负责人将分组的所有权转让给团队内的另一个成员，可选择是否保留原负责人的管理员权限。
- **触发入口**: 分组列表页 → 点击分组行的操作菜单 → 选择"转让负责人"
- **路由路径**: /account/team?teamTab=group（弹窗操作，不改变路由）
- **页面组件位置**: pageComponents/account/team/GroupManage/GroupTransferOwnerModal.tsx
- **涉及角色**: 分组负责人（isOwner）
- **前置条件**: 用户是该分组的负责人

## 删除分组

- **业务描述**: 分组负责人删除不再需要的分组。
- **触发入口**: 分组列表页 → 点击分组行的操作菜单 → 选择"删除"
- **路由路径**: /account/team?teamTab=group（弹窗操作，不改变路由）
- **页面组件位置**: pageComponents/account/team/GroupManage/index.tsx（内联确认弹窗）
- **涉及角色**: 分组负责人（isOwner）
- **前置条件**: 用户是该分组的负责人

## 场景关系

- **查看分组列表** → **创建分组**: 从列表页点击创建按钮进入创建弹窗
- **查看分组列表** → **编辑分组信息**: 从列表页操作菜单进入编辑弹窗
- **查看分组列表** → **管理分组成员**: 从列表页操作菜单或成员头像区域进入成员管理弹窗
- **查看分组列表** → **转让分组负责人**: 从列表页操作菜单进入转让弹窗
- **查看分组列表** → **删除分组**: 从列表页操作菜单触发删除确认
- 创建、编辑、成员管理、转让操作成功后均自动刷新分组列表
