# 成员 — 模块概览

## 1. 模块业务概述

成员管理是团队管理下的核心子模块，提供团队成员的全生命周期管理能力。该模块以表格形式展示团队内所有成员的信息，支持成员的搜索、筛选、邀请、编辑、移除和恢复等操作。模块面向团队所有者、管理员和普通成员三类角色，不同角色可见的操作入口和可执行操作有显著差异。

在整体产品中，成员管理是团队协作的基础——所有团队功能（应用协作、知识库共享、权限分配）都依赖于成员列表的正确性和完整性。成员管理位于"账户 → 团队管理 → 成员"Tab 下，是团队管理页面的默认展示 Tab。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 团队所有者（owner） | 成员列表 | 查看、搜索、筛选、邀请、编辑名称、移除/禁用、恢复、导出、同步、标签同步、转让所有权 | 拥有完整管理权限，可见所有操作按钮；是唯一可执行"转让所有权"的角色 |
| 团队管理员（hasManagePer） | 成员列表 | 查看、搜索、筛选、邀请、编辑名称、移除/禁用、恢复、同步、标签同步 | 拥有管理权限，可管理普通成员但不可转让所有权；非所有者时可见退出团队入口 |
| 普通成员 | 成员列表 | 查看、搜索、筛选、退出团队 | 仅可查看成员信息和退出团队，不可执行管理操作 |

**权限模型**: 基于 `userInfo.team.permission` 对象的前端条件渲染。权限判断逻辑入口位于 `MemberTable.tsx` 组件中，通过以下关键字段控制 UI 差异：
- `permission.hasManagePer` — 是否具备管理权限（控制邀请、编辑、移除、同步等按钮可见性）
- `permission.isOwner` — 是否为团队所有者（控制转让所有权和导出的可见性）

此外，系统配置 `feConfigs.register_method` 决定是否处于同步模式（`isSyncMode`），影响按钮文案和交互行为。

## 4. 对外共享

本模块为团队管理内部的叶子节点，组件均为内部使用。MemberTable 仅被父级页面 `team/index.tsx` 通过动态导入引用，InviteModal 和 TransferOwnershipModal 仅被 MemberTable 内部引用，不对外共享。
