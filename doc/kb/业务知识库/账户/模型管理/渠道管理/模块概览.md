# 渠道管理 — 模块概览

## 1. 模块业务概述

渠道管理是 AI 模型提供商渠道的集中配置模块，允许管理员为平台接入和管理多个 AI 模型供应商（如 OpenAI、Azure、HuggingFace 等）的 API 渠道。每个渠道包含供应商类型、API 密钥、基础 URL、可选模型列表及模型名称映射等配置信息。

本模块服务于 **root 超级管理员**，是平台 AI 能力的底层基础设施配置入口。渠道配置直接影响系统中所有可用的 AI 模型列表，进而影响工作流、Agent 等上层功能的模型选择范围。

渠道支持优先级排序和启用/禁用切换，管理员可以根据成本和性能灵活调度多个供应商的 API 请求。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| root | 模型管理→渠道 Tab | 创建、编辑、删除渠道；启用/禁用渠道；调整优先级；测试模型 | 仅 root 用户且系统配置 `show_aiproxy` 开启时可见渠道管理相关 Tab |
| 普通管理员 | 无 | 无 | 模型管理页仅显示"可用模型"和"模型配置"两个 Tab，无法访问渠道管理 |

**权限模型**: 基于用户身份的前端条件渲染，通过 `userInfo.username === 'root'` 判断是否为超级管理员，结合系统配置 `feConfigs.show_aiproxy` 控制渠道 Tab 的可见性。
