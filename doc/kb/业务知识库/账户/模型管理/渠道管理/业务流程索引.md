# 渠道管理 — 业务流程索引

## 业务场景

### 查看渠道列表

- **业务描述**: 查看系统中已配置的所有 AI 供应商渠道信息，包括渠道名称、类型、状态、优先级等。
- **触发入口**: 进入模型管理页面，切换到"渠道"Tab
- **路由路径**: /account/model
- **页面组件位置**: `projects/app/src/pageComponents/account/model/Channel/index.tsx`
- **涉及角色**: root
- **前置条件**: 用户为 root 且系统开启 `show_aiproxy` 配置

### 创建渠道

- **业务描述**: 新增一个 AI 供应商渠道，配置供应商类型、名称、API 密钥、基础 URL、可选模型及模型映射。
- **触发入口**: 渠道列表页→"创建渠道"按钮
- **路由路径**: /account/model
- **页面组件位置**: `projects/app/src/pageComponents/account/model/Channel/EditChannelModal.tsx`
- **涉及角色**: root
- **前置条件**: 用户为 root，已打开渠道列表页

### 编辑渠道

- **业务描述**: 修改已有渠道的配置信息，包括名称、供应商类型、模型列表、模型映射、API 密钥、基础 URL 等。
- **触发入口**: 渠道列表→操作菜单→"编辑"
- **路由路径**: /account/model
- **页面组件位置**: `projects/app/src/pageComponents/account/model/Channel/EditChannelModal.tsx`
- **涉及角色**: root
- **前置条件**: 渠道已存在

### 启用渠道

- **业务描述**: 将已禁用的渠道恢复为启用状态，使其关联的模型重新可用。
- **触发入口**: 渠道列表→操作菜单→"启用"
- **路由路径**: /account/model
- **页面组件位置**: `projects/app/src/pageComponents/account/model/Channel/index.tsx`
- **涉及角色**: root
- **前置条件**: 渠道当前状态为"已禁用"

### 禁用渠道

- **业务描述**: 将已启用的渠道设为禁用状态，停止该渠道关联模型的服务。
- **触发入口**: 渠道列表→操作菜单→"禁用"
- **路由路径**: /account/model
- **页面组件位置**: `projects/app/src/pageComponents/account/model/Channel/index.tsx`
- **涉及角色**: root
- **前置条件**: 渠道当前状态为"已启用"

### 删除渠道

- **业务描述**: 从系统中移除一个渠道配置。
- **触发入口**: 渠道列表→操作菜单→"删除"
- **路由路径**: /account/model
- **页面组件位置**: `projects/app/src/pageComponents/account/model/Channel/index.tsx`
- **涉及角色**: root
- **前置条件**: 渠道已存在

### 测试模型

- **业务描述**: 对渠道下关联的模型进行连通性测试，验证 API 密钥和端点是否可用。
- **触发入口**: 渠道列表→操作菜单→"测试"
- **路由路径**: /account/model
- **页面组件位置**: `projects/app/src/pageComponents/account/model/Channel/ModelTest.tsx`
- **涉及角色**: root
- **前置条件**: 渠道存在且关联了模型，系统模型列表已加载

### 调整渠道优先级

- **业务描述**: 修改渠道的优先级数值，影响多渠道时的请求分发顺序。
- **触发入口**: 渠道列表→优先级输入框
- **路由路径**: /account/model
- **页面组件位置**: `projects/app/src/pageComponents/account/model/Channel/index.tsx`
- **涉及角色**: root
- **前置条件**: 渠道已存在

## 场景关系

- **查看渠道列表** 是所有操作的基础入口，编辑、删除、启停、测试、优先级调整均从列表页发起
- **创建渠道** 不依赖已有渠道，可独立执行
- **编辑渠道** 可跳转到模型编辑弹窗（AddModelBox），创建或修改系统级模型定义
- 其他操作场景之间无严格先后依赖关系
