# 配置模型 — 业务流程索引

## 业务场景列表

### 查看模型列表

- **业务描述**：进入「配置模型」Tab 后，系统展示所有已接入的模型，按供应商和类型分组，支持多维度筛选。
- **触发入口**：账户 → 模型管理 → 点击「模型配置」Tab
- **路由路径**：/account/model?tab=config
- **页面组件位置**：`projects/app/src/pageComponents/account/model/ModelConfigTable.tsx`
- **涉及角色**：root、团队管理员、团队成员
- **前置条件**：用户已登录；页面加载时自动获取模型列表

### 筛选与搜索模型

- **业务描述**：通过供应商下拉框、模型类型筛选、名称搜索、激活状态筛选四个维度快速定位目标模型。
- **触发入口**：配置模型页面的筛选栏
- **路由路径**：/account/model
- **页面组件位置**：`projects/app/src/pageComponents/account/model/ModelConfigTable.tsx`
- **涉及角色**：root、团队管理员、团队成员
- **前置条件**：模型列表已加载

### 创建模型

- **业务描述**：管理员从「新增模型」下拉菜单中选择模型类型，弹出编辑窗口，填写模型参数后创建新的自定义模型。
- **触发入口**：配置模型页面 → 点击「新增模型」按钮 → 选择模型类型
- **路由路径**：/account/model
- **页面组件位置**：`projects/app/src/pageComponents/account/model/AddModelBox.tsx`
- **涉及角色**：root、有模型创建权限的团队成员
- **前置条件**：用户拥有模型创建权限（`hasModelCreatePer`）

### 编辑模型

- **业务描述**：管理员点击模型的「编辑」图标，弹出编辑窗口修改模型参数（名称、价格、上下文长度、请求地址等）。
- **触发入口**：模型列表行 → 点击「编辑」按钮
- **路由路径**：/account/model
- **页面组件位置**：`projects/app/src/pageComponents/account/model/AddModelBox.tsx`（ModelEditModal）
- **涉及角色**：root、拥有模型管理权限的团队成员
- **前置条件**：操作者对目标模型拥有 `hasManagePer` 权限

### 启用/禁用模型

- **业务描述**：通过开关切换模型的启用/禁用状态，控制模型是否对外可用。
- **触发入口**：模型列表行 → 切换「激活」开关
- **路由路径**：/account/model
- **页面组件位置**：`projects/app/src/pageComponents/account/model/ModelConfigTable.tsx`
- **涉及角色**：root、拥有模型管理权限的团队成员
- **前置条件**：操作者对目标模型拥有 `hasManagePer` 权限；如模型被引用，可能因冲突阻止操作

### 测试模型

- **业务描述**：点击模型的「测试」按钮，向模型发送测试请求，验证模型连通性与响应。
- **触发入口**：模型列表行 → 点击「发送」测试图标
- **路由路径**：/account/model
- **页面组件位置**：`projects/app/src/pageComponents/account/model/ModelConfigTable.tsx`
- **涉及角色**：所有角色
- **前置条件**：模型列表已加载

### 管理模型权限

- **业务描述**：为模型分配协作者，设定哪些团队成员可以查看或管理该模型。
- **触发入口**：模型列表行 → 点击「钥匙」权限图标
- **路由路径**：/account/model
- **页面组件位置**：`projects/app/src/pageComponents/account/model/ModelConfigTable.tsx`（LazyCollaboratorProvider）
- **涉及角色**：root、拥有模型管理权限的团队成员
- **前置条件**：操作者对目标模型拥有 `hasManagePer` 权限

### 删除模型

- **业务描述**：删除自定义模型，系统检查模型是否被其他资源引用，如有引用则弹出警告阻止删除。
- **触发入口**：模型列表行 → 点击「删除」图标 → 确认弹窗
- **路由路径**：/account/model
- **页面组件位置**：`projects/app/src/pageComponents/account/model/ModelConfigTable.tsx`
- **涉及角色**：root、拥有模型管理权限的团队成员
- **前置条件**：目标模型为自定义模型（`isCustom === true`）；操作者拥有 `hasManagePer` 权限

### JSON 配置文件管理

- **业务描述**：root 用户可通过 JSON 编辑器查看和全量覆盖模型配置，实现批量配置的导入导出。
- **触发入口**：配置模型页面 → 点击「配置文件」按钮
- **路由路径**：/account/model
- **页面组件位置**：`projects/app/src/pageComponents/account/model/ModelConfigTable.tsx`（JsonConfigModal）
- **涉及角色**：仅 root
- **前置条件**：用户为 root

## 场景关系

- **查看模型列表** → **筛选与搜索模型**：筛选操作基于已加载的列表数据，不触发额外 API 请求
- **创建模型** → **查看模型列表**：创建成功后自动刷新列表，新模型出现在列表中
- **编辑模型** → **查看模型列表**：编辑成功后自动刷新列表
- **启用/禁用模型** → **查看模型列表**：状态变更后自动刷新列表
- **删除模型** → **查看模型列表**：删除成功后自动刷新列表
- **JSON 配置文件管理** → **查看模型列表**：覆盖成功后自动刷新列表
- 管理模型权限、测试模型与列表查看为独立操作，无严格先后关系
