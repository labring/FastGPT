# 配置模型 — 模块概览

## 1. 模块业务概述

配置模型是「账户 → 模型管理」下的子能力，提供系统级模型的集中配置与管理功能。管理员可在此查看所有已接入的模型（包含 LLM、Embedding、TTS、STT、Rerank 五类），对模型进行启用/禁用、参数编辑、权限分配等操作，并支持新增自定义模型。

本模块面向**系统管理员（root）**和**有模型管理权限的团队成员**，是整个 AI Agent 平台中模型资源池的统一管控入口。与「可用模型」Tab（侧重于模型的直接使用）不同，本模块侧重于模型的**配置与管理**——包括模型参数的修改、权限归属的设定以及模型的上下架控制。

本模块为叶子能力，无子能力。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| root | 配置模型 Tab | 全部操作：查看所有模型、创建模型、编辑模型、启用/禁用、权限管理、删除模型、JSON 配置文件导入 | 可见「JSON 配置文件」按钮，可通过 JSON 全量覆盖模型配置 |
| 团队管理员 | 配置模型 Tab | 查看团队可见模型、创建模型、编辑有权限的模型、启用/禁用、权限管理、删除自有模型 | 不可见「JSON 配置文件」按钮；仅能编辑/删除有管理权限的模型 |
| 团队成员 | 配置模型 Tab | 查看团队可见模型、测试模型 | 不可新增/编辑/删除模型，操作按钮禁用或隐藏 |

**权限模型**：基于 `ModelPermission` 的细粒度权限控制（继承自 `Permission` 基类），每个模型拥有独立的 `hasManagePer` 权限位。模型创建权限通过 `userInfo.team.permission.hasModelCreatePer` 判断。root 用户拥有全部操作权限。
