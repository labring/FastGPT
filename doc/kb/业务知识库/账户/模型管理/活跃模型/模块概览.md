# 活跃模型 — 模块概览

## 1. 模块业务概述

活跃模型模块是「账户 → 模型管理」页面下的核心 Tab，用于集中展示和管理团队中所有处于活跃状态的 AI 模型。该模块以双 Tab（基座模型 / 自定义模型）的表格视图呈现模型的完整生命周期管理能力，覆盖模型查看、筛选、排序、训练、权限配置、默认模型设置等日常运维场景。

本模块面向三类角色：**root 用户**可设置系统级默认模型；**团队管理员**可批量配置模型的团队协作权限；**普通团队成员**可浏览模型列表、查看训练详情和价格信息。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| root 用户 | 模型管理 → 活跃模型 | 查看全部模型列表、筛选/排序、设置默认模型、查看训练详情、创建训练任务 | 可见「默认模型」按钮，可设置系统级默认模型 |
| 团队管理员 | 模型管理 → 活跃模型 | 查看有管理权限的模型、批量配置权限、查看训练详情、创建训练任务 | 可见批量权限操作栏，模型列表中显示复选框和权限列 |
| 团队成员 | 模型管理 → 活跃模型 | 浏览模型列表、查看训练详情、查看价格信息 | 仅可查看，无管理操作入口 |

**权限模型**: 基于 `TeamPermission` 的团队权限体系。组件通过 `userInfo.team.permission.hasManagePer` 判断当前用户是否为团队管理员；通过 `userInfo.username === 'root'` 判断是否为 root 用户。模型级别通过 `item.permission.hasManagePer` 逐行控制复选框可用性。
