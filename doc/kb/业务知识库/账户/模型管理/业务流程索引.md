# 模型管理 — 业务流程索引

## Tab 结构

模型管理页面使用 `MyTabs` 组件实现 Tab 切换，共 5 个一级 Tab：

```
模型管理 (/account/model)
├── 活跃模型 (model)        — 所有角色可见
│   ├── 基础模型 (base)      — ModelTable 内部 Tab
│   └── 自定义模型 (custom)  — ModelTable 内部 Tab
├── 配置模型 (config)        — 所有角色可见
├── 渠道管理 (channel)       — root 专属 [动态可见]
├── 渠道日志 (channel_log)   — root 专属 [动态可见]
└── 模型监控 (account_model) — root 专属 [动态可见]
```

> root 专属 Tab 的可见条件：`isRoot && feConfigs?.show_aiproxy`

## 公共操作

以下操作不隶属于特定 Tab，属于页面级公共功能：

| 操作 | 描述 | 角色 |
|------|------|------|
| 页面初始化 | 加载服务端渲染数据（i18n 资源命名空间：account, account_model, user, app, train, chat） | 全部 |
| Tab 切换 | 通过路由 `/account/model` 进入，默认激活"活跃模型"Tab；点击顶部 MyTabs 切换 | 全部 |
| 账户侧边栏导航 | 左侧 DashboardNavbar 提供账户子页面切换入口 | 全部 |
| 退出登录 | 通过 AccountContainer 侧边栏"退出登录"选项触发 | 全部 |

## 场景关系

- **活跃模型 Tab** 与 **配置模型 Tab** 各自独立，无先后依赖关系。活跃模型展示从 Store 中获取的模型数据（支持筛选和批量权限操作），配置模型通过 API 获取全量系统模型数据
- **渠道管理** → **渠道日志**：渠道管理中的模型测试操作会触发 API 调用，其结果记录在渠道日志中
- **渠道管理** → **模型监控**：渠道请求的调用量、Token 消耗等数据汇聚到模型监控的 Dashboard 统计中
- root 专属的三个 Tab（渠道管理、渠道日志、模型监控）仅当 `feConfigs.show_aiproxy` 启用且用户为 root 时出现
