# 模型监控 — 模块概览

## 1. 模块业务概述

模型监控是模型管理模块下的一个管理 Tab，为 root 用户提供 AI 模型调用数据的可视化监控能力。本模块以图表和表格两种视图展示模型调用的核心指标，包括请求量、错误率、Token 消耗、AI 积分花费、响应时间、缓存命中率等，帮助管理员实时掌握模型服务的运行状态和资源消耗情况。

本模块仅对 root 用户可见（需同时满足 `show_aiproxy` 功能开关启用），属于管理员专属的数据分析工具。模块提供图表视图（默认）和表格视图两种查看模式，支持按时间范围、渠道和模型维度进行筛选，时间粒度可根据时间跨度自动调整为分钟/小时/天。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| root | 模型监控 Tab | 查看图表视图、切换表格视图、按时间范围/渠道/模型筛选、切换 Token 用量维度、按表格列排序 | root 用户唯一可见此 Tab，其他角色在父页面导航中不显示此入口 |
| owner/admin/member | 不可见 | — | 模型监控 Tab 不在导航列表中渲染 |

**权限模型**：通过父页面 `ModelProvider` 组件中的 `isRoot && feConfigs?.show_aiproxy` 双重条件控制导航 Tab 的可见性。`isRoot` 由 `userInfo.username === 'root'` 判定，`show_aiproxy` 来自 `useSystemStore` 中的 `feConfigs` 系统配置。

## 3. 跨模块依赖摘要

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| AI 渠道服务（@/web/core/ai/channel） | 渠道列表获取、Dashboard V2 监控数据查询 | API |
| AI 模型配置服务（@/web/core/ai/config） | 系统模型列表获取 | API |
| 系统配置（useSystemStore） | feConfigs 功能开关（isPlus 积分展示控制）、模型提供商信息 | Store |
| 共享图表组件（AreaChartComponent） | 面积图渲染，支持多折线、累加模式、自定义 Tooltip | 组件 |
| 共享 UI 组件库 | MyBox 加载容器、DateRangePicker 日期范围选择器、MySelect 下拉选择器、FillRowTabs 选项卡切换 | 组件 |

## 4. 对外共享

本模块为纯消费型模块，不对外提供可复用的组件或能力。
