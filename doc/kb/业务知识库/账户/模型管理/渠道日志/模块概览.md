# 渠道日志 — 模块概览

## 1. 模块业务概述

渠道日志模块为 **root 管理员** 提供 AI 代理渠道（AIProxy）的请求日志查询与详情查看能力。在 FastGPT 的账户管理体系中，当管理员配置了 AI 代理渠道后，该模块用于**监控和排查所有通过渠道转发的 AI 模型请求**，包括请求状态、耗时、Token 消耗等关键指标。

本模块是「模型管理」能力下的一个 Tab 子页面，与渠道管理、模型监控等能力配合使用，共同构成完整的 AI 代理渠道运维视图。渠道日志聚焦于**历史请求的追溯与问题定位**——管理员可在此快速发现失败请求、分析响应时间、查看请求/响应体详情。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| root | 渠道日志列表页 + 日志详情弹窗 | 查看全部渠道日志、按渠道/模型/状态筛选、按日期范围筛选、按 request_id 搜索、查看请求/响应详情 | 拥有 request_id 精确搜索功能，可查看完整请求体和响应体 |
| 其他角色 | 不可见 | 无 | 渠道日志 Tab 仅在 `isRoot && feConfigs.show_aiproxy` 为 true 时显示，普通管理员不可访问 |

**权限模型**: 页面级可见性控制。渠道日志入口的 Tab 渲染条件为 `isRoot && feConfigs?.show_aiproxy`，仅在当前用户名为 root 且系统配置开启了 AI 代理功能时展示。日志详情弹窗中请求体（request_body）和响应体（response_body）仅在接口返回数据时展示。
