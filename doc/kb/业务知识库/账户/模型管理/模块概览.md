# 模型管理 — 模块概览

## 1. 模块业务概述

模型管理模块是 FastGPT 账户中心下的核心功能模块，为用户提供 AI 大模型的全生命周期管理能力。本模块面向**所有已登录用户**，承担模型浏览、配置管理、渠道调度和用量监控四大职能。

普通用户可在"活跃模型"Tab 中浏览系统已启用的模型列表（按基础模型/自定义模型分类），在"配置模型"Tab 中查看所有系统模型的状态和归属信息。root 用户额外可见"渠道管理"、"渠道日志"和"模型监控"三个管理 Tab，用于管理 API 渠道（如 OpenAI 代理）、查看请求日志和监控模型调用数据。

模块通过顶部 MyTabs 选项卡组织 5 个子能力页面，所有内容包裹在 `AccountContainer` 账户统一布局中。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| root | 全部 5 个 Tab（活跃模型、配置模型、渠道管理、渠道日志、模型监控） | 浏览模型、管理配置、创建/编辑/删除渠道、启用/禁用渠道、查看日志详情、设置默认模型、模型训练管理、Json 配置导入 | root 用户是系统超级管理员，拥有所有管理权限，额外可见渠道和监控功能 |
| owner/admin | 活跃模型、配置模型 | 浏览模型列表、查看配置模型详情、创建新模型（如有 `hasModelCreatePer` 权限）、测试模型 | 不能管理渠道、查看日志和监控 |
| member | 活跃模型、配置模型 | 浏览模型列表、查看配置模型详情 | 不能创建模型和修改配置 |

**权限模型**：基于 `userInfo.username === 'root'` 判断 root 身份；模型创建权限通过 `userInfo.team.permission.hasModelCreatePer` 控制；渠道相关功能通过 `feConfigs.show_aiproxy` 功能开关 + root 身份双重控制。

## 3. 跨模块依赖摘要

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| 账户布局（AccountContainer） | 统一侧边栏导航、账户页面容器 | 组件 |
| 用户状态管理（useUserStore） | 用户信息、root 判断、团队权限位 | Store |
| 系统配置（useSystemStore） | feConfigs 功能开关、模型提供商列表、默认模型配置 | Store |
| AI 模型配置服务（@/web/core/ai/config） | 系统模型 CRUD API、模型测试 API | API |
| AI 渠道服务（@/web/core/ai/channel） | 渠道 CRUD API、日志查询 API、Dashboard 数据 API | API |
