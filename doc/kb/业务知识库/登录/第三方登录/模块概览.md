# 第三方登录 — 模块概览

## 1. 模块业务概述

第三方登录模块是 FastGPT 登录体系的 OAuth/SSO 回调处理中心，负责接收第三方身份提供商（如 GitHub、Google、微信、Microsoft、企业微信等）返回的授权凭证，完成用户身份验证和会话建立。

当用户在登录页面选择第三方登录方式后，浏览器会跳转至对应平台的授权页面；授权成功后，第三方平台将用户重定向回本模块的 `/login/provider` 页面，携带授权码和状态参数。本模块解析这些参数，调用后端 OAuth 接口完成登录，最终将用户引导至目标页面。

本模块是登录流程的"最后一公里"，衔接第三方授权与系统内部会话管理，为所有支持 OAuth 登录的用户提供统一的回调入口。

## 2. 角色差异

本模块为回调处理页面，不区分角色 —— 所有尝试通过第三方登录的用户均经过相同的处理流程。页面本身无权限校验逻辑，登录成功后的角色和权限由后端 API 返回的用户信息决定。

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 所有用户 | `/login/provider` | 完成第三方 OAuth 回调登录 | 无差异，统一处理流程 |

**权限模型**：本模块无前端权限判断，登录结果和用户权限完全由后端 `oauthLogin` API 返回决定。

## 3. 跨模块依赖摘要

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| 系统 Store | `useSystemStore` — 读取/写入 loginStore（含 provider、lastRoute、state），读取 feConfigs | Store |
| 用户 Store | `useUserStore` — 登录成功后写入用户信息 setUserInfo | Store |
| 用户认证 API | `oauthLogin` — OAuth 登录接口；`clearToken` — 退出登录并清除令牌 | API |
| 团队 API | `postAcceptInvitationLink` — 登录后自动接受团队邀请链接 | API |
| 营销工具 | `getBdVId`、`getMsclkid`、`getFastGPTSem`、`getSourceDomain`、`getInviterId` — 获取营销追踪参数 | Utils |
| URI 校验 | `validateRedirectUrl` — 防止开放重定向攻击 | Utils |
