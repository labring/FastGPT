# 注册 — 模块概览

## 1. 模块业务概述

注册模块是登录首页中的一个 Tab 子页面，为未登录用户（访客）提供账号注册功能。用户通过填写用户名（支持邮箱或手机号）、验证码、密码及确认密码来完成注册流程。

注册模块在整体产品中承担**用户准入**角色——将访客转化为平台注册用户。注册成功后自动登录并跳转至应用首页，无需用户额外操作。

本模块为登录首页的 Tab 子能力，无下级子能力。

## 2. 角色差异

本模块面向单一角色：**访客（未登录用户）**。

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 访客 | 注册表单 | 输入用户名、获取验证码、设置密码、提交注册 | 唯一可用角色；已登录用户不会访问此页面 |

**权限模型**：本模块无权限判断逻辑——注册页面仅对未登录用户开放，由登录容器 `LoginContainer` 根据页面类型状态切换显示，不涉及 RBAC 权限位。

## 3. 跨模块依赖摘要

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| 用户认证 API 层 | `postRegister` 注册接口、`sendAuthCode` 发送验证码 | API |
| 验证码发送 | `useSendCode` Hook — 发送验证码并管理倒计时状态 | Hook |
| 系统配置 | `useSystemStore` — 读取 `feConfigs.register_method` 决定注册方式 | Store |
| 营销追踪 | `getInviterId`、`getBdVId`、`getMsclkid`、`getFastGPTSem` 等营销参数采集 | 工具函数 |
| 密码校验 | `checkPasswordRule` — 前端密码规则校验 | 工具函数 |

## 4. 对外共享

本模块为纯消费型 Tab 组件，不对外提供共享能力。`RegisterForm` 组件仅被登录容器 `LoginContainer` 内部使用（通过 `next/dynamic` 动态加载）。

| 共享能力 | 消费模块 | 能力类型 |
|---------|---------|---------|
| 无 | — | — |
