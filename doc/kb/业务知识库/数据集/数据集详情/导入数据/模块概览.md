# 导入数据 — 模块概览

## 1. 模块业务概述

导入数据模块是数据集详情页的核心功能之一，负责将各种来源的数据导入到知识库中。用户可以通过此模块将本地文件、网页链接、自定义文本、外部文件、API 数据集、图片、数据库等多种数据源转换为知识库的集合（Collection），为后续的知识检索和 AI 问答提供数据基础。

本模块支持 **8 种导入方式**，涵盖文件上传（本地文件、图片）、链接抓取（网页链接、外部文件 URL）、文本输入、API 数据集对接、已有数据重训练以及数据库直连。管理员和具有管理权限的成员可以对数据集执行导入操作，普通查看者仅能查看已导入的内容。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 管理员 | 数据集详情 → 导入 Tab | 全部 8 种导入方式，含参数配置、数据预览、上传确认 | 可管理所有数据集 |
| 成员（有管理权限） | 数据集详情 → 导入 Tab | 全部 8 种导入方式 | 需要 `permission.hasManagePer` 权限 |
| 普通查看者 | 不直接可见 | 不可导入 | 无管理权限时不展示导入入口 |

**权限模型**: 基于数据集级别的权限控制。导入功能通过 `DatasetPageContext` 中的 `datasetDetail.permission.hasManagePer` 判断当前用户是否有管理权限。导入 Tab 的可见性和操作按钮的可用性均由此权限控制。

## 3. 跨模块依赖摘要

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| 数据集上下文 | `DatasetPageContext` — 获取数据集详情、ID、向量模型等信息 | Context |
| 文件上传 | `getUploadDatasetFilePresignedUrl`、`putFileToS3` — 文件上传至 S3 存储 | API |
| 集合管理 | `postCreateDatasetFileCollection` 等 7 个创建 API — 创建不同类型的知识库集合 | API |
| 公共组件 | `LightRowTabs`、`MyIcon`、`MyBox`、`MyModal`、`FolderPath` — 通用 UI 组件 | 组件 |
| 国际化 | `useTranslation` — 多语言文案 | Hook |

## 6. 子能力索引

本模块为叶子节点，无子能力。8 种导入方式统一在本模块内管理，通过 URL 参数 `source` 区分当前激活的导入源类型。
