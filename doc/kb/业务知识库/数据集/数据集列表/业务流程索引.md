# 数据集列表 — 业务流程索引

## 业务场景

### 浏览数据集列表

- **业务描述**: 用户进入数据集管理页面，以卡片网格的形式浏览当前目录下的所有数据集和子文件夹。默认展示根目录下的所有数据集，可点击文件夹进入子目录。
- **触发入口**: 左侧导航栏 → 数据集菜单项 → 进入列表页
- **路由路径**: `/dataset/list`
- **页面组件位置**: `projects/app/src/pages/dataset/list/index.tsx`
- **涉及角色**: 全部（管理员、协作者、只读用户）
- **前置条件**: 用户已登录，团队已创建

### 搜索数据集

- **业务描述**: 用户通过搜索框输入关键词，按数据集名称或描述进行模糊匹配搜索。
- **触发入口**: 数据集列表页顶部搜索框
- **路由路径**: `/dataset/list`
- **页面组件位置**: `projects/app/src/pages/dataset/list/index.tsx`
- **涉及角色**: 全部
- **前置条件**: 已进入数据集列表页

### 创建数据集

- **业务描述**: 用户通过"新建数据集"下拉菜单选择数据集类型，创建不同类型的知识库。支持的类型包括：通用数据集、网站数据集、文件数据库（结构化文档）、外部数据库、API文件数据集、飞书数据集、语雀数据集等。
- **触发入口**: 数据集列表页右上角"新建数据集"按钮 → 下拉菜单
- **路由路径**: `/dataset/list`
- **页面组件位置**: `projects/app/src/pages/dataset/list/index.tsx` → `CreateModal`
- **涉及角色**: 管理员、协作者（需要有数据集创建权限）
- **前置条件**: 用户团队权限 `hasDatasetCreatePer` 为 true；商业版功能（如网站数据集）需 `feConfigs.isPlus` 为 true

### 创建文件夹

- **业务描述**: 用户在数据集列表中创建新文件夹，用于组织和管理数据集。创建时需要填写名称、描述，可上传头像。
- **触发入口**: 数据集列表页右上角"新建文件夹"按钮
- **路由路径**: `/dataset/list`
- **页面组件位置**: `projects/app/src/pages/dataset/list/index.tsx` → `EditFolderModal`
- **涉及角色**: 管理员、协作者（需要有写入权限）
- **前置条件**: 当前文件夹有写入权限（`folderDetail.permission.hasWritePer`）或用户有全局数据集创建权限

### 编辑数据集信息

- **业务描述**: 用户编辑已有数据集的名称、描述和头像信息。
- **触发入口**: 数据集卡片悬浮 → 更多按钮 → 编辑信息；或顶部编辑按钮（针对文件夹）
- **路由路径**: `/dataset/list`
- **页面组件位置**: `projects/app/src/pageComponents/dataset/list/NewList.tsx` → `EditFolderModal` / `CreateModal`
- **涉及角色**: 管理员、协作者（需要有管理权限）
- **前置条件**: 数据集卡片有管理权限（`dataset.permission.hasManagePer`）

### 配置权限

- **业务描述**: 用户为数据集或文件夹配置访问权限，包括添加/移除协作者、修改协作者角色、更改所有者、恢复继承权限等。
- **触发入口**: 数据集卡片悬浮 → 更多按钮 → 权限；或顶部权限按钮（针对文件夹）
- **路由路径**: `/dataset/list`
- **页面组件位置**: `projects/app/src/pages/dataset/list/index.tsx` / `projects/app/src/pageComponents/dataset/list/NewList.tsx` → `ConfigPerModal`
- **涉及角色**: 管理员、协作者（需要有管理权限）
- **前置条件**: 当前用户对目标数据集有管理权限

### 移动数据集

- **业务描述**: 用户通过拖拽将数据集移动到其他文件夹中，或通过菜单选项移动数据集。
- **触发入口**: 拖拽数据集卡片到目标文件夹；或卡片菜单 → 移动
- **路由路径**: `/dataset/list`
- **页面组件位置**: `projects/app/src/pageComponents/dataset/list/NewList.tsx` → `useFolderDrag`
- **涉及角色**: 管理员、协作者（需要有父级数据集的管理权限）
- **前置条件**: 目标父文件夹有管理权限

### 删除数据集

- **业务描述**: 用户删除数据集或文件夹。删除时需输入数据集名称确认。文件夹删除前检查是否有关联应用。被应用引用的数据集不可删除。
- **触发入口**: 数据集卡片悬浮 → 更多按钮 → 删除
- **路由路径**: `/dataset/list`
- **页面组件位置**: `projects/app/src/pageComponents/dataset/list/NewList.tsx`
- **涉及角色**: 管理员、协作者（需要有管理权限）
- **前置条件**: 数据集有管理权限；数据集未被任何应用引用（appCount 为 0）

### 导出数据集

- **业务描述**: 用户将数据集内容导出为 CSV 文件下载。导出前检查团队导出限额。
- **触发入口**: 数据集卡片悬浮 → 更多按钮 → 导出
- **路由路径**: `/dataset/list`
- **页面组件位置**: `projects/app/src/pageComponents/dataset/list/NewList.tsx`
- **涉及角色**: 管理员、协作者（需要有管理权限）
- **前置条件**: 数据集类型为非文件夹、非数据库类型；团队导出限额未超

## 场景关系

- **浏览数据集列表** → **创建数据集** / **创建文件夹**: 用户需先浏览列表才能决定在何处创建
- **浏览数据集列表** → **编辑** / **权限配置** / **移动** / **删除** / **导出**: 所有管理操作均以列表浏览为入口
- **搜索** → **浏览结果**: 搜索是对列表的筛选，结果仍以列表形式呈现
