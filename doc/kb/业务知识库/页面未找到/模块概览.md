# 页面未找到 — 模块概览

## 1. 模块业务概述

"页面未找到"模块对应 FastGPT 的 404 错误页面（路由 `/404`），是 Next.js 框架在用户访问不存在的路由时自动呈现的兜底页面。

本模块的核心行为是**静默重定向**：用户访问任何无效路径时，页面组件挂载后立即将用户导航至应用工作台首页（`/dashboard/agent`），不展示任何 404 错误信息或"页面不存在"提示。这一设计将无效路径访问统一引导至应用主入口，确保用户始终能够回到可用页面。

本模块为顶层叶子节点，无子能力，不涉及角色区分。

## 2. 角色差异

本模块无角色差异。页面组件不包含任何权限校验逻辑，所有访问者（含未登录用户）均执行相同的重定向行为。

## 3. 跨模块依赖

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| Next.js 路由系统 | `next/router` 的 `useRouter` Hook，执行客户端路由跳转 | Hook |
| React | `useEffect` Hook，处理组件挂载时的副作用 | Hook |
| i18n 工具 | `@/web/common/i18n/utils` 的 `serviceSideProps`（已导入但未在组件内使用，预留给 SSR 场景） | 工具函数 |

## 4. 对外共享

本模块无对外共享能力。页面组件 `NonePage` 仅在 `/404` 路由挂载，不被其他模块引用。
