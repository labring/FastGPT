# 应用 — 模块概览

## 1. 模块业务概述

应用模块是 FastGPT 的核心业务单元，负责 AI 应用的创建、配置、发布与管理。FastGPT 中的"应用"代表一个完整的 AI Agent 实例，支持多种应用类型：简易应用（对话型）、Chat Agent（带知识库和工作流的智能体）、工作流、插件工具、MCP 工具集、HTTP 工具集以及智能客服（Assistant）。

应用模块为不同角色提供差异化的视图：管理员可完整编辑和发布应用，协作者可在授权范围内参与配置，只读用户仅能查看对话日志。应用详情页根据应用类型动态加载对应的编辑器组件，实现从简单问答到复杂工作流编排的全覆盖。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| 管理员 (Owner) | 应用详情、编辑器、工作流、日志看板、对话记录 | 编辑应用配置、发布应用、删除应用、查看全部日志、管理协作成员 | 拥有完整管理权限，可执行所有操作 |
| 协作者 (Collaborator) | 应用详情、编辑器、工作流（视权限） | 编辑配置、发布（需写权限）、查看日志 | 读写权限由管理员分配控制 |
| 只读用户 (Viewer) | 应用详情（自动跳转日志页） | 仅查看日志 | 无编辑权限，进入详情页自动重定向至日志 Tab |

**权限模型**: 基于应用级别的 RBAC 权限控制（`permission.hasWritePer` / `hasReadChatLogPer`），页面加载时从 `/core/app/detail` API 返回权限信息，上下文通过 `AppContext` 分发。

## 3. 跨模块依赖摘要

| 依赖模块 | 使用的具体能力 | 能力类型 |
|---------|---------------|---------|
| 工作台 | 应用创建入口、应用列表管理 | 页面/API |
| 对话 | 应用对话测试（ChatTest 组件）、聊天 Store | 组件/Store |
| 数据集 | 知识库关联选择、快速创建知识库弹窗 | 组件 |
| 技能 | 技能选择器、技能关联配置 | 组件 |
| 配置 | 工具配置与工具选择 | 组件 |

| 应用 API 层（getMyApps、getAppDetailById） | 工作台、对话、技能 | API |
