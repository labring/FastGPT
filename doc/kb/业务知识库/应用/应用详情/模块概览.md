# 应用详情 — 模块概览

## 1. 模块业务概述

应用详情模块是 FastGPT 平台的核心编辑器入口，为所有类型的 AI 应用提供统一的编辑、调试、发布和监控能力。用户从工作台应用列表点击任意应用后进入此模块，根据应用类型自动加载对应的编辑器界面。

本模块支持七种应用类型的编辑：

- **简单应用**：基于表单配置的对话应用，适合快速搭建
- **对话 Agent**：基于 Agent 模式的多步骤对话应用，支持技能编排
- **工作流**：基于可视化画布的工作流编辑器（ReactFlow），支持复杂的节点编排
- **插件（工作流工具）**：将工作流封装为可复用的工具插件
- **MCP 工具集**：通过 MCP 协议连接外部工具服务
- **HTTP 工具集**：通过 HTTP API 定义和调用外部接口
- **智能客服**：基于知识库的智能客服助手

每种编辑器内部通过统一的 Tab 系统（编辑/看板/日志/发布）切换不同功能视图。

## 2. 角色差异

| 角色 | 可见页面 | 可执行操作 | 差异说明 |
|------|---------|-----------|---------|
| Owner | 编辑/看板/日志/发布 | 全部操作 | 拥有完整权限，额外会在保存时收到模型引用有效性校验警告 |
| 管理员(manage) | 编辑/看板/日志/发布 | 全部操作 | 与 Owner 权限相同，但不触发模型引用警告 |
| 编辑者(write) | 编辑/日志/发布 | 查看看板以外的全部操作 | 无法查看数据看板（Dashboard Tab 不可见） |
| 只读者(read) | 日志 | 仅查看日志 | 进入页面即自动跳转到日志 Tab，编辑和发布 Tab 不可见 |
| 日志查看者(readChatLog) | 看板/日志 | 查看看板和日志 | 可查看数据看板，但无法编辑或发布 |

**权限模型**：基于位掩码（bitmask）的 RBAC 权限体系。应用权限继承自 `Permission` 基类并扩展 `hasReadChatLogPer`（位值 0b1000）。角色权限映射：读=read(0b0001)，写=read+write(0b0011)，管理=read+write+manage+readChatLog(0b1111)，日志查看=read+readChatLog(0b1001)。

权限判断逻辑入口：`projects/app/src/pageComponents/app/detail/context.tsx` 和 `RouteTab.tsx`。
