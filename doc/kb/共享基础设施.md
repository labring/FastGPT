# 共享基础设施

## 基础设施/共享模块

> 跨模块共享的底层代码。修改这些目录中的代码可能影响多个业务功能模块，需评估变更范围。

| 目录 | 类型 | 定位说明 |
|------|------|---------|
| `packages/global/` | 共享包 | 全局类型定义、常量、工具函数、核心业务领域模型。包含 `common/`（通用工具：错误处理、文件操作、国际化、数学计算、限流等）、`core/`（核心业务领域类型：AI、聊天、数据集、评测、工作流等）、`sdk/`（SDK 封装）、`support/`（支撑功能类型：权限、用户、钱包等）、`openapi/`（API 契约定义） |
| `packages/service/` | 共享包 | 后端服务实现层，包含数据库模型、API 控制器、工作流引擎。`common/`（通用服务：API 包装、缓存、文件、日志、指标等）、`core/`（核心业务逻辑实现）、`support/`（支撑功能实现）、`worker/`（后台任务处理：文本分块、文件读取等）、`thirdProvider/`（第三方服务集成：代码沙箱、文档解析等） |
| `packages/web/` | 共享包 | 共享前端组件、Hooks、状态管理、样式主题、国际化。`components/`（公共 UI 组件：Avatar、Markdown 渲染、日期选择器等）、`hooks/`（通用 Hooks：确认弹窗、复制、无限滚动、分页等）、`store/`（全局 Zustand Store）、`context/`（React Context 定义）、`styles/`（Chakra UI 主题配置）、`i18n/`（国际化词条，支持中文/英文/日文） |
| `packages/templates/` | 共享包 | 模板定义，为各业务模块提供标准化模板 |
| `packages/diting-rag-ts/` | 共享包 | RAG 工具包，提供检索增强生成相关能力 |
| `plugins/` | 插件目录 | 外部插件扩展，包含模型提供商适配（`model/`）、网页爬虫（`webcrawler/`）等 |
| `deploy/` | 部署配置 | Docker 镜像构建、Helm Chart、K8s 部署模板，为所有 services 提供统一的部署方案 |
| `projects/code-sandbox/` | 共享服务 | 代码执行沙箱服务（Bun + Hono），为应用提供安全的代码运行环境 |
| `projects/mcp_server/` | 共享服务 | MCP（Model Context Protocol）服务器，为 AI Agent 提供标准化工具接入 |
| `projects/agent-sandbox/` | 共享服务 | Agent 沙箱环境，提供隔离的 Agent 执行上下文 |

## 使用规范

- **基础设施/共享模块**：修改需评估全局影响，涉及公共组件/Hooks/工具函数变更应在团队内同步；建议通过 Changeset 或 PR 描述中明确标注影响范围
- **packages 层级依赖**：`global` → `service` / `web`（`global` 为最底层共享类型，`service` 和 `web` 分别依赖 `global`，不应反向依赖）
- **跨业务功能模块的共享能力**通过 `packages/` 下的共享包实现，避免在 `projects/app/` 内跨页面直接引用
- **插件扩展**通过 `plugins/` 目录接入，遵循统一的 Provider 接口规范
- **部署配置**集中于 `deploy/`，新增服务需同步更新部署模板
- **第三方服务集成**统一通过 `packages/service/thirdProvider/` 封装，业务代码不直接调用外部 SDK
