## 词条 key 命名规范

### 命名模式

项目采用 **扁平 key-value** 结构，key 使用**点分隔层级命名**，通过命名空间前缀区分模块：

```
{namespace}:{section}.{sub_section}.{label}
```

### 三种命名模式

**模式 A：扁平单词（通用词条）**

用于通用 UI 元素，key 为单个英文单词或短语：

| Key | zh-CN | en |
|-----|-------|-----|
| `common:Delete` | 删除 | Delete |
| `common:Save` | 保存 | Save |
| `common:Confirm` | 确认 | OK |
| `common:Cancel` | 取消 | Cancel |
| `common:Loading` | 加载中... | Loading... |

**模式 B：模块前缀 + 路径层级（业务词条）**

用于模块内业务相关的词条，以 `core.{module}.{section}.{label}` 层级组织：

| Key | zh-CN |
|-----|-------|
| `common:core.app.Name and avatar` | 头像 & 名称 |
| `common:core.app.Share link` | 免登录窗口 |
| `common:core.app.error.App name can not be empty` | 应用名不能为空 |
| `common:core.dataset.Delete Confirm` | 确认删除该知识库？ |
| `common:core.chat.response.module model` | 模型 |

**模式 C：子命名空间模式**

特定功能模块使用独立命名空间，key 为描述性英文文本：

| Key | 命名空间 | zh-CN |
|-----|---------|-------|
| `workflow:Node.Open_Node_Course` | workflow | 查看节点教程 |
| `app:Tool_name` | app | 工具名称 |
| `app:app.error.unExist_app` | app | 部分组件缺失，请删除 |
| `chat:core.chat.New Chat` | chat | 新建对话 |

### 插值变量

支持 `{{variable}}` 双花括号插值语法：

```
common:chat_chatId                 → "会话Id: {{chatId}}"
common:core.dataset.Create dataset → "创建一个{{name}}"
app:HTTP_tools_list_with_number    → "工具列表: {{total}}"
error.upload_file_error_filename   → "{{name}} 上传失败"
```

### 语言文件物理结构

每个命名空间一个 JSON 文件，采用扁平结构（无深层嵌套对象）：

```json
{
  "Delete": "删除",
  "Save": "保存",
  "core.app.Share link": "免登录窗口",
  "core.dataset.Delete Confirm": "确认删除该知识库？",
  "error.fileNotFound": "文件找不到了~"
}
```

## 词条使用方式

项目存在**三套翻译机制**，分别服务于不同的运行环境。

### 客户端组件中：`useTranslation()` + `t()`

React 组件中使用 next-i18next 的 `useTranslation` hook，通过 `t('namespace:key')` 调用：

```tsx
import { useTranslation } from 'next-i18next';

const MyComponent = () => {
  const { t, i18n } = useTranslation();

  return (
    <Button>{t('common:Delete')}</Button>
    <span>{t('account_bill:bill_record')}</span>
  );
};
```

### 全局常量/工具函数中：`i18nT()`

在 `packages/global/` 和 `packages/service/` 等非 React 环境中，使用恒等函数 `i18nT()`。该函数返回 key 本身，实际翻译由客户端 `t()` 完成：

```typescript
import { i18nT } from '@fastgpt/global/common/i18n/utils';

// 工作流节点定义的 name/intro/label 字段
const nodeConfig = {
  name: i18nT('workflow:customer_service.enable_fallback_reply_node_name'),
  intro: i18nT('workflow:execute_different_branches_based_on_conditions'),
  label: i18nT('workflow:judgment_result'),
};
```

### 服务端：`parseI18nString()`

用于解析 `I18nStringType` 对象（包含三语种的结构化翻译），在服务端按当前语言提取对应文本：

```typescript
import { parseI18nString, I18nStringType } from '@fastgpt/global/common/i18n/utils';

const SANDBOX_NAME: I18nStringType = {
  'zh-CN': '虚拟机',
  'zh-Hant': '虛擬機',
  en: 'Sandbox'
};

// 服务端消费
const displayName = parseI18nString(SANDBOX_NAME, lang);
// lang='zh-CN' → "虚拟机", lang='en' → "Sandbox"
```

**回退策略**：繁体中文缺失时回退简体中文，再缺失回退英文。

### 翻译函数对比

| 函数 | 运行环境 | 输入 | 输出 |
|------|---------|------|------|
| `t('ns:key')` | React 客户端 | namespace:key | 当前语言的翻译文本 |
| `i18nT('ns:key')` | 全局常量/Node | namespace:key | key 自身（恒等） |
| `parseI18nString(obj, lang)` | 服务端/全局 | I18nStringType + 语言 | 指定语言的文本 |

## 新增/修改词条流程

1. **在对应模块的语言文件中添加 key-value**
   - 确定词条所属命名空间（如 `common`、`app`、`workflow`）
   - 在 `packages/web/i18n/zh-CN/{namespace}.json` 中添加键值对
   - key 遵循命名规范：用点分隔的层级结构

2. **同步到所有语言文件**
   - 在 `packages/web/i18n/en/{namespace}.json` 中添加英文翻译
   - 在 `packages/web/i18n/zh-Hant/{namespace}.json` 中添加繁体中文翻译
   - 三语种必须同步 key 名称，值可为空字符串（运行时回退处理）

3. **在组件中使用**
   - 客户端组件：`t('namespace:key')`
   - 全局常量：`i18nT('namespace:key')`
   - 服务端结构化文本：定义 `I18nStringType` 对象，用 `parseI18nString()` 解析

4. **Fallback 机制**
   - 运行时词条缺失时，next-i18next 回退到默认语言 zh-CN 的对应值
   - `parseI18nString` 的 fallback 链：指定语言 → zh-Hant 回退 zh-CN → en

## 多语言切换

| 机制 | 实现 |
|------|------|
| 触发方式 | `I18nLngSelector` 语言下拉选择器组件，嵌入登录页、个性化设置等多处 |
| 存储位置 | Cookie `NEXT_LOCALE`（30 天过期）+ localStorage `NEXT_LOCALE` |
| 是否需刷新 | 切换到中文（简/繁）需刷新页面；切换到英文支持热切换，资源未加载时回退刷新 |
| 组件库联动 | Chakra UI 通过 i18next 的 `changeLanguage` 联动 |
| 后端同步 | 登录用户切换语言后通过 `updateUserInfo({ language })` 同步到后端数据库 |

### 语言切换流程

1. 用户通过 `I18nLngSelector` 选择目标语言
2. `useI18nLng().onChangeLng(lng)` 被调用
3. 语言值写入 Cookie（`NEXT_LOCALE`，30 天过期）和 localStorage
4. 已登录用户同步调用 `updateUserInfo({ language: lng })` 持久化到后端
5. 切换逻辑：
   - 切换到中文（zh-CN / zh-Hant）→ `window.location.reload()` 让 Next.js SSR 重新渲染
   - 切换到英文 → `i18n.changeLanguage(lang)` 热切换，资源包未加载时 fallback 刷新
6. SSR 页面通过 `serviceSideProps()` 从 Cookie 读取语言，在 `getServerSideProps` 中加载对应命名空间翻译

### 首次访问语言判定

通过 `useI18nLng().setUserDefaultLng()` 初始化：
1. 优先读取 localStorage/Cookie 中的 `NEXT_LOCALE`
2. 无存储值时，取浏览器语言 `navigator.language`，映射到支持的语言
3. 都不可用时默认 `zh-CN`
