## 一、项目约定

### 1.1 包管理器

本项目使用 **pnpm workspaces** 管理依赖。

```bash
# 安装依赖（在根目录执行）
pnpm install

# 仅为特定 workspace 安装依赖
pnpm --filter @fastgpt/service add <package>

# 运行脚本
pnpm lint          # 对所有 TS 文件运行 ESLint 并自动修复
pnpm test          # 运行所有测试
pnpm test <path>   # 运行指定测试文件
```

环境要求：
- Node.js >= 20.x
- pnpm >= 9.x

### 1.2 TypeScript 严格模式

全项目启用 TypeScript 严格模式，须遵循以下规范：

**使用 `type` 而非 `interface`：**

```typescript
// ❌ 不好的实践
interface User {
  id: string;
  name: string;
}

// ✅ 好的实践
type User = {
  id: string;
  name: string;
}
```

**用 Zod schema 同时承担校验和类型推导：**

```typescript
// ❌ 不好的实践：类型与 schema 重复定义
type MessageParam = { role: 'user' | 'assistant'; content: string };
const MessageParamSchema = z.object({ role: z.enum(['user', 'assistant']), content: z.string() });

// ✅ 好的实践：从 schema 推导类型
export const MessageParamSchema = z.object({
  role: z.enum(['user', 'assistant']),
  content: z.string()
});
export type MessageParam = z.infer<typeof MessageParamSchema>;
```

### 1.3 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名（组件） | PascalCase | `ChatBox.tsx`, `AccordionSection.tsx` |
| 文件名（工具/模块） | camelCase | `controller.ts`, `schema.ts`, `utils.ts` |
| 文件名（API 路由） | camelCase | `create.ts`, `detail.ts`, `delHistory.ts` |
| 变量/函数 | camelCase | `appId`, `createApp`, `getUserInfo` |
| React 组件 | PascalCase | `const ChatBox: React.FC<Props> = ...` |
| 类型/接口 | PascalCase + Type/Schema 后缀 | `AppSchemaType`, `CreateAppBodySchema` |
| 常量/枚举 | PascalCase（枚举成员）或 UPPER_SNAKE（全局常量） | `AppTypeEnum.workflow`, `AppCollectionName` |
| Mongoose Model | `Mongo` + PascalCase | `MongoApp`, `MongoChat` |

### 1.4 代码风格要点

**IIFE 取代 if/else 赋值：**

```typescript
// ❌ 不好的实践
if (condition) {
  value = true;
} else {
  value = false;
}

// ✅ 好的实践
const value = (() => {
  if (condition) return true;
  return false;
})();
```

**可选链调用回调：**

```typescript
// ❌ 不好的实践
if (onProgress) onProgress({ phase: 'done' });

// ✅ 好的实践
onProgress?.({ phase: 'done' });
```

**空值合并取默认值：**

```typescript
// ❌ 不好的实践（0、false、'' 会被错误覆盖）
const version = lastVersion?.version || 0;

// ✅ 好的实践
const version = (lastVersion?.version ?? -1) + 1;
```

**函数参数不超过 2 个，多参数用对象：**

```typescript
// ❌ 不好的实践
function createVersion(skillId: string, teamId: string, tmbId: string, version: number) {}

// ✅ 好的实践
function createVersion(data: {
  skillId: string;
  teamId: string;
  tmbId: string;
  version: number;
}) {}
```

**非关键清理用 `.catch()` 链：**

```typescript
// 次要操作不污染主流程
await client.delete().catch(() => {});
```

## 二、代码组织规范

采用 **DDD 架构**，按「业务域 → 子功能 → 固定文件」三层划分。

### 2.1 目录结构约定

```
packages/
├── global/core/              # 类型、常量（前后端共享）
│   └── app/
│       ├── type.ts           # 顶层聚合类型
│       ├── constants.ts      # 枚举常量
│       └── workflow/
│           ├── type.ts
│           └── constants.ts
│
└── service/core/             # 后端业务逻辑（禁止前端引用）
    └── app/
        ├── schema.ts         # Mongoose Schema + Model 导出
        ├── controller.ts     # 数据访问封装 + 业务逻辑（findById / create / updateById 等）
        ├── utils.ts          # 纯函数工具，无副作用
        └── version/
            ├── schema.ts
            └── controller.ts
```

**叶子目录固定文件职责：**

| 文件 | 职责 |
|------|------|
| `schema.ts` | Mongoose Schema 定义，导出 Model 和 SchemaType |
| `controller.ts` | 数据访问封装（findById、create、updateById 等）+ 业务逻辑（跨模块协调、业务规则）|
| `utils.ts` | 纯函数工具，无副作用，可独立单测 |

**层级约束：**
- `global/core/` 只放类型和常量，**禁止**引入 mongoose、服务端 SDK
- `service/core/` 只在服务端使用，**禁止**被 `packages/web/` 或前端页面直接引用
- 子功能目录不超过 **3 层**嵌套
- 多个 schema 文件**合并**到单个 `schema.ts`，不拆分

### 2.2 API 路由编写规范

参考设计规范：`FastGPT/.claude/design/api/index.md`

**核心原则：**
1. 所有 API 必须使用 Zod schema 定义入参和出参
2. Schema 定义在 `packages/global/openapi/` 中，类型从 schema 导出
3. API 路由中使用 `schema.parse()` 验证入参和出参
4. 路由文件不导出类型别名（消费者直接从 openapi 包导入）
5. 必须在 OpenAPI index 中注册路由

**步骤 1：在 `packages/global/openapi/` 定义 Zod Schema**

```typescript
// packages/global/openapi/core/app/common/api.ts
import { z } from 'zod';
import { ObjectIdSchema } from '../../../common/type/mongo';
import { ParentIdSchema } from '../../../common/parentFolder/type';

/* ============================================================================
 * API: 创建 App
 * Route: POST /api/core/app/create
 * ============================================================================ */
export const CreateAppBodySchema = z.object({
  parentId: ParentIdSchema.meta({ description: '父级文件夹 ID' }),
  name: z.string().meta({ example: '我的应用', description: '应用名称' }),
  type: z.nativeEnum(AppTypeEnum).meta({ description: '应用类型' })
});
export type CreateAppBodyType = z.infer<typeof CreateAppBodySchema>;

export const CreateAppResponseSchema = ObjectIdSchema.meta({
  description: '新创建的 App ID'
});
export type CreateAppResponseType = z.infer<typeof CreateAppResponseSchema>;
```

**步骤 2：在 `projects/app/src/pages/api/` 实现路由**

```typescript
// projects/app/src/pages/api/core/app/create.ts
import { NextAPI } from '@/service/middleware/entry';
import type { ApiRequestProps } from '@fastgpt/service/type/next';
import {
  CreateAppBodySchema,
  CreateAppResponseSchema,
  type CreateAppBodyType
} from '@fastgpt/global/openapi/core/app/common/api';

async function handler(req: ApiRequestProps<CreateAppBodyType>) {
  // 1. 入参验证（使用 safeParseAsync 或 parse）
  const parseResult = await CreateAppBodySchema.safeParseAsync(req.body);
  const body = parseResult.success ? parseResult.data : req.body;
  const { parentId, name, type } = body;

  // 2. 权限校验
  const { teamId, tmbId } = await authUserPer({ req, authToken: true, per: ... });

  // 3. 业务逻辑
  const appId = await onCreateApp({ parentId, name, type, teamId, tmbId });

  // 4. 出参验证
  return CreateAppResponseSchema.parse(appId);
}

export default NextAPI(handler);
```

**关键点：**
- 使用 `NextAPI` 包裹 handler，提供统一的错误处理、日志、跨域
- `ApiRequestProps<BodyType>` 带泛型以获得类型推断
- 使用 `authApp` 或 `authUserPer` 进行权限校验
- `mongoSessionRun` 用于需要事务的操作
- 出参必须用 `ResponseSchema.parse(result)` 验证

**公共 Schema 参考：**

| Schema | 说明 | 导入路径 |
|--------|------|----------|
| `ObjectIdSchema` | MongoDB ObjectId（自动转 string） | `@fastgpt/global/common/type/mongo` |
| `ParentIdSchema` | 父级 ID（`z.string().nullish()`） | `@fastgpt/global/common/parentFolder/type` |
| `PaginationSchema` | 偏移分页（pageSize、offset、pageNum） | `@fastgpt/global/openapi/api` |
| `PaginationResponseSchema<T>` | 分页响应（total、list） | `@fastgpt/global/openapi/api` |

> **ObjectId 陷阱**：MongoDB 返回的 `_id` 是 ObjectId 对象，不是 string。出参不能用 `z.string()` 验证，必须使用 `ObjectIdSchema`（内置自动转换）。

### 2.3 Mongoose 数据模型编写规范

参考示例：`FastGPT/packages/service/core/app/schema.ts`

```typescript
import { AppTypeEnum } from '@fastgpt/global/core/app/constants';
import { Schema, getMongoModel } from '../../common/mongo';
import type { AppSchemaType as AppType } from '@fastgpt/global/core/app/type';
import { TeamCollectionName } from '@fastgpt/global/support/user/team/constant';

// 1. 定义集合名称常量
export const AppCollectionName = 'apps';

// 2. 定义 Schema
const AppSchema = new Schema(
  {
    teamId: {
      type: Schema.Types.ObjectId,
      ref: TeamCollectionName,
      required: true          // 必填字段加 required
    },
    name: {
      type: String,
      required: true
    },
    type: {
      type: String,
      default: AppTypeEnum.workflow,
      enum: Object.values(AppTypeEnum)  // 枚举限制
    },
    intro: {
      type: String,
      default: ''             // 字符串字段给空字符串默认值
    },
    modules: {
      type: Array,
      default: []
    },
    deleteTime: {
      type: Date,
      default: null           // null 表示未删除（软删除模式）
    }
  },
  {
    minimize: false           // 保留空对象字段
  }
);

// 3. 定义索引（高频查询字段必须建索引）
AppSchema.index({ teamId: 1, updateTime: -1 });
AppSchema.index({ teamId: 1, type: 1 });
// 部分索引（条件索引）
AppSchema.index(
  { scheduledTriggerConfig: 1, scheduledTriggerNextTime: -1 },
  {
    partialFilterExpression: {
      scheduledTriggerConfig: { $exists: true }
    }
  }
);

// 4. 导出 Model（泛型参数为 global 中定义的 TypeScript 类型）
export const MongoApp = getMongoModel<AppType>(AppCollectionName, AppSchema);
```

**Schema 约定：**
- 集合名称用 `export const XxxCollectionName = 'xxx'` 导出
- 关联字段使用 `Schema.Types.ObjectId` + `ref` 指向集合名称常量
- 高频查询字段必须建索引
- 软删除使用 `deleteTime: { type: Date, default: null }` 模式
- 导出 Model 时，TypeScript 类型定义在 `packages/global/` 中

### 2.4 Controller 编写规范

参考示例：`FastGPT/packages/service/core/app/controller.ts`

**controller.ts（数据访问 + 业务逻辑，统一放在一个文件中）：**

```typescript
// packages/service/core/app/controller.ts
export const findAppById = (id: string) =>
  MongoApp.findById(id).lean();

export const createApp = (data: AppCreateParams, session?: ClientSession) =>
  MongoApp.create([data], { session });

export const getAppBasicInfoByIds = async ({
  teamId,
  ids
}: {
  teamId: string;
  ids: string[];
}) => {
  const apps = await MongoApp.find(
    { teamId, _id: { $in: ids } },
    '_id name avatar'
  ).lean();

  return apps.map((item) => ({
    id: item._id,
    name: item.name,
    avatar: item.avatar
  }));
};
```

**跨模块协调示例（在 controller.ts 中实现）：**

```typescript
// packages/service/core/app/controller.ts
import { mongoSessionRun } from '../../common/mongo/sessionRun';
import { MongoAppVersion } from './version/schema';

export const createAppAndInitVersion = async (
  data: AppCreateParams,
  session?: ClientSession
) => {
  const create = async (session: ClientSession) => {
    const [app] = await MongoApp.create([data], { session });
    await MongoAppVersion.create([{ appId: String(app._id), ... }], { session });
    return app;
  };

  if (session) return create(session);
  return mongoSessionRun(create);  // 无 session 时自动管理事务
};
```

### 2.5 React 组件编写规范

参考示例：`FastGPT/projects/app/src/components/core/app/AccordionSection.tsx`

**基本组件结构：**

```typescript
import React from 'react';
import { Box, Flex, Button } from '@chakra-ui/react';
import { useTranslation } from 'next-i18next';

// Props 类型定义（使用 type，不用 interface）
type MyComponentProps = {
  title: string;
  children: React.ReactNode;
  onConfirm?: () => void;     // 可选回调用 ?
  disabled?: boolean;
};

// 函数式组件（使用 React.FC 泛型）
const MyComponent: React.FC<MyComponentProps> = ({
  title,
  children,
  onConfirm,
  disabled = false

      </Flex>
```

**Chakra UI 使用约定：**

| 场景 | 方案 |

| 跨组件共享（局部树） | `React Context`（见 `context/` 目录） |
| 全局应用状态 | `Zustand` store |
| 服务端数据缓存 | `SWR` + API 请求 |

## 三、国际化（i18n）规范

参考规范：`FastGPT/.claude/skills/i18n/SKILL.md`

词条文件位于 `FastGPT/packages/web/i18n/`，支持三种语言：

| 语种 | 目录 |
|------|------|
| 简体中文 | `packages/web/i18n/zh-CN/` |
| 英文 | `packages/web/i18n/en/` |
| 繁体中文 | `packages/web/i18n/zh-Hant/` |

### 3.1 基本用法

```typescript
import { useTranslation } from 'next-i18next';

const { t } = useTranslation();

// t() 调用必须使用 namespace:key 格式
t('app:learn')
t('common:Confirm')
t('dataset:create_dataset')

// 带变量插值
t('file:selected_file_count', { count: 3 })
// 对应词条：{ "selected_file_count": "已选择 {{count}} 个文件" }
```

### 3.2 命名空间约定

常用命名空间（共 28 个，完整列表见源码）：

| 命名空间 | 用途 |
|----------|------|
| `common` | 通用词条（操作、状态等） |
| `app` | 应用相关 |
| `dataset` | 知识库相关 |
| `workflow` | 工作流相关 |
| `chat` | 聊天相关 |

> 完整 28 个命名空间定义见：`FastGPT/packages/web/i18n/constants.ts`

### 3.3 新增词条规则

新增词条时，必须**同时**在三个语种文件添加：

```json
// packages/web/i18n/zh-CN/app.json
{ "learn": "学习" }

// packages/web/i18n/en/app.json
{ "learn": "learn" }

// packages/web/i18n/zh-Hant/app.json
{ "learn": "學習" }   // 必须是真正的繁体中文，不能复制简体
```

**key 命名规范：**
- 使用英文单词，`PascalCase` 或 `snake_case`（与文件已有风格一致）
- 支持点号层级：`setting.basic_config`
- 禁止直接使用中文作为 key

### 3.4 检查清单

- [ ] `t()` 使用了 `namespace:key` 格式
- [ ] key 为英文且语义化
- [ ] 三个语种文件同步添加
- [ ] `zh-Hant` 使用正确的繁体中文
- [ ] 变量插值使用 `{{variable}}` 语法

## 四、配置文件结构

配置文件：`FastGPT/projects/app/data/config.json`（使用 JSON5 格式，支持注释）

主要配置节点：

```jsonc
{
  "feConfigs": {
    // 前端功能开关和外部服务地址
    "lafEnv": "https://laf.dev",
    "mcpServerProxyEndpoint": "http://localhost:3003"
  },
  "systemEnv": {
    // 后端运行参数
    "datasetParseMaxProcess": 10,   // 知识库文件解析最大线程数
    "vectorMaxProcess": 10,          // 向量处理线程数
    "qaMaxProcess": 10,              // 问答拆分线程数
    "tokenWorkers": 30               // Token 计算线程数
  }
}
```

**如何添加新配置项：**

1. 在 `FastGPT/packages/global/common/system/types/index.ts` 中扩展 `SystemEnvType`（服务端配置）或 `FastGPTFeConfigsType`（前端配置）类型
2. 在 `FastGPT/projects/app/data/config.json` 添加默认值（带注释说明用途）
3. 在业务代码中通过以下方式读取配置：
   - **服务端**：环境变量通过 `packages/service/env.ts` 中的 `env` 对象读取（基于 `@t3-oss/env-core`，自动校验类型）；`config.json` 中的 `systemEnv`/`feConfigs` 通过 `global.systemEnv` / `global.feConfigs` 全局变量读取
   - **前端**：通过 `useSystemStore()` 获取 `feConfigs`，通过 `useInitApp()` 初始化应用级配置

### 6.2 包内路径规则

```
@fastgpt/global/
```

## 七、错误处理规范

### 7.1 统一错误码

错误码定义：`FastGPT/packages/global/common/error/errorCode.ts`

```typescript
import { ERROR_ENUM, ERROR_RESPONSE } from '@fastgpt/global/common/error/errorCode';

// 通用错误枚举
export enum ERROR_ENUM {
  unAuthorization = 'unAuthorization',       // 未授权（403）
  insufficientQuota = 'insufficientQuota',   // 额度不足（510）
  unAuthModel = 'unAuthModel',               // 模型无权限（511）
  tooManyRequest = 'tooManyRequest'          // 请求过多（429）
}
```

各模块专属错误码（共 16 个文件，见 `FastGPT/packages/global/common/error/code/`）：
- `FastGPT/packages/global/common/error/code/app.ts` — App 相关
- `FastGPT/packages/global/common/error/code/chat.ts` — 聊天相关
- `FastGPT/packages/global/common/error/code/dataset.ts` — 知识库相关
- `FastGPT/packages/global/common/error/code/common.ts` — 通用
- 其他还包括：`agentSkill.ts`、`database.ts`、`evaluation.ts`、`openapi.ts`、`outLink.ts`、`plugin.ts`、`s3.ts`、`skill.ts`、`system.ts`、`team.ts`、`train.ts`、`user.ts`

### 7.2 服务端抛出错误

```typescript
import { CommonErrEnum } from '@fastgpt/global/common/error/code/common';

// 1. 使用错误枚举抛出（推荐）
return Promise.reject(CommonErrEnum.missingParams);

// 2. 直接抛出字符串（仅在无对应枚举时）
return Promise.reject('App not found');

// 3. 抛出用户可见错误
import { UserError } from '@fastgpt/global/common/error/utils';
throw new UserError('操作失败，请稍后重试');
```

`NextAPI` 中间件会自动捕获这些错误，将 `ERROR_RESPONSE` 中定义的错误码映射到正确的 HTTP 状态码和客户端消息。

### 7.3 客户端错误处理

```typescript
import { getErrText } from '@fastgpt/global/common/error/utils';
import { useToast } from '@fastgpt/web/hooks/useToast';

const { toast } = useToast();

try {
  await createApp(params);
} catch (err) {
  // getErrText 自动处理：Axios 响应、Error 对象、字符串等多种格式
  toast({
    title: getErrText(err, '创建失败'),
    status: 'error'
  });
}
```

### 7.4 错误响应结构

API 统一返回格式（由 `packages/service/common/response/index.ts` 处理）：

```typescript
// 成功响应
{
  code: 200,
  message: '',
  data: T
}

// 错误响应
{
  code: 403,          // 业务错误码
  message: '未授权',  // 用户可读消息（经过 i18n 处理）
  data: null
}
```

```typescript
import { getLogger, LogCategories } from '@fastgpt/service/common/logger';

const logger = getLogger(LogCategories.MODULE.APP.FOLDER);

// 记录不同级别日志
logger.debug('调试信息', { appId, parentId });
```

## 十、快速参考

### 10.1 新增一个 API 的完整流程

1. **定义 Schema**：在 `packages/global/openapi/<module>/api.ts` 添加入参/出参 Schema
2. **注册 OpenAPI**：在对应 `index.ts` 中注册路由文档
3. **实现路由**：在 `projects/app/src/pages/api/<path>/<route>.ts` 实现 handler
4. **前端调用**：从 `@fastgpt/global/openapi/` 导入类型，调用 API
5. **编写测试**：在对应 `test/` 目录下添加测试用例

### 10.2 新增一个数据模型的完整流程

1. **定义类型**：在 `packages/global/core/<module>/type.ts` 添加 TypeScript 类型
2. **定义常量**：在 `packages/global/core/<module>/constants.ts` 添加枚举
3. **定义 Schema**：在 `packages/service/core/<module>/schema.ts` 添加 Mongoose Schema
4. **实现 controller**：在 `FastGPT/packages/service/core/<module>/controller.ts` 添加数据访问封装（findById、create、updateById 等）和跨模块协调的业务逻辑
5. **实现 utils**：在 `FastGPT/packages/service/core/<module>/utils.ts` 添加纯函数工具（无副作用，可独立单测）

## 十一、近期重要变更（基于 develop-1.3.0 git 历史）

> 此章节归纳本仓库最近 3 个月内对编码规范和模块边界产生影响的核心变更，新代码请优先沿用新模式。

### 11.1 日志：从 addLog 全面迁移到 LogTape Logger

近期一批 commit（如 `84d176c9d [UPDATE] 修改打印较多的日志到新logger`、`08560ed27 feat: remove debug log and reranker add error log`）将分散在 service 层的 `addLog`/`console.*` 调用统一替换为 `getLogger(LogCategories.X)` 形态。

**编码要求：**

- 新代码不要再使用 `addLog.info/warn/error`，请直接 `import { getLogger, LogCategories } from '@fastgpt/service/common/logger'`，在模块顶层创建 logger。
- 分类参考 `FastGPT/packages/service/common/logger/categories.ts`，常用新增分类：
  - `LogCategories.INFRA.REDIS` — Redis 连接与队列
  - `LogCategories.INFRA.QUEUE` — `MongoDatasetTraining` 等队列消费 Worker
  - `LogCategories.MODULE.AI.AGENT` / `LogCategories.MODULE.AI.LLM` / `LogCategories.MODULE.AI.EMBEDDING`
- "次要清理"日志（不影响主流程）一律用 `.catch((err) => logger.warn(...))`，**不要**继续吞掉错误。

### 11.2 Agent 引擎：新增 piAgent 与 context compaction

- 新文件：`FastGPT/packages/service/core/workflow/dispatch/ai/agent/piAgent/index.ts`、`piAgent/toolAdapter.ts`、`piAgent/modelBridge.ts`、`piAgent/compaction.ts`。
- 通过环境变量 `AGENT_ENGINE=pi` 切换；默认引擎仍是 master/plan 双层编排。
- piAgent 支持 `useAgentSandbox + SANDBOX_TOOLS` 直接调用沙箱工具集，避免重复构造工具描述（见 `12501eb98 feat: piAgent support useAgentSandbox and SANDBOX_TOOLS`）。
- 新增 **context compaction** 能力（`ec51c573a feat(piAgent): add context compaction with env toggle`），通过环境变量开关，由 piAgent 自行调用 LLM 压缩历史并复用 dataset params resolution。
- 编写 piAgent 工具适配器时，复用 `toolAdapter.ts` 中的转换函数；不要在 piAgent 内联工具描述。
- 系统提示词通过 `parsePiAgentSystemPrompt` 函数解析（导入自 `../sub/plan/prompt`），piAgent 目录下**不存在** `systemPrompt.ts` 文件。
