# 系统工具 — 组件列表

## 页面组件

| 组件路径 | 用途 | 跨模块依赖 | 是否共享 | Props 摘要 |
|---------|------|-----------|---------|-----------|
| `projects/app/src/pages/dashboard/systemTool/index.tsx` | 系统工具市场页面入口 | 内部 | 否 | 路由页面，无外部 Props |

该文件包含三个导出符号：
- **`ToolKitProvider`**：系统工具市场核心组件，管理工具列表加载、标签筛选、搜索、安装/卸载操作、详情抽屉等全部业务逻辑。接收 `MenuIcon` prop 用于移动端菜单图标。
- **`ContextRender`**：路由入口，包裹 `DashboardContainer` 并将 `MenuIcon` 传递给 `ToolKitProvider`。
- **`getServerSideProps`**：Next.js 服务端渲染函数，加载国际化词条。

### 组件关系说明

- **`DashboardContainer`**：提供工作台统一布局，包括侧边栏导航、背景装饰、移动端适配。通过 render props 模式将 `MenuIcon`、`templateTags`、`templateList` 传递给子组件。
- **`ToolCard`**：工具卡片展示组件，支持展示工具名称、描述、图标、作者、标签、安装状态。提供 `onInstall`、`onDelete`、`onClickCard` 回调。接收 `mode="team"` 控制团队模式下的展示样式。
- **`ToolDetailDrawer`**：工具详情抽屉组件，从右侧滑出展示工具完整信息。通过 `onFetchDetail` 回调获取工具详情数据。支持 `onToggleInstall` 切换安装状态。
- **`MyTabs`**：页面级导航标签，在「我的工具」和「系统工具」之间切换（通过路由跳转实现）。
- **`MyTabBar`**：标签筛选栏，用于工具分类过滤。接收 `tabs` 数组和 `activeKey`，点击标签时触发 `onChange`。
- **`SearchInput`**：搜索输入框，支持实时搜索过滤。

### 内部状态管理

系统工具页面使用以下 React 状态：
- `tools`（useState）：工具列表数据
- `searchText`（useState）：搜索关键词
- `selectedTool`（useState）：当前选中查看详情的工具
- `selectedTagId`（useState）：当前选中的标签筛选 ID
- `loadingPluginIds`（useReducer）：正在执行安装/卸载操作的工具 ID 集合，用于按钮 loading 状态
- `loadingPromisesRef`（useRef）：并发控制 Map，存储每个工具正在执行的异步操作 Promise

### 页面布局结构

```
DashboardContainer
└── ToolKitProvider
    ├── MyTabs（页面导航：我的工具 / 系统工具）
    ├── MyTabBar（标签筛选栏）
    ├── SearchInput（搜索框）
    ├── Grid（工具卡片网格）
    │   └── ToolCard × N
    └── ToolDetailDrawer（工具详情抽屉，条件渲染）
```
