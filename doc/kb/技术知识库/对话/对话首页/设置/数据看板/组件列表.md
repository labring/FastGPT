## 组件详细规格

### `DataDashboard`

- **文件**: `projects/app/src/pageComponents/chat/ChatSetting/DataDashboard.tsx`
- **类型**: React 函数式组件（`React.memo` 包裹）
- **Props**:
  ```typescript
  type Props = {
    Header: React.FC<{ children?: React.ReactNode }>;
  };
  ```
- **State 来源**:
  | 状态 | 来源 | 访问方式 |
  |------|------|---------|
  | `appId` | `ChatPageContext.chatSettings?.appId` | `useContextSelector(ChatPageContext, (v) => v.chatSettings?.appId \|\| '')` |
- **渲染逻辑**:
  1. 从 ChatPageContext 获取 appId
  2. 用 LogsContextProvider 包裹子组件（传入 appId）
  3. 渲染垂直 Flex 布局：Header → LogsFilterBar → LogChart
- **性能优化**: `React.memo` 包裹导出，Header prop 由父组件通过 `useCallback` 缓存

### `LogsFilterBar`

- **文件**: `DataDashboard.tsx:12`（内部，不导出）
- **类型**: React 函数式组件（无 Props）
- **消费的 Context**: `LogsContext`（通过 use-context-selector 精确订阅）
  | 字段 | 类型 | 用途 |
  |------|------|------|
  | `dateRange` | `DateRangeType` | 当前日期范围 |
  | `setDateRange` | `(value: DateRangeType) => void` | 设置日期范围 |
  | `chatSources` | `ChatSourceEnum[]` | 已选渠道来源 |
  | `setChatSources` | `(value: ChatSourceEnum[]) => void` | 设置渠道来源 |
  | `isSelectAllSource` | `boolean` | 是否全选渠道 |
  | `setIsSelectAllSource` | `Dispatch<SetStateAction<boolean>>` | 设置全选状态 |
- **渲染**: `<HeaderControl>` 组件（来自 LogChart），传入上述 6 个属性和 `px={[2, 0]}`
- **与 LogDetails 的差异**: LogDetails 中的 LogsFilterBar 额外消费 `chatSearch`/`setChatSearch` 和 `logKeys`/`setLogKeys` 并渲染搜索框 + LogKeysConfigPopover；DataDashboard 版本更精简，仅做日期和渠道筛选

## 组件通信方式

| 通信方向 | 方式 | 数据 |
|---------|------|------|
| ChatSetting → DataDashboard | Props | `Header` 组件引用 |
| DataDashboard → LogsContextProvider | Props | `appId` |
| DataDashboard → LogChart | Props | `appId` |
| ChatPageContext → DataDashboard | Context (use-context-selector) | `chatSettings.appId` |

| LogsContext → LogChart | Context (use-context-selector) | 筛选状态 |
