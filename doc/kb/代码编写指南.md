# 代码编写指南

> 本文档面向开发者，从现有代码库中提炼编码约定，包含组件结构模板、命名规则、常见反模式和国际化操作指南。

## 编码约定

### 组件结构模板

项目使用 **React 函数组件 (FC)** 模式，基于 Next.js 14 + Chakra UI + TypeScript。以下是分析 `projects/app/src/pages/account/inform.tsx`、`projects/app/src/pages/account/setting.tsx`、`projects/app/src/pages/account/usage/index.tsx` 等代表性组件后归纳的标准结构。

#### 页面组件模板

```tsx
'use client';
// 1. React 核心
import React, { useState, useCallback, useMemo } from 'react';
// 2. Chakra UI 组件
import { Box, Flex, Card, useTheme } from '@chakra-ui/react';
// 3. 项目内部 hooks
import { usePagination } from '@fastgpt/web/hooks/usePagination';
import { useLoading } from '@fastgpt/web/hooks/useLoading';
// 4. 状态管理
import { useUserStore } from '@/web/support/user/useUserStore';
// 5. i18n
import { useTranslation } from 'next-i18next';
// 6. 共享组件
import EmptyTip from '@fastgpt/web/components/common/EmptyTip';
// 7. 页面子组件
import AccountContainer from '@/pageComponents/account/AccountContainer';
// 8. SSR i18n 注入
import { serviceSideProps } from '@/web/common/i18n/utils';

// 9. 组件定义（默认使用 React.memo 或直接 export）
const PageComponent = () => {
  // 9a. hooks（i18n、store、自定义 hooks）
  const { t } = useTranslation();
  const theme = useTheme();
  const { userInfo } = useUserStore();
  const { Loading } = useLoading();

  // 9b. 本地 state
  const [selectedItem, setSelectedItem] = useState<any>(null);

  // 9c. 分页/数据获取 hook（包含 data, isLoading, total, Pagination, getData）
  const {
    data: items,
    isLoading,
    total,
    pageSize,
    Pagination,
    getData,
    pageNum
  } = usePagination(apiMethod, {
    defaultPageSize: 20
  });

  // 9d. 事件处理函数
  const handleClick = useCallback((item) => {
    setSelectedItem(item);
  }, []);

  // 9e. 内联样式对象（置于 return 前）
  const textStyles = {
    title: { color: '#111824', fontSize: 'md', fontWeight: 'bold' }
  };

  // 9f. JSX 渲染
  return (
    <AccountContainer>
      <Flex flexDirection="column" h="100%">
        {/* 主内容区 */}
        {items.map((item) => (
          <Box key={item._id} onClick={() => handleClick(item)}>
            {item.title}
          </Box>
        ))}

        {/* 空态 */}
        {!isLoading && items.length === 0 && (
          <EmptyTip text={t('common:no_data')} />
        )}

        {/* 分页 */}
        {total > pageSize && (
          <Flex mt={4} justifyContent="center">
            <Pagination />
          </Flex>
        )}

        {/* 加载态 */}
        <Loading loading={isLoading && items.length === 0} fixed={false} />
      </Flex>
    </AccountContainer>
  );
};

// 10. SSR 数据注入（页面组件必有）
export async function getServerSideProps(content: any) {
  return {
    props: {
      ...(await serviceSideProps(content, ['namespace1', 'namespace2']))
    }
  };
}

// 11. 默认导出
export default PageComponent;
```

**模板来源**: 基于 `projects/app/src/pages/account/inform.tsx`（InformTable）、`projects/app/src/pages/account/setting.tsx`（Individuation）、`projects/app/src/pages/account/usage/index.tsx`（UsageTable）的结构归纳。三者在 import 顺序、hooks 组织、SSR 注入模式上高度一致。

## 命名规则

以下命名规范从项目 `projects/app/src/` 和 `packages/` 目录实际代码统计得出。

| 类别 | 规范 | 示例 | 说明 |
|------|------|------|------|
| 组件文件 | PascalCase | `ResizableDivider.tsx`、`AIModelSelector.tsx`、`EmptyTip.tsx` | 占约 95%，极少数例外 |
| 页面路由文件 | 全小写 / kebab-case | `index.tsx`、`apikey.tsx`、`fastlogin.tsx` | Next.js 文件系统路由约定 |
| 页面路由目录 | kebab-case | `account/bill/`、`chat/share/`、`dataset/list/` | 占约 75%，存在少量 camelCase 不一致 |
| 页面子组件文件 | PascalCase | `CreateModal.tsx`、`ToolModal.tsx` | 与组件文件一致 |
| 工具函数文件 | `utils.ts`（固定名） | `packages/global/common/file/utils.ts` | 按领域目录放置，不另命名为 `helpers.ts` |
| 工具函数导出 | camelCase（动词开头） | `getHandleId`、`isCSVFile`、`parseI18nString` | 语义清晰，100% 遵循 |
| 类型文件 | `type.ts` / `type.d.ts` | `packages/global/support/user/type.ts` | 类型声明文件统一此模式 |
| 组件目录命名 | 混合（PascalCase 为主） | `Badge/`、`MyImage/`、`SideTabs/` | 顶层分组目录用小写 `common/`、`core/`、`support/` |
| 变量命名 | camelCase | `userInfo`、`isLoading`、`pageNum` | |
| 函数命名 | camelCase（动词开头） | `handleClick`、`onclickSave`、`getData` | 事件处理以 `handle` 或 `on` 开头 |
| 组件命名 | PascalCase | `InformTable`、`AccountContainer`、`EmptyTip` | |
| Store 文件 | `useXxxStore.ts` | `useUserStore.ts`、`useChatStore.ts`、`useSystemStore.ts` | 80% 遵循此模式 |
| Store 导出 | `useXxxStore` | `useUserStore`、`useSystemStore` | 100% 遵循 Zustand hook 风格 |
| CSS class | Chakra UI `sx` prop 内联 | `sx={{ lineHeight: '16px' }}` | 项目不使用 CSS Modules，样式通过 Chakra 组件 props (`px`、`py`、`fontSize`、`color`) 或 inline `sx` 控制 |

## 常见反模式

以下反模式来自 ESLint 配置、`eslint-disable` 注释统计、代码 TODO/FIXME 标记和项目实际代码扫描。

| 反模式 | 说明 | 正确替代 |
|--------|------|---------|
| 忽略 `react-hooks/exhaustive-deps` | 项目中最常被禁用的 ESLint 规则（占所有 disable 的 69%），在 `useEffect`/`useCallback`/`useMemo` 中省略依赖 | 完整声明所有依赖数组，或重构代码以避免不必要的依赖 |
| 使用 `any` 类型 | `@typescript-eslint/no-explicit-any` 被禁用；state 初始化常用 `useState<any>(null)` | 使用明确的类型定义或泛型 `useState<ItemType | null>(null)` |
| 目录命名不一致 | 路由目录混用 camelCase（`templateMarket/`、`mcpServer/`）和 kebab-case（`chat/share/`） | 统一使用 kebab-case 作为路由目录命名 |
| `T` 泛型参数单字母大写 | `outLink/runtime/utils.ts` 中使用 HACK 注释标记 `app: T` 泛型模式 | 使用语义化泛型名，如 `TApp` 或具体接口 |
| 直接修改 props | `@typescript-eslint` 开启了 `consistent-type-imports` 强制使用 `type` 导入类型 | 类型导入使用 `import type { X }` 而非 `import { X }` |
| i18n 占位符残留 | `packages/web/i18n/en/skill.json` 中存在 `"TODO"` 占位文本 | 及时补充英文翻译，或使用 CI 检查阻断未翻译词条 |
| SSR 数据注入省略命名空间 | 页面 `getServerSideProps` 未声明该页实际使用的 i18n 命名空间 | 在 `serviceSideProps` 的第二个参数中列出所有需要的 namespace |
| `eslint-disable-next-line` 单行禁用 | 对需要长期跳过的规则，未添加注释说明原因 | 使用 `// eslint-disable-next-line <rule> -- <reason>` 格式附带说明 |

## 国际化使用指南

> 项目使用 `next-i18next` 实现三语种（简体中文 `zh-CN`、繁体中文 `zh-Hant`、英文 `en`）支持。词条文件统一存放于 `packages/web/i18n/`，共 30 个命名空间。

### 快速上手

**1. 在组件中使用翻译**

```tsx
import { useTranslation } from 'next-i18next';

const MyComponent = () => {
  const { t } = useTranslation();
  return <Box>{t('common:Confirm')}</Box>;
};
```

翻译 key 格式为 `命名空间:键名`，示例：
- `t('common:Cancel')` — 通用 UI 文本
- `t('account_inform:no_notifications')` — 业务模块文本
- `t('common:core.chat.Audio Speech Error')` — 嵌套 key 用点号分隔

**2. 页面组件注入 SSR 翻译**

```tsx
import { serviceSideProps } from '@/web/common/i18n/utils';

export async function getServerSideProps(content: any) {
  return {
    props: {
      ...(await serviceSideProps(content, ['namespace_a', 'namespace_b']))
    }
  };
}
```

> `serviceSideProps` 会自动注入 `common`、`app`、`account`、`account_info` 四个基础命名空间，无需在参数中重复声明。

**3. 新增翻译词条**

1. 确定词条归属的命名空间（参考 `packages/web/i18n/constants.ts` 中的 `I18N_NAMESPACES` 列表）
2. 在 `packages/web/i18n/zh-CN/{namespace}.json` 中添加中文词条
3. 在 `packages/web/i18n/en/{namespace}.json` 中添加英文翻译
4. 在 `packages/web/i18n/zh-Hant/{namespace}.json` 中添加繁体中文翻译
5. 在组件中通过 `t('namespace:key')` 引用

**4. 语言切换**

```tsx
import { useI18nLng } from '@fastgpt/web/hooks/useI18n';

const { onChangeLng, currentLang } = useI18nLng();
// onChangeLng('en') — 切换到英文
// onChangeLng('zh-CN') — 切换到简体中文
```

语言偏好存储在 Cookie (`NEXT_LOCALE`) 和 localStorage 中，页面刷新后保持。

### 常见陷阱

- **避免在模块顶层调用 `t()`**：`useTranslation()` 是 React hook，只能在组件函数体或自定义 hook 内部调用。服务端代码中使用 `packages/global/common/i18n/utils.ts` 中的 `i18nT()` 或 `parseI18nString()`。
- **动态 key 需确保响应性**：`t('common:status.' + status)` 在语言切换时会自动更新，但需确保 `status` 变量本身是响应式的。
- **嵌套 key 使用点号**：`t('common:support.wallet.subscription.mode.Month')` 而非逐层展开。
- **带参数的翻译**：使用 `t('chat:home.tools', { num: selectedTools.length })` 传入插值变量。
- **命名空间必须存在**：`useSafeTranslation`（位于 `packages/web/hooks/useSafeTranslation.ts`）会校验命名空间是否在 `I18N_NAMESPACES_MAP` 中注册，建议使用此 hook 替代原生 `useTranslation` 以提前发现拼写错误。
- **服务端多语言数据**：API 返回的多语言字段使用 `I18nStringType` 类型（`{ en: string, 'zh-CN'?: string, 'zh-Hant'?: string }`），通过 `parseI18nString(data.field)` 按当前语言解析。
