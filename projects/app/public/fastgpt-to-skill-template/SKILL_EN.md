## Usage

All calls are made through `scripts/chat.js`. No direct HTTP handling required.

> `chat()` returns `{ reply, chatId }`:
> - `reply` — the AI response text
> - `chatId` — the session ID used for this call; **pass it back for multi-turn conversations and file uploads**

### Basic Chat

```javascript
const { chat } = require('./scripts/chat');

const { reply } = await chat({ message: 'Hello, please introduce yourself' });
console.log(reply);
```

### Multi-turn Conversation

Pass the `chatId` from the previous response to continue in the same session:

```javascript
const { chat } = require('./scripts/chat');

const first  = await chat({ message: 'First question' });
const second = await chat({ message: 'Continue from above', chatId: first.chatId });
console.log(second.reply);
```

## Response

On success, `chat()` returns `{ reply, chatId }`.

Common error codes:

| Status | Cause | Action |
|--------|-------|--------|
| 401 | Invalid or deleted API Key | Regenerate the Key in FastGPT console |
| 403 | Key not bound to this app | Verify the Key matches the App ID |
| 404 | Application not found | Check the App ID is correct |
| 429 | Too many requests | Wait and retry |
| 500 | Internal server error | Check if FastGPT service is running |

## Notes

1. The API Key is scoped to this application only and cannot access other apps
2. Multi-turn conversations and file uploads must use the same `chatId`
3. Use HTTPS in production environments to ensure transport security
4. To modify application settings, use the FastGPT console
