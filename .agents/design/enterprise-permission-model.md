# 企业权限模型

## 目标

定义企业内部使用 FastGPT 时的资源、角色和权限边界。该模型用于指导商业版配置或社区版二开，不直接假设社区版已具备完整能力。

## 资源类型

| 资源 | 说明 |
| --- | --- |
| Team | 团队或部门空间 |
| User | 用户账号 |
| App | 应用、工作流、工具 |
| Dataset | 知识库 |
| Collection | 知识库文件或分组 |
| APIKey | OpenAPI Key |
| ShareLink | 分享链接、iframe 发布 |
| ModelConfig | 模型配置和供应商 |
| AuditLog | 审计日志 |
| RuntimeConfig | 部署、系统和安全配置 |

## 角色

| 角色 | 说明 |
| --- | --- |
| Owner | 资源所有者，可转移所有权 |
| Admin | 管理资源和协作者，不能绕过审计 |
| Editor | 可编辑内容和配置 |
| User | 可使用资源，但不能修改 |
| Viewer | 只读查看 |
| Auditor | 可查看审计和运行记录 |

## 权限矩阵

| 操作 | Owner | Admin | Editor | User | Viewer | Auditor |
| --- | --- | --- | --- | --- | --- | --- |
| 查看资源 | 是 | 是 | 是 | 是 | 是 | 按审计范围 |
| 使用应用 | 是 | 是 | 是 | 是 | 否 | 否 |
| 编辑应用 | 是 | 是 | 是 | 否 | 否 | 否 |
| 管理协作者 | 是 | 是 | 否 | 否 | 否 | 否 |
| 删除资源 | 是 | 按策略 | 否 | 否 | 否 | 否 |
| 导出数据 | 是 | 按策略 | 按策略 | 否 | 否 | 按审计范围 |
| 查看审计 | 按策略 | 按策略 | 否 | 否 | 否 | 是 |
| 修改模型配置 | 否 | 平台管理员 | 否 | 否 | 否 | 否 |
| 创建 API Key | 是 | 是 | 按策略 | 否 | 否 | 否 |

## 默认策略

### 单团队 POC

1. 全员可使用指定应用。
2. 只有试点负责人可编辑应用和知识库。
3. 只有平台管理员可创建 API Key。
4. 禁止公开分享链接。
5. 审计日志仅平台管理员和安全人员可查看。

### 多部门正式使用

1. 按部门建立团队或空间。
2. 知识库默认只授权本部门。
3. 跨部门使用必须由数据负责人批准。
4. API Key 必须绑定用途、负责人和到期时间。
5. 发布到第三方渠道必须审批。

## 权限回收

触发条件：

1. 离职。
2. 转岗。
3. 项目结束。
4. 资料敏感级别调整。
5. 账号异常。

回收动作：

1. 禁用用户。
2. 删除或停用 API Key。
3. 移除应用和知识库协作者。
4. 转移 Owner。
5. 下线分享链接。
6. 记录审计事件。

## 商业版映射

商业版已有团队、成员组、部门和资源权限能力时，优先映射到商业版：

1. Team -> 团队空间。
2. User -> 成员。
3. 部门/群组 -> 成员组或组织。
4. App/Dataset 权限 -> 资源协作者权限。

## 社区版二开边界

社区版短期只建议实现：

1. 单团队。
2. 管理员/普通用户分级。
3. API Key 创建限制。
4. 分享链接限制。
5. 审计记录。

不建议短期实现完整多部门权限体系。
