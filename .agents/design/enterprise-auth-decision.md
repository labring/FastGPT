# 企业身份与权限方案决策

## 目标

决定 FastGPT 企业内部使用时，身份认证、成员同步与权限管理采用商业版能力，还是在社区版基础上二次开发。

## 当前事实

社区版适合 POC 和小范围部门试用，但企业级能力存在边界：

1. SSO 登录属于商业版能力。
2. 外部成员和组织架构同步属于商业版能力。
3. 团队空间、成员组、部门、细粒度权限属于商业版能力。
4. 管理后台属于商业版能力。

当前社区版已有基础账号、团队和资源权限模型，但缺少企业所需的完整管理闭环。

## 方案 A：采用商业版

### 适用条件

1. 目标是全公司或多部门正式使用。
2. 需要接入公司 IdP、SAML、OAuth、飞书、企业微信或钉钉。
3. 需要组织架构同步、离职停用、权限回收。
4. 需要团队权限、管理后台、发布安全、商业支持。

### 优点

1. 上线周期短。
2. SSO、成员同步、团队权限与管理后台由官方维护。
3. 后续升级成本低于自行重做企业权限体系。
4. 遇到版本升级、数据迁移、权限问题时可获得官方支持。

### 风险

1. 有授权费用。
2. 商业版镜像不可修改，深度定制受限。
3. 仍需要企业侧完成部署、安全、备份、监控和流程治理。

## 方案 B：社区版二开

### 适用条件

1. 只服务单部门或小团队。
2. 身份需求较简单，例如只需要 OIDC 登录和禁用自注册。
3. 接受后续长期维护权限、升级冲突和安全风险。
4. 不需要商业版的团队权限和管理后台。

### 最小二开范围

1. OIDC/OAuth 登录。
2. 禁用自注册。
3. 单团队模式或固定团队。
4. 用户白名单或域名白名单。
5. 离职用户禁用接口。
6. 审计日志。

### 不建议自行重做

1. 完整多团队/多租户。
2. 成员组、部门、组织架构同步。
3. 资源级复杂权限。
4. 管理后台。
5. 发布审批和第三方应用发布。

这些能力和核心业务耦合深，重做后会持续增加升级成本。

## 建议结论

如果目标是企业内部正式平台，优先试用或采购商业版。社区版 fork 继续承担：

1. 企业部署安全基线。
2. 运维 runbook。
3. 审计和监控增强。
4. 公司特定集成。
5. POC 和二开验证。

如果短期预算或采购流程未完成，可以先用社区版做单团队 POC，并严格限制范围：

1. 禁止公网访问。
2. 只开放给试点成员。
3. 不接入高敏感资料。
4. 不承诺全公司权限隔离。

## 决策表

| 需求 | 商业版 | 社区版二开 |
| --- | --- | --- |
| OIDC/OAuth/SAML 登录 | 推荐 | 可做，但需维护 |
| 飞书/企业微信成员同步 | 推荐 | 不建议 |
| 部门/群组权限 | 推荐 | 不建议 |
| 管理后台 | 推荐 | 不建议 |
| 单团队内部 POC | 可用 | 推荐 |
| 企业部署安全基线 | 仍需自建 | 推荐在 fork 中维护 |
| 审计/监控公司定制 | 可补充 | 可补充 |

## 下一步

1. 向 FastGPT 官方确认商业版授权、报价、SSO 支持范围和升级 SLA。
2. 准备一个 2 周 POC：同一份资料分别验证社区版和商业版。
3. 评估公司 IdP：OIDC、SAML、LDAP、飞书、企业微信、钉钉。
4. 若选择社区版二开，先实现最小账号治理，不做完整多租户。
