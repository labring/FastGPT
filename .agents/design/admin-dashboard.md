# 管理后台与运营看板

## 目标

建立企业内部 FastGPT 的管理和运营可视化，让管理员不需要直接查数据库也能掌握平台状态。

## 第一阶段：只读看板

### 核心指标

1. 用户数。
2. 团队数。
3. 应用数。
4. 知识库数。
5. API Key 数。
6. 今日调用量。
7. 今日模型错误率。
8. 今日 token 用量。
9. 知识库导入失败数。
10. 高风险分享链接数量。

### 列表

1. 最近管理员操作。
2. 最近登录失败。
3. 最近发布应用。
4. 即将过期 API Key。
5. 最近知识库同步失败。

## 第二阶段：管理操作

1. 禁用用户。
2. 回收 API Key。
3. 下线分享链接。
4. 暂停应用发布。
5. 冻结知识库。
6. 导出审计日志。

所有管理操作必须写入审计。

## 权限

| 角色 | 权限 |
| --- | --- |
| Platform Admin | 查看和操作全部 |
| Security Auditor | 查看审计和风险 |
| Ops Viewer | 查看运行状态 |
| Team Admin | 查看本团队 |

## 技术路线

如果采用商业版，优先使用商业版管理后台。

如果社区版二开：

1. 第一阶段只做只读 API 和页面。
2. 数据来自 MongoDB 聚合和日志集合。
3. 管理操作分批加入。
4. 所有操作必须调用 enterprise audit。

## 验收

1. 管理员能看到平台基本规模。
2. 安全人员能看到登录失败和高风险发布。
3. 运维能看到错误率和知识库任务失败。
4. 管理操作都有审计记录。
