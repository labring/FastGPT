# 企业内部部署强化设计

## 方案

新增 `deploy/enterprise` 目录，作为企业内部私有化部署的加固入口。该目录不替代官方部署模板，而是在部署前给出安全配置样板和自动检查能力。

## 文件

1. `deploy/enterprise/.env.example`
   - 企业部署环境变量样板。
   - 所有敏感值使用占位符，不提供可直接上线的默认值。
   - 明确启用 IP 限流、内网 IP 检查、可信代理模式、CORS 白名单、登录锁定和密码过期。

2. `deploy/enterprise/check-env.mjs`
   - 读取指定 env 文件。
   - 检查必要变量是否存在。
   - 拦截已知弱默认值和过短密钥。
   - 检查生产域名、对象存储外部地址、CORS、安全开关和沙盒内网访问开关。
   - 输出错误与警告，错误存在时以非 0 退出。

3. `deploy/enterprise/README.md`
   - 描述企业内部部署建议流程。
   - 说明如何生成密钥、运行检查、接入反向代理与 CI。

4. 根 `package.json`
   - 新增 `security:check-env` script。
