# RAG POC 评估集

## 目标

用真实部门资料建立可重复评估集，避免只靠主观体验判断 RAG 效果。

## 评估范围

第一批选择一个部门，一个知识库，一个应用。

建议资料：

1. 制度文件。
2. SOP。
3. 产品说明。
4. 常见问题。
5. 历史案例。

## 问题集

准备 30 到 50 个问题，覆盖：

| 类型 | 数量 | 示例 |
| --- | --- | --- |
| 事实查询 | 10 | 某流程审批时间是多少 |
| 条件判断 | 10 | 什么情况下需要主管审批 |
| 多文档综合 | 10 | A 和 B 流程差异是什么 |
| 边界问题 | 5 | 文件没有写时是否承认不知道 |
| 高风险问题 | 5 | 涉及敏感资料或权限边界 |

## 评估表

| 字段 | 说明 |
| --- | --- |
| `question` | 用户问题 |
| `expectedAnswer` | 标准答案 |
| `sourceDocument` | 来源文件 |
| `sourceSection` | 来源段落或页码 |
| `mustCite` | 是否必须引用 |
| `riskLevel` | low、medium、high |
| `actualAnswer` | 实际回答 |
| `retrievedDocs` | 召回文档 |
| `score` | 0 到 5 |
| `errorType` | recall、reasoning、outdated、hallucination、permission |

## 评分标准

| 分数 | 标准 |
| --- | --- |
| 5 | 正确、完整、有引用 |
| 4 | 正确但不够完整 |
| 3 | 部分正确 |
| 2 | 召回正确但回答错误 |
| 1 | 召回错误或严重遗漏 |
| 0 | 幻觉、越权或危险回答 |

## 通过门槛

POC 通过建议：

1. 平均分 >= 4.0。
2. 高风险问题不得出现 0 分。
3. 事实查询准确率 >= 90%。
4. 必须引用的问题引用命中率 >= 90%。
5. 不知道的问题能明确说不知道。

## 调参记录

每次调整记录：

1. chunk 大小。
2. overlap。
3. embedding 模型。
4. rerank 模型。
5. topK。
6. 相似度阈值。
7. prompt。

## 验收

1. 评估集可以重复运行。
2. 每次变更能比较前后效果。
3. 错误样本能回溯到文档、切块或模型配置。
4. 业务负责人确认标准答案。
