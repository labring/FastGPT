# 内容安全策略

## 目标

定义企业内部 FastGPT 的输入、检索、模型输出和发布渠道安全策略，降低敏感资料外泄、不当输出和越权访问风险。

## 风险分类

| 风险 | 说明 |
| --- | --- |
| 敏感资料外泄 | 回答中包含个人资料、客户资料、密钥、内部财务等 |
| 越权回答 | 用户取得无权限知识库内容 |
| 幻觉决策 | 模型编造政策、流程或数据 |
| 危险操作 | 引导绕过安全、攻击内网、生成恶意代码 |
| 不当内容 | 仇恨、骚扰、成人、暴力等内容 |

## 输入策略

1. 对用户输入做敏感关键词和正则扫描。
2. 对高风险输入打标，而不是一律阻断。
3. 对明显攻击请求直接拒绝，例如索取系统 prompt、密钥、绕过权限。
4. 对包含个人资料或客户资料的问题记录审计事件。

## 检索策略

1. 应用只能绑定授权知识库。
2. 高敏感知识库默认不允许公开分享。
3. 检索结果必须保留来源引用。
4. 不允许跨部门知识库默认混用。
5. RAG prompt 必须要求不知道时回答不知道。

## 输出策略

1. 输出前检测敏感字段：身份证、手机号、邮箱、密钥、token、客户编号等。
2. 对高风险输出执行拦截或替换。
3. 对政策、法务、财务、人事类回答强制带引用。
4. 对低置信度回答提示用户核对来源。
5. 对触发拦截的内容写入审计。

## 审核动作

| 级别 | 动作 |
| --- | --- |
| low | 放行并记录 |
| medium | 放行但提示风险，记录审计 |
| high | 拦截，提示联系管理员 |
| critical | 拦截，告警安全人员 |

## 第一阶段实现

1. 配置高风险关键词表。
2. 在应用级 prompt 中加入引用和不知道策略。
3. 对分享链接和 API 调用增加审计。
4. 接入统一模型网关或 AIProxy 的内容审核能力。

## 验收

1. 高风险问题不会泄露密钥或敏感资料。
2. 无来源的问题会提示无法确认。
3. 拦截事件能在审计日志中查询。
4. 业务负责人确认高敏感知识库不会被公开应用调用。
