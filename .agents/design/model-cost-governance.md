# 模型与成本治理

## 目标

控制企业内部模型调用范围、成本、质量和风险，避免模型滥用或预算失控。

## 治理维度

| 维度 | 说明 |
| --- | --- |
| 模型白名单 | 应用可以使用哪些模型 |
| 用量额度 | 用户、团队、应用、API Key 的额度 |
| 成本归属 | token 与费用归属到团队和应用 |
| 质量评估 | 回答质量、失败率、延迟 |
| 供应商治理 | Azure OpenAI、OpenAI、本地模型、模型网关 |

## 模型分层

| 层级 | 用途 |
| --- | --- |
| 标准模型 | 普通问答和工作流 |
| 高能力模型 | 高价值应用，需审批 |
| embedding 模型 | 知识库索引 |
| rerank 模型 | 检索增强 |
| 本地模型 | 高敏感资料或离线场景 |

## 额度策略

第一阶段：

1. 应用级 QPM。
2. API Key 到期时间。
3. API Key 最大用量。
4. 团队月度 token 预算。

第二阶段：

1. 用户额度。
2. 部门额度。
3. 高峰限流。
4. 自动降级模型。

## 成本字段

每次模型调用建议记录：

1. `teamId`
2. `tmbId`
3. `appId`
4. `apiKeyId`
5. `model`
6. `provider`
7. `inputTokens`
8. `outputTokens`
9. `totalTokens`
10. `estimatedCost`
11. `latencyMs`
12. `result`

## 告警

1. 单应用日用量超过预算 80%。
2. 单用户异常高频调用。
3. 模型错误率超过 10%。
4. 单次调用 token 超过阈值。
5. 高成本模型被未授权应用调用。

## 验收

1. 能按团队、应用、模型查看用量。
2. API Key 有到期时间和负责人。
3. 高成本模型不能被任意应用使用。
4. 预算超限有告警和处理动作。
