#### 【问题】
系统工具管理页面选择导入工具时报错，页面弹出错误提示。

#### 【问题原因】
`ImportPluginModal.tsx` 组件在打开时自动调用 `getMarketPlaceToolTags` 获取 marketplace 工具标签列表。该请求通过 `useRequest` hook 发起，而 `useRequest` 默认 `errorToast: 'Error'`，当请求失败时会自动弹出错误 toast。

在测试环境中，marketplace 服务可能未启动或网络异常，导致 `GET /marketplace/api/tool/tags` 请求返回 500 错误，前端因此弹出报错提示。但标签获取失败不应影响导入工具的核心功能，也不应该报错打断用户操作。

#### 【修复方案】
在 `ImportPluginModal.tsx` 中，`useRequest(getMarketPlaceToolTags)` 调用增加 `errorToast: ''` 参数，使标签请求失败时静默处理，不再弹出错误 toast。

修改文件：`projects/app/src/pageComponents/config/ImportPluginModal.tsx`

```diff
   const { data: allTags = [] } = useRequest(getMarketPlaceToolTags, {
-    manual: false
+    manual: false,
+    errorToast: ''
   });
```

当 `allTags` 为空时，标签映射逻辑会回退为空字符串，标签列显示为 `-`，不影响文件上传、解析和确认导入等核心流程。

#### 【影响模块】
- **主要影响**：`projects/app/src/pageComponents/config/ImportPluginModal.tsx` — 导入工具弹窗
- **功能影响**：仅影响 tags 请求失败时的错误提示行为，核心导入功能不受影响
- **风险等级**：🟢 低

#### 【测试建议】
1. 在 marketplace 服务不可用的环境下打开导入工具弹窗，确认不再弹出错误 toast
2. 正常导入 .pkg 文件，确认文件上传、解析、标签显示、确认导入流程正常
3. 在 marketplace 服务可用的环境下，确认标签正常显示
