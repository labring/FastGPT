#### 【问题】
系统工具管理 -> 自定义工具配置弹窗中，字段标签为"关联工具"，但未选择工具点击确认时，报错提示为"关联应用不能为空"。文案与实际字段名称不一致，用户无法理解。

#### 【问题原因】
i18n 国际化词条 `custom_plugin_associated_plugin_required` 在三语种中均使用了"应用"(application)相关词汇，而实际 UI 标签为"关联工具"，导致报错文案与界面字段名称不匹配。

#### 【修复方案】
修改三语种 i18n 文件中的 `custom_plugin_associated_plugin_required` 词条：
- zh-CN: "关联应用不能为空" → "关联工具不能为空"
- zh-Hant: "關聯應用不能為空" → "關聯外掛不能為空"
- en: "Associated applications cannot be empty" → "Associated tool cannot be empty"

#### 【影响模块】
- `packages/web/i18n/zh-CN/app.json`
- `packages/web/i18n/zh-Hant/app.json`
- `packages/web/i18n/en/app.json`
- 调用位置：`projects/app/src/pageComponents/config/tool/WorkflowToolConfigModal.tsx:186`

#### 【测试建议】
1. 打开系统工具管理 -> 自定义工具配置弹窗
2. 不选择任何工具，直接点击确认
3. 验证报错提示为"关联工具不能为空"（或对应语种的正确文案）
