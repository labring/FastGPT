#### 【问题】
应用市场点击"立即创建"按钮后，应用列表多出两个相同的模板应用。

#### 【问题原因】
`AppMarketCard` 组件中，`MyCard` 整体绑定了 `onClick` 事件，同时悬浮操作区 `hoverAction` 内的 `Button` 也绑定了 `onClick` 事件。点击按钮时，按钮自身的 `onClick` 先触发，随后事件冒泡到父级 `MyCard` 的 `onClick`，导致 `onUseTemplate` 被调用两次，后端接口被请求两次，创建出两个相同的应用。

#### 【修复方案】
在 `Button` 的 `onClick` 处理器中添加 `e.stopPropagation()`，阻止事件继续冒泡到 `MyCard`。

修改文件：`projects/app/src/pageComponents/dashboard/templateMarket/AppMarketCard.tsx`

```tsx
// 修改前
onClick={() => onClick(item)}

// 修改后
onClick={(e) => {
  e.stopPropagation();
  onClick(item);
}}
```

#### 【影响模块】
- `projects/app/src/pageComponents/dashboard/templateMarket/AppMarketCard.tsx` — 应用市场卡片组件

#### 【测试建议】
1. 进入应用市场，点击任意模板的"立即创建"按钮
2. 确认只创建一个应用，应用列表中无重复
3. 同时验证点击卡片非按钮区域也能正常创建应用（MyCard 的 onClick 不受影响）
