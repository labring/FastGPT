#### 【问题】

系统工具管理页面中，"选择现有应用"按钮文案不准确。用户点击后进入的是自定义工具配置页面，没有"选择应用"的地方，文案与实际功能不符，用户无法理解该功能的用途。

#### 【问题原因】

i18n 词条 `toolkit_select_app` 使用了"应用"一词，但该功能实际上是选择已有的工作流/工具，而非应用。文案表述与实际功能不匹配，造成用户困惑。

#### 【修复方案】

将三语种 i18n 词条中的"应用"改为"工具"：

- `packages/web/i18n/zh-CN/app.json`: `"选择现有应用"` → `"选择现有工具"`
- `packages/web/i18n/zh-Hant/app.json`: `"選擇現有應用"` → `"選擇現有工具"`
- `packages/web/i18n/en/app.json`: `"Select an existing app"` → `"Select an existing tool"`

#### 【影响模块】

- 系统工具管理页面 (`/config/tool`)
- 仅影响菜单按钮文案，不影响任何功能逻辑

#### 【测试建议】

1. 访问系统工具管理页面
2. 点击"添加资源"下拉菜单
3. 确认第三项显示为"选择现有工具"（原"选择现有应用"）
4. 切换语言（中/英/繁中），确认各语言文案正确
