# TD 2026041200014 修复文档

#### 【问题】
工作流配置混合检索后，完整响应中"检索方式"字段显示比例异常：
`混合检索(语义检索：全文检索 = 0.67：0.3299999999999999996)`，两比例相加不等于 1。

#### 【问题原因】
`WholeResponseModal.tsx` 第 549 行直接使用 `1 - activeModule.embeddingWeight` 进行浮点运算，
触发 JavaScript 浮点精度问题（`1 - 0.67 = 0.32999999...`），未做精度截断处理。

#### 【修复方案】
文件：`projects/app/src/components/core/chat/components/WholeResponseModal.tsx`（Line 546-551）

对 `emb` 和 `text` 两个参数均使用已有工具函数 `formatNumber(val, 100)` 保留 2 位小数：

```typescript
// 修改前
{
  emb: activeModule.embeddingWeight,
  text: 1 - activeModule.embeddingWeight
}

// 修改后
{
  emb: formatNumber(activeModule.embeddingWeight, 100),
  text: formatNumber(1 - activeModule.embeddingWeight, 100)
}
```

#### 【影响模块】
- `WholeResponseModal.tsx`：仅影响完整响应弹窗中混合检索权重的显示格式，无逻辑变更。

#### 【测试建议】
1. 配置混合检索比例（如 0.67:0.33），运行工作流后打开完整响应，确认显示为 `0.67 : 0.33`。
2. 测试边界值：0.5:0.5、0.9:0.1、0.1:0.9，确认各比例显示正确。
3. 切换语言（中/英/繁体）确认显示均正常。
