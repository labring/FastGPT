#### 【问题】
工作流知识库检索节点（datasetSearchNode）模型参数未下发：
1. 重排模型（rerankModel）UI 上显示了默认值，但保存时参数为空
2. 向量模型（embeddingModel）初次选择知识库后不自动配置

#### 【问题原因】
**rerankModel 未下发**：
`SelectDatasetParams.tsx` 中 `data.rerankModel` 初始化为 `defaultModels.rerank?.model`（保证 UI 有显示），但 `inputs[rerankModel].value` 在新建节点时为 `undefined`。inputs useEffect 中使用 `input.value ?? state[input.key]`，当 `input.value` 为 undefined 时不写回 inputs，导致保存时读取的 inputs 值仍是 undefined，参数未下发。

**embeddingModel 不自动配置**：
知识库变更 effect 的条件 `prev !== undefined && prev !== datasetVectorModel` 中，`prev !== undefined` 阻止了初次选择知识库（`prev === undefined`）时的更新逻辑。

#### 【修复方案】
**文件**：`projects/app/src/pageComponents/app/detail/WorkflowComponents/Flow/nodes/render/RenderInput/templates/SelectDatasetParams.tsx`

**rerankModel 修复**：
- 添加 `hasInitializedRerankModelRef` ref 标记，在 inputs useEffect 首次找到 rerankModel input 时执行一次初始化
- 若 `rerankModelInput.value` 为空且有默认 rerank 模型，调用 `onChangeNode` 将默认值写入 inputs

**embeddingModel 修复**：
- 将原条件替换为 `isFirstSelect || isSwitchDataset`
- `isFirstSelect`：prev === undefined（初次）且 inputs 中 embeddingModel 无已保存值（新节点）→ 自动配置
- `isSwitchDataset`：prev !== undefined 且知识库切换 → 更新

#### 【影响模块】
- 工作流编辑器 - 知识库检索节点参数配置弹窗

#### 【测试建议】
1. 新建工作流 → 添加知识库检索节点 → 选择有向量模型的知识库 → 验证向量模型自动填充
2. 新建工作流 → 添加知识库检索节点（不修改 rerank 模型）→ 保存 → 重新打开验证 rerankModel 有值
3. 编辑已保存工作流 → 已配置 embeddingModel 的节点 → 不改知识库 → 验证 embeddingModel 不被覆盖
4. 切换知识库（从知识库A切换到知识库B）→ 验证向量模型自动更新为新知识库的向量模型
