# TD 2026052000305 修复记录

#### 【问题】
对基于bge模型的知识库进行 Qwen3-Embedding 类模型训练后，训练结果（Precision@10、MRR@10）全部显示为 0.0%/0.00。

#### 【问题原因】
`BaseModelTrainModal` 组件在知识库选择时未校验向量模型兼容性，允许用户选择任意知识库进行训练。当所选知识库的 `vectorModel.model` 与当前训练的基础模型（`selectedBaseModel`）不一致时，训练数据无法被正确处理，导致结果全为 0.0。

#### 【修复方案】
在 `BaseModelTrainModal.tsx` 中添加向量模型兼容性检查：
- 新增 `isDatasetDisabled` 函数：仅对 `ModelTypeEnum.embedding` 类型的训练检查，当 `item.vectorModel?.model !== selectedBaseModel` 时返回 `true`
- 对禁用知识库：展示但禁用（`opacity: 0.5`、`cursor: not-allowed`、`isDisabled` Checkbox）；鼠标悬浮显示 Tooltip 提示"该知识库的向量模型与本模型不同，不支持训练"
- 修改 `toggleFolderSelection`、`getFolderCheckState`、`isAllSelected`、`handleSelectAll`，均过滤掉不兼容的知识库，确保批量操作不会选入禁用项
- 添加三语种 i18n 词条 `train_dataset_vector_model_mismatch`

#### 【影响模块】
- `projects/app/src/components/core/ai/ModelTable/BaseModelTrainModal.tsx`
- `packages/web/i18n/zh-CN/account_model.json`
- `packages/web/i18n/en/account_model.json`
- `packages/web/i18n/zh-Hant/account_model.json`

#### 【测试建议】
1. 打开 embedding 模型训练弹窗，验证与当前模型向量不同的知识库显示灰色禁用状态
2. 悬浮禁用知识库，验证 Tooltip 提示文字正确显示
3. 验证批量选择（全选/文件夹选择）不会选入不兼容的知识库
4. 验证与当前模型向量相同的知识库可正常选择
5. 验证 rerank 类型训练不受影响（所有知识库均可选）
